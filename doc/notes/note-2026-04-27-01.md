# Violations report — `connections` 0.1.0 publish-readiness audit

**Date:** 2026-04-27
**Source:** Three parallel read-only audits (`src/`, `tests/`, repo config) plus
spot verification against `CLAUDE.md` and the current state of `main`.
**Context:** Pre-publication audit ahead of the first crates.io release.
The implementation sprint that addresses these findings is
[Plan 15](../plans/plan-2026-04-27-03.md).

The crate is **not ready to publish as-is**. There are 7 blockers, 10
warnings, and 3 nits. None of the blockers are deep — they are
metadata, missing files, and two pockets of property-test coverage
gaps. Total fix budget: ~6–8 hours of focused work.

The src/ audit's verdict that the crate is "publish-ready as-is" was
optimistic: it missed the float-side proptest gap, did not check
crates.io metadata, and (separately) flagged time-conn names as
violations when the all-letter `ABCDXYZW` shape is in fact permitted
(per Chris, 2026-04-27).

---

## Blockers (must fix to publish)

### Packaging / metadata

- **B1 — Cargo.toml missing publish metadata.** `Cargo.toml` lacks
  `repository`, `homepage`, `keywords`, `categories`, and an explicit
  `readme = "README.md"`. crates.io renders these on the crate page
  and uses `keywords` / `categories` for discoverability.
- **B2 — No LICENSE file.** `license = "MIT"` is set but no
  `LICENSE` / `LICENSE-MIT` file exists at the repo root. crates.io
  packages the license file into the published `.crate` archive; its
  absence is a packaging error and a legal issue.
- **B3 — No `[package.metadata.docs.rs]` section.** Without
  `all-features = true`, `property::arb` (gated on `testing`) is
  invisible on docs.rs. Without `rustdoc-args = ["--cfg", "docsrs"]`,
  `#[cfg_attr(docsrs, doc(cfg(feature = "testing")))]` annotations
  cannot signal feature-gating to readers.
- **B4 — Public canonical URL not verified.** The git remote is
  `gitlab.com:cmk/connections.git` (private). crates.io requires a
  publicly-reachable `repository` URL. Confirm the GitHub mirror is
  public and current, or make the GitLab project public.
- **B5 — `cargo publish --dry-run` not run.** Catches packaging
  problems (missing includes, oversized archives, etc.) before they
  hit the registry.

### Test coverage gaps

- **B6 — Float Conns lack predicate-based Galois proptests.**
  `F064F032`, `F064F016`, `F064B016`, `F032F016`, `F032B016` (in
  `src/conn/float/{f32,f64}.rs`) have walk-correctness loops inside
  their ceil/floor implementations but no `proptest!` blocks calling
  `crate::property::laws::*`. The walks prove local correctness for
  any single value; they do not prove the adjoint laws hold over
  random pairs `(a, b)`. Float precision is the crate's headline
  feature — Galois-law violations here would be embarrassing.

- **B7 — Decimal `float_conn!` macros lack predicate-based Galois
  proptests.** `F064FD00`, `F064FD01`, `F064FD02`, `F064FD03`,
  `F064FD06`, `F064FD09`, `F064FD12` (in `src/conn/fixed/decimal.rs`,
  generated by the `float_conn!` macro at lines 284–290) are the sole
  bridges between IEEE doubles and the decimal-SI time ladder. Same
  issue as B6 — walks are local, laws are not asserted.

## Warns (should fix; non-blocking but expected for a 0.1.0)

### Structure

- **W1 — `conn::int` / `conn::uint` placement is irregular.** Every
  other Conn family (`fixed`, `float`, `time`, `sample`) is a
  parent-`.rs` + same-name directory. `int.rs` and `uint.rs` are bare
  top-level files. Recommendation: introduce
  `conn::primitive::{int, uint}` with a parent `primitive.rs` so the
  pattern is uniform. Folding into `conn::fixed` is wrong — these are
  primitive-int ladders, not fixed-point.

- **W2 — Test-location split is arbitrary.**
  `tests/conn_fixed_u{08,16,32,64,128}_galois.rs` exist as integration
  tests but the parallel signed families (`fixed::i{08,16,32,64,128}`)
  and every other family (`fixed::decimal`, `float::*`, `int`, `uint`,
  `sample`, `time::*`) keep proptests colocated. The split was driven
  by per-family memory budget, not policy. Pick one rule and apply
  uniformly. Preferred: integration tests for everything (decouples
  test code from `pub(crate)` types and bounds the lib-test binary).

- **W3 — Move `conn::sample` downstream.** Dependency analysis is
  clean: `sample` depends inward on `Conn` + `FD12` only; nothing in
  this crate depends back on `sample`. Lift it into a sibling crate
  before publish so the v0.1.0 surface is "Galois connections + the
  numeric ladders that motivate them", with audio-specific adapters
  living elsewhere.

### CI / docs

- **W4 — `deny` + `gitleaks` run on every push.** Each push pays ~2
  minutes for audits that don't change between sibling commits.
  Schedule them nightly or gate on `$CI_COMMIT_BRANCH == "main"`.
  (Per Chris's brief: "perhaps only run deny once".)
- **W5 — No `cargo doc -D warnings` CI job.** Broken intra-doc links
  and rustdoc warnings only surface when docs.rs builds the published
  release. Add a `doc` job to the `check` stage running
  `cargo doc --no-deps --all-features --document-private-items -- -D warnings`.
- **W6 — 12 redundant rustdoc link-target warnings.** From
  `cargo doc --all-features` — module names like `i16` collide with
  primitive types. Cosmetic but easy to fix; W5 will gate this from
  recurring.
- **W7 — `scripts/check_readme_mirror.sh` is not invoked.** It checks
  that `F064B016`'s README block matches the doctest in source. Wire
  into the pre-commit hook (it lives in `.claude/settings.json`) or
  the `check` CI stage.
- **W10 — Feature-gated items lack docs.rs `cfg(feature = …)`
  annotations.** `property::arb` should be marked
  `#[cfg_attr(docsrs, doc(cfg(feature = "testing")))]` so docs.rs
  renders the gating badge. Pairs with B3.

### Documentation

- **W8 — README missing explicit MSRV statement.** `Cargo.toml` says
  `rust-version = "1.85"` but the README doesn't surface it. Add a
  single sentence under a "Requirements" or "MSRV" heading.
- **W9 — No `CHANGELOG.md`.** Conventional for libraries; useful for
  downstream upgraders. A single 0.1.0 entry is enough at this point.

## Nits

- **N1 — GitLab Pages docs hosting.** Skip for 0.1.0; docs.rs is
  free, automatic, and crates.io-integrated. Revisit if branded docs
  become important post-1.0.
- **N2 — Decimal cross-tier coverage table** in
  `src/conn/fixed/decimal.rs` is implicit through `fix_fix!`
  expansion. Not a real gap — the macro guarantees uniform coverage —
  but a one-paragraph comment listing the 21 covered pairs would help
  future auditors confirm at a glance.
- **N3 — Float special-value spot tests.** `Bot` / `Top` / `NaN` /
  `±∞` / subnormals are handled correctly by inspection of the walk
  loops, but no spot tests force these branches. Once B6's proptest
  coverage lands, the strategies in `crate::property::arb::arb_f64`
  already cover NaN / ±∞; subnormal bias would still be a nice add.

## Items already clean (audit confirmed)

- No `mod.rs` antipatterns. `src/property/mod.rs` is the lone `mod.rs`
  and is appropriately leaf-only.
- No `unsafe` (forbidden by `#![forbid(unsafe_code)]`).
- No `#[ignore]`d tests.
- No leftover `TODO` / `FIXME` / `HACK` comments.
- No hand-rolled Galois-law assertions inside test modules — every
  `#[cfg(test)]` block delegates to `crate::property::laws::*` and
  draws strategies from `crate::property::arb`. (The float and
  decimal-float gaps in B6/B7 are about *missing* proptest blocks,
  not non-conformant ones.)
- Conn-name compliance is clean across `fixed::*`, `float::*`,
  `int`, `uint`, `sample`. The all-letter `ABCDXYZW` shape used in
  `conn::time` (`DATEJDAY`, `TIMENANO`, `TIMESECS`, `DURNSECS`,
  `PDTMDATE`) is permitted; CLAUDE.md's "reserved — no current uses"
  text is stale.

## Top 3 unblockers, in order

1. **Land B1–B5** (publish metadata + LICENSE + dry-run). 90 minutes,
   pure mechanical.
2. **Close B6 + B7** (float and decimal-float Galois proptests). 2–3
   hours; reuses `arb::arb_f64` / `arb::arb_f32` and the existing
   `laws::conn_galois_l`/etc. predicates.
3. **Pick the test-location policy and apply it (W2).** Coupled with
   the `conn::primitive` move (W1) and the `sample` lift-out (W3)
   it's a single restructuring sprint; defer if a faster
   "metadata-only" 0.1.0 is preferred and ship structure work in
   0.2.0.

The publish-prep sprint sequences these to land structure and tests
before metadata, with the metadata fixes as the final commit before
`cargo publish`. See [Plan 15](../plans/plan-2026-04-27-03.md).
