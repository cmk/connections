# Plan 11 — Drop `Ple`; adopt `Eq + PartialOrd`

## Goal

Replace the crate-local `Ple` trait with the standard-library bound
`Eq + PartialOrd`. Net new trait surface in this crate: zero. The
math is unchanged — every type that was `Ple`-able now satisfies
the lawful framework — but the framing is now standard Rust idiom
that any Rustacean reads without ceremony.

## Locked design decisions

### D1. The lawful framework is `T: Eq + PartialOrd`

- `Eq` certifies reflexive `PartialEq` — the actually-broken-for-floats
  trait, fixed at the wrapper boundary.
- `PartialOrd` derives a genuine partial order from reflexive
  `PartialEq`. Combined: a true partial order in the math sense,
  contains every preorder Galois laws need.
- Net new traits in the crate: zero.

### D2. `impl Eq for ExtendedFloat<T>` is the entire float fix

The `impl_float_ext!` macro at `src/conn/float.rs:32-78` already
patches `PartialEq` so `Finite(NaN) == Finite(NaN)`. Adding the
empty marker `impl Eq for ExtendedFloat<$T> {}` certifies that to
the type system; floats now flow through every `T: Eq + PartialOrd`
predicate.

### D3. Raw `f32`/`f64` are no longer Conn endpoints

Raw floats can't impl `Eq` (NaN ≠ NaN under standard `PartialEq`),
so they cannot satisfy the law-machinery bound. The single
raw-float Conn (`f64_f32() -> Conn<f64, f32>`) is migrated to a
const `F064F032: Conn<ExtendedFloat<f64>, ExtendedFloat<f32>>`.
Callers wrap inputs at the boundary: `F064F032.ceil(ExtendedFloat::Finite(x))`.

f128 / f16 cross-tier conns (`F128F064`, `F032F016`, etc.) remain
deferred until `f128`/`f16` stabilise past the Rust 1.85 toolchain
pin.

### T3 — `task: Replace Ple bounds in laws.rs with Eq + PartialOrd`

`src/property/laws.rs`:
- Every `T: Ple` / `Copy + Ple` → `T: Eq + PartialOrd` /
  `Copy + Eq + PartialOrd` (Galois-law predicates), or `T: PartialOrd`

  the standard `x ≤ y ∧ y ≤ x ⟹ x = y` law works under proper Eq.
