# Plan 36 — `ConnL`/`ConnR`/`ConnK` rename + README/rustdoc refactor

## Goal

Unify the `ViewL`/`ViewR` capability traits with the `ConnL`/`ConnR`
value-type aliases under one name each (`ConnL`, `ConnR`, plus
`ConnK` replacing `Triple`); drop pre-1.0 back-compat aliases
(`ceiling`, `inner`); reorder README + module preambles so an
everyday reader meets `.ceil`/`.upper`/`.floor`/`.lower` before any
trait name. Same expressive power, fewer concepts to learn.

## Tasks

### T1 — Reshape `ViewL`/`ViewR` into `ConnL`/`ConnR` traits

`src/conn.rs`:

- Drop `pub type ConnL<A, B> = Conn<A, B, L>;` (line 73) and the
  `ConnR` analogue.
- Rename trait `ViewL<A, B>` → `ConnL<A, B>`; same for R.
- Replace the `const L: ConnL<A, B>` associated const with a
  `fn conn_l(&self) -> Conn<A, B, L>` method. Const slots in trait
  impls can't reference `self`, so the blanket impl on the value
  type requires a method shape.
- Trait surface (no back-compat aliases pre-1.0):
  ```rust
  pub trait ConnL<A: Copy, B: Copy> {
      fn conn_l(&self) -> Conn<A, B, L>;
      fn ceil(&self, a: A) -> B { self.conn_l().ceil(a) }   // lower adjoint
      fn upper(&self, b: B) -> A { self.conn_l().upper(b) } // upper adjoint
  }
  pub trait ConnR<A: Copy, B: Copy> {
      fn conn_r(&self) -> Conn<A, B, R>;
      fn floor(&self, a: A) -> B { self.conn_r().floor(a) } // upper adjoint
      fn lower(&self, b: B) -> A { self.conn_r().lower(b) } // lower adjoint
  }
  ```
  - `ConnL`: `ceil` (rounds up; lower adjoint = `f`), `upper` (embeds
    back; upper adjoint = `g`).
  - `ConnR`: `floor` (rounds down; upper adjoint = `f`), `lower`
    (embeds back; lower adjoint = `g`).
- Blanket impls so the value type satisfies the trait:
  ```rust
  impl<A: Copy, B: Copy> ConnL<A, B> for Conn<A, B, L> {
      fn conn_l(&self) -> Conn<A, B, L> { *self }
  }
  impl<A: Copy, B: Copy> ConnR<A, B> for Conn<A, B, R> {
      fn conn_r(&self) -> Conn<A, B, R> { *self }
  }
  ```
- Trim inherent methods on `Conn<A,B,L>` to `ceil` + `upper` (drop
  `ceiling` and `inner` aliases). Same on `Conn<A,B,R>`: `floor` +
  `lower` (drop `inner`). Inherent wins method resolution, so direct
  calls on a value still bypass the trait.
- Update two-sided helpers (`round`, `truncate`, `interval`,
  `midpoint`, `median`): replace `<T as ViewL>::L` with `t.conn_l()`,
  `<T as ViewR>::R` with `t.conn_r()`.

### T1b — Rename `Triple` to `ConnK`

`src/conn.rs:386`:

- `pub trait Triple<A,B>: ViewL<A,B> + ViewR<A,B>` becomes
  `pub trait ConnK<A,B>: ConnL<A,B> + ConnR<A,B>`.
- Update all bounds, doc references, and re-exports.
- The blanket `impl<T> Triple for T where T: ViewL + ViewR` becomes

### T4 — README refactor

`README.md` (currently 682 lines):

- Insert "Get started in 30 seconds" block before "Why this crate":
  ```rust
  use connections::time::DURNSECS;
  let half = Duration::seconds(5) + Duration::nanoseconds(1);
  assert_eq!(DURNSECS.ceil(half),  Extended::Finite(6));
  ```

  as Conns themselves. One concrete example.

  connections work."
- Substitute `.inner(...)` per receiver: `.upper(...)` for L-side or
  triple Conns (most), `.lower(...)` for R-side. Update `ConnL/ConnR`
