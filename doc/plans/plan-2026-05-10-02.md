# Plan 2026-05-10-02 — `ConnL` / `ConnR` to associated types

## Goal

Reshape `ConnL` and `ConnR` so the `(A, B)` Galois pair lives in the
trait body as associated types rather than as trait type parameters.
This aligns the trait with the functional dependency `T → (A, B)`
that the library has always honored in practice, and as a side
effect erases the rustdoc-impl-header alias-normalization noise that
made the MR !92 batch render `Extended<FixedI16<UInt<UInt<UInt<UInt<UTerm,
B1>, B1>, B0>, B0>>>` instead of `Extended<FixedI16<U12>>`.

## Why now

Pre-1.0; no downstream stability cost. Today's trait shape allows
multi-instance impls (`T: ConnL<A1, B1>` *and* `T: ConnL<A2, B2>` on
the same `T`) that no caller in this crate or downstream uses or
wants. The shape is over-general; the fix is structural.

## Tasks

### T1 — Reshape `ConnL` / `ConnR`

Replace type parameters with associated types in `src/conn.rs`:

```rust
pub trait ConnL {
    type A: Copy;
    type B: Copy;
    fn conn_l(&self) -> Conn<Self::A, Self::B, L>;
    fn ceil(&self, a: Self::A) -> Self::B { self.conn_l().ceil(a) }
    fn upper(&self, b: Self::B) -> Self::A { self.conn_l().upper(b) }
}

pub trait ConnR {
    type A: Copy;
    type B: Copy;
    fn conn_r(&self) -> Conn<Self::A, Self::B, R>;
    fn floor(&self, a: Self::A) -> Self::B { self.conn_r().floor(a) }
    fn lower(&self, b: Self::B) -> Self::A { self.conn_r().lower(b) }
}
```

`Conn<A, B, K>` (the value type) keeps its current type parameters —
only the *capability traits* move to associated types.

### T2 — Blanket impls on `Conn<A, B, K>`

```rust
impl<A: Copy, B: Copy> ConnL for Conn<A, B, L> {
    type A = A;
    type B = B;
    fn conn_l(&self) -> Conn<A, B, L> { *self }
}
impl<A: Copy, B: Copy> ConnR for Conn<A, B, R> {
    type A = A;
    type B = B;
    fn conn_r(&self) -> Conn<A, B, R> { *self }
}
```

### T3 — `ConnK` super-trait

Express the cross-trait equality that was implicit in the
type-parameter form:

```rust
pub trait ConnK: ConnL
    + ConnR<A = <Self as ConnL>::A, B = <Self as ConnL>::B>
{}

impl<T> ConnK for T
where
    T: ConnL + ConnR<A = <T as ConnL>::A, B = <T as ConnL>::B>,
{}
```

This is strictly *more* expressive than today: today the consistency
between an `L` impl's `(A, B)` and an `R` impl's `(A, B)` lives only
inside the `where` clauses of helper fns; under the new shape it
lives in the trait definition.

### T4 — Codegen macros

Update `conn_k!` (`src/conn.rs`), `iso!` (delegates to `conn_k!`),
`compose_k!` (lattice/composition), and `float_fixed!`
(`src/fixed.rs`). The new emission shape is:

```rust
$vis struct $name;
impl $crate::conn::ConnL for $name {
    type A = $A;
    type B = $B;
    fn conn_l(&self) -> $crate::conn::Conn<$A, $B, $crate::conn::L> { … }
}
impl $crate::conn::ConnR for $name {
    type A = $A;
    type B = $B;
    fn conn_r(&self) -> $crate::conn::Conn<$A, $B, $crate::conn::R> { … }
}
```

The `$A` / `$B` substitutions are unchanged from today — they still
get spliced from the macro args. The rustdoc win comes from those
types disappearing from the impl *header* (which rustdoc fully
normalizes) and reappearing in the impl body and method signatures.
Method signatures render alias-preserving (`Conn<…,
Extended<FixedI16<U12>>, L>`); the associated-type *assignment*
lines (`type B = …`) still expand in rustdoc 1.85 but are
secondary to the headline impl header.

### T5 — Helper fns and prop predicates

Every `T: ConnK<A, B>` bound in `src/conn.rs` and `src/prop/conn.rs`
becomes `T: ConnK` with `A` / `B` either inferred from `T::A` / `T::B`
or threaded explicitly when the function is generic over them. Three
shapes:

1. **`A` and `B` are not separately needed by the body.** Drop them
   from the function generics: `fn round<T: ConnK>(t: &T, x: T::A) -> T::A`.
2. **The function uses `A` / `B` as type names in the body.** Bind
   them via `where T: ConnK<A = A, B = B>`.
3. **The function constrains `A` / `B` further (e.g. `B: Eq`).** Use
   the equality form: `where T: ConnK<A = A, B = B>, B: Eq`.

Audit list (from grep): `round`, `round1`, `round2`, `truncate`,
`truncate1`, `truncate2`, `interval`, `midpoint`, `median`,
`prop::conn::*`.

## Deferred

- **Migrating downstream crates** (e.g. `agogo`) is out of scope —
  pre-1.0, downstream pins this crate by git rev and updates on its
  own cadence.
- **Removing `Copy` from `Conn<A, B, K>`'s `A: Copy` / `B: Copy`
  bounds.** Independent question, separate sprint.
- **Adding `where Self: Sized` / object-safety** on `ConnL` / `ConnR`.
  The new trait is *not* object-safe (associated types break dyn);
  no current call site uses `dyn ConnL` so this is fine, but a
  `dyn`-friendly variant could be added later if a use case appears.
- **Suppressing typenum expansion in associated-type *value* lines.**
  Rustdoc 1.85 normalizes `type B = …` assignment values through
  every alias level even when the source type spells `U12`. The
  workarounds (newtype wrappers around `FixedI16<Frac>`, custom
  `Display` impls in rustdoc) cost more than the residual noise is
  worth. Revisit if upstream rustdoc grows an alias-preservation
  flag.

## Review

### Outcome

T1–T8 landed as planned; library builds cleanly with associated
types in `ConnL` / `ConnR`. All existing law batteries continue to
pass unchanged (1356 lib tests + 168×9 host-file integration
batteries — the migration is behavior-preserving as predicted).

### Deviation from plan: `ConnK` is no longer usable as a
constraint shorthand

The plan proposed `T: ConnK<A = A, B = B>` for helper-fn bounds.
That shape compiles in trait *definition* position (where ConnK
declares its supertraits) but is **ambiguous in bound position**
because ConnK inherits `A` / `B` through two paths (its `ConnL`
super-trait and its `ConnR` super-trait via the cross-trait
equality). The compiler reports `E0222: ambiguous associated type
'A' in bounds of 'ConnK'` at every helper.

The fix is to bind on the explicit pair:

```rust
pub fn round<T, A, B>(t: &T, x: A) -> B
where
    T: ConnL<A = A, B = B> + ConnR<A = A, B = B>,
```

This is what `ConnK` *means* — its definition (`ConnL + ConnR<A =
ConnL::A, B = ConnL::B>`) is exactly the explicit-pair bound — but
the trait-as-shorthand sugar isn't available. `ConnK` still exists
as a marker trait (and the blanket impl still derives it for any
`T: ConnL + ConnR` over the same `(A, B)`) — it's useful for
documenting the concept and for `T: ConnK` bounds where the user
doesn't need to name `A` / `B`. Helper signatures that *do* name
them use the explicit pair.

### Rustdoc outcome: partial win, larger than I expected to keep

`impl ConnL<…> for F032Q012` headers expanding the typenum
chain — the original complaint — are gone. The rendered impl block
now reads:

```text
impl ConnL for F032Q012
    type A = ExtendedFloat<f32>
    type B = Extended<FixedI16<UInt<UInt<UInt<UInt<UTerm, B1>, B1>, B0>, B0>>>
    fn conn_l(&self) -> Conn<ExtendedFloat<f32>, Extended<FixedI16<U12>>, L>
    fn ceil(&self, a: Self::A) -> Self::B
    fn upper(&self, b: Self::B) -> Self::A
```

The impl header, the `Conn<…>` return type in `conn_l`, and the
method signatures (which use `Self::A` / `Self::B`) are all clean.
Only the `type B = …` assignment line still expands the typenum
chain — rustdoc 1.85 normalizes assignment values through every
alias level even though the source spells `U12`.

Aggregate count of typenum link references under
`target/doc/connections/`:

- pre-sprint baseline: **3888**
- post-sprint:         **1964** (49% reduction)

The remaining 1964 are entirely in the per-impl `type A = … / type
B = …` assignment lines. The dominant noise sites — impl headers
