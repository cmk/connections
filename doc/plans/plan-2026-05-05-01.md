### T1 — Inherent-method doctests on `Conn<_, _, L>` / `Conn<_, _, R>`

Four doctests at definition sites in `src/conn.rs`:

| Item                    | Site             | Source example                                        |
|-------------------------|------------------|-------------------------------------------------------|
| `Conn<_, _, L>::upper`  | `src/conn.rs:155`| π widening f32→f64 via `F064F032.conn_l().upper(...)` |
| `Conn<_, _, L>::ceil`   | `src/conn.rs:148`| π ceiling — the f32 round-up at π                     |
| `Conn<_, _, L>::ceil1`  | `src/conn.rs:184`| Closure probe `2π − x`, ceil narrows up               |
| `Conn<_, _, R>::floor1` | `src/conn.rs:~244` | Same probe, floor narrows down (one f32 ULP below)  |

Adaptation from the original:
- The pre-`5a4a033` examples called free fns: `upper(&F064F032, x)`,
  `ceiling(&F064F032, a)`, etc. In the new API these are inherent
  methods on the underlying `Conn` (one-sided view of the triple),
  reached via `F064F032.conn_l()` / `F064F032.conn_r()`.
- `ceiling` was renamed `ceil`; `ceiling1` → `ceil1`.

### T3 — Free-fn `ConnK` doctests

Nine functions in `src/conn.rs`, ten doctest blocks (`median` carries
two):

| Function   | Site            | Example                                                       |
|------------|-----------------|---------------------------------------------------------------|
| `interval` | `~L398`         | π bracket equipoise: `interval(.., midpoint(.., π)) == Equal` |
| `midpoint` | `~L412`         | π's distance from f32 bracket centre (~3.18e-8 asymmetry)     |
| `truncate` | `~L429`         | π > 0 → f32 floor (one ULP below `std::f32::consts::PI`)      |
| `truncate1`| `~L454`         | Closure probe `2π − x`, positive result truncates to floor    |
| `truncate2`| `~L468`         | `2π32 + 2π32` in f64, narrowed to f32 floor                   |
| `round`    | `~L483`         | Round-to-nearest f32 of true π == `std::f32::consts::PI`      |
| `round1`   | `~L500`         | Newton step on `sin` near π: `\|a\| a − a.tan()`              |
| `round2`   | `~L514`         | Catastrophic cancellation: `(2^24 − 1 + 2) − (2^24 − 1)`      |
| `median`   | `~L529` (×2)    | i32 ordered lattice + N5 lattice with NaN → `Top`             |

Adaptation from the originals:
- The `round` example called `F064F032.inner(...)` to widen the
  result back to f64 for the `pi32_err` assertion. `Conn::inner`
  no longer exists; substitute `F064F032.conn_l().upper(...)`.
- The `median` examples built ad-hoc `Conn::new(ceil, inner, floor)`
  triples. The 3-arg `Conn::new` constructor is gone; build the
  triple with the [`conn_k!`](crate::conn_k) declaration macro and
  reference its generated marker type in the assertion.
