# Plan 42 — `hifi/epoch.rs` §1 TAI baseline + §2 UTC (Sprint 2 of hifitime integration)

## Goal

Land six `hifitime::Epoch` Conns in two scale-families:

- **TAI** (storage scale, hifitime-native, J1900 reference):
  `ETAIHDUR`, `ETAINANO`, `ETAIF064`.
- **UTC** (leap-second-aware, UNIX reference):
  `EUTCHDUR`, `EUTCNANO`, `EUTCF064`.

Reference-epoch choice is **deliberate and asymmetric**: ETAI uses
J1900 TAI (matches `to_tai_duration` / `from_tai_duration`'s
hifitime-native semantic); EUTC uses UNIX EPOCH UTC (matches
`OFDTNANO`'s established "unix nanoseconds" semantic so callers
moving between `time::OffsetDateTime` and `hifitime::Epoch` get
matching numeric values). Documented prominently in the module
docs.

## Tasks

### T1 — Arb strategies

In `src/prop/arb.rs` (cfg-gated `hifi`):

- `arb_hifi_epoch()` — wrap `arb_hifi_duration()` via
  `Epoch::from_tai_duration`. Bias toward J1900 TAI, J2000 TAI,
  GPST reference, UNIX reference, BDT/GST references.
- `arb_extended_hifi_epoch()` — `prop_oneof![1 NegInf, 1 PosInf,
  2 Finite(arb)]` mirror.
- `arb_hifi_epoch_bounded_f64()` / `_f32()` — bounded for float-
  bridge ULP-walk budgets.
- `arb_extended_hifi_epoch_bounded_f64()` / `_f32()` — Extended
  variants.
- `arb_hifi_tai_nanos_in_range()` — `i128` strategy for the
  `ETAINANO.roundtrip_ceil` proptest. Range `[HD::MIN.total_ns(),
  HD::MAX.total_ns()]`.
- `arb_hifi_unix_nanos_in_range()` — same, but offset by the UNIX
  epoch in TAI nanoseconds.

### T2 — `ETAIHDUR`

`Conn<Epoch, HDuration>` — `iso!` (degenerate ConnK).

- `forward(e: Epoch) -> HDuration` = `e.to_tai_duration()`. Rebases
  any non-TAI-scale Epoch to TAI and returns the storage Duration.
- `back(d: HDuration) -> Epoch` = `Epoch::from_tai_duration(d)`
  (const, returns TAI-scale Epoch).

No `Extended` either side: both endpoints are bounded and signed,
and `back ∘ forward` is the "rebase to TAI" idempotent operation
(equal under `Epoch::Eq` which is instant-based). `iso!` exposes
both `ConnL` and `ConnR`; the per-scale-tag idempotence makes both
adjoint laws hold trivially.

### T3 — `ETAINANO`

`Conn<Extended<Epoch>, i128>` — `conn_l!`, OFDTNANO shape.

- `ceil`: NegInf → `i128::MIN`; Finite(e) → `e.to_tai_duration().total_nanoseconds()`;
  PosInf → `HD::MAX.total_nanoseconds() + 1`.
- `inner`: `n < HD::MIN.total_ns() →` NegInf;
  `n > HD::MAX.total_ns() →` PosInf;
  else → `Finite(Epoch::from_tai_duration(HD::from_total_nanoseconds(n)))`.

Same asymmetric saturation pattern as HDURNANO (NegInf → rung's
absolute min so the L-law `ceil(NegInf) ≤ b` holds for all `b`).

### T4 — `ETAIF064`

`Conn<F064, Extended<Epoch>>` — `conn_l!`, F064DURN shape.

- `shift_epoch(n: i32, e: Epoch) -> Epoch` = wrap `shift_hd` from
  `hifi/duration.rs` (re-export visibility as `pub(crate)`).
- `epoch_to_f64(e: Epoch) -> f64` = `e.to_tai_seconds()`.
- `def_walk_helpers!(f64_etai_walks, f64, Epoch, shift_epoch, epoch_to_f64);`
- `f064etai_ceil` follows `f064durn_ceil` pattern with
  `HD::MAX.to_seconds()` / `HD::MIN.to_seconds()` as bounds and
  `Epoch::from_tai_seconds(v)` as the estimate.
- f32 sibling deferred (same shape, just different walk module);
  add only if Sprint 2 review surfaces a need.

Actually, **defer f32 epoch bridges** to a follow-up — the f32
plateau at distant-from-J1900 magnitudes is huge (~10⁵ ns plateau
at ~1000 s magnitude; much wider for 100-year offsets). The f64
bridge is the high-value one; f32 epoch is a niche use case.

### T5 — `EUTCHDUR`

`Conn<Epoch, HDuration>` — `iso!` (degenerate ConnK).

- `forward(e: Epoch) -> HDuration` = unix-anchored UTC Duration:
  `e.to_duration_in_time_scale(TimeScale::UTC) - UNIX_REF_EPOCH.to_utc_duration()`
  (`UNIX_REF_EPOCH` is a public const in `hifitime`).
- `back(d: HDuration) -> Epoch` = `Epoch::from_unix_duration(d)`
  (returns UTC-scale Epoch with the right instant).

Round-trip analysis (verified in plan, proptest enforces):
- `forward ∘ back = id` (back constructs UTC Epoch with duration
  `UNIX_REF + d`, forward extracts UTC duration and subtracts UNIX_REF).
- `back ∘ forward` = "rebase scale to UTC, preserve instant" —
  equal under `Epoch::Eq`.

Why not ConnL despite leap seconds: the UTC↔TAI mapping IS
bijective (one UTC second per TAI second except at the leap-second
insertion, which adds a UTC second; both sides of the leap remain
distinct). `from_unix_duration ∘ to_unix_duration` is identity
modulo scale-tag rebase.

### T6 — `EUTCNANO`

`Conn<Extended<Epoch>, i128>` — `conn_l!`, OFDTNANO shape.

- Reference: UNIX EPOCH UTC (matches `OFDTNANO`).
- `ceil(Finite(e))` = `(e.to_duration_in_time_scale(UTC) - UNIX_REF_EPOCH.to_utc_duration()).total_nanoseconds()`.
  Saturate Finite values whose unix-anchored duration is outside HD's range
  to the rung's saturation marker (same shape as HDURNANO's saturation).
- `inner(n)`: in-range → `Finite(Epoch::from_unix_duration(HD::from_total_nanoseconds(n)))`;
  out-of-range → NegInf/PosInf.
- Range bounds: `[HD::MIN.total_ns(), HD::MAX.total_ns()]` shifted
  to UNIX-anchored space. Computed once at module init via
  helper fns.

### T7 — `EUTCF064`

`Conn<F064, Extended<Epoch>>` — `conn_l!`, F064DURN shape.

- `epoch_to_unix_f64(e: Epoch) -> f64` = `e.to_unix_seconds()`.
- Walk module: reuse `shift_epoch` (TAI-anchored) but compare in
  unix-seconds space — `def_walk_helpers!(f64_eutc_walks, f64,
  Epoch, shift_epoch, epoch_to_unix_f64);`. The walk advances
  the underlying TAI Duration by 1 ns; the comparison wraps
  through `to_unix_seconds` (which itself does a UTC conversion).
- Estimate seed: `Epoch::from_unix_seconds(v)`.

### End-to-end scenario

```rust
use connections::hifi::{ETAINANO, EUTCNANO};
use connections::extended::Extended;
use hifitime::{Epoch, UNIX_REF_EPOCH};

let unix_epoch = UNIX_REF_EPOCH;
// ETAI: J1900 TAI nanoseconds
assert_eq!(
    ETAINANO.ceil(Extended::Finite(unix_epoch)),
    UNIX_REF_EPOCH.to_tai_duration().total_nanoseconds(),
);
// EUTC: unix nanoseconds (matches OFDTNANO semantic)
assert_eq!(EUTCNANO.ceil(Extended::Finite(unix_epoch)), 0);
```

## Deferred

- **f32 Epoch bridges** (`F032ETAI`, `F032EUTC`) — f32 plateau at
  the multi-decade magnitudes typical of Epoch is too wide; deferred
  until a clear use case surfaces.
- **GNSS scales** (GPST, GST, BDT, QZSST) → Sprint 3.
- **Relativistic scales** (TT, ET, TDB) → Sprint 4.
- **Pre-1972 leap-second handling**: hifitime's
  `leap_seconds(true).unwrap_or(0.0)` returns 0 for pre-1972 epochs,
  which is correct (UTC == TAI before 1972). Proptests will hit
  this branch automatically; no special-case test needed.
- **Cross-scale Epoch arb** (UTC-scale, GPST-scale, etc.) — Sprint 2
  arb only produces TAI-scale Epochs. Cross-scale arbs deferred to
  Sprint 3+ when scale-specific Conns need them.

## Review (sprint close)

- **EUTCHDUR reference epoch decision flipped during T5.** Initial
  plan was UNIX-anchored UTC HDuration (forward subtracts UNIX_REF
  from the UTC duration; back adds it). That broke `iso!`'s
  round-trip at the HD boundary: `back(HD::MAX) =
  from_unix_duration(HD::MAX)` saturates the inner `from_utc_duration
  (UNIX_REF.utc + HD::MAX)` because UNIX_REF.utc + HD::MAX overflows
  HD's range. Fix: anchor EUTCHDUR at J1900 UTC (matches ETAIHDUR's
  J1900-TAI choice). UNIX-anchoring lives on EUTCNANO / EUTCF064.
  Documented the asymmetry in the module docs and the EUTCHDUR
  Conn doc.
- **EUTCNANO valid range is asymmetric** about UNIX_REF.utc.
  - Lower bound: `HD::MIN.total_ns()` (NOT `HD::MIN.total_ns() -
    UNIX_REF.utc.total_ns()`) — at the would-be-shifted lower bound,
    `ceil`'s `epoch.to_utc_duration() − UNIX_REF.utc` subtraction
    underflows HD because the stored UTC duration is already at
    HD::MIN.
  - Upper bound: `HD::MAX.total_ns() − UNIX_REF.utc.total_ns()` —
    above this, the inner `n + UNIX_REF.utc` addition overflows HD
    when constructing the stored UTC duration.
  - Found via `roundtrip_ceil` proptest at the boundary value;
    documented the derivation inline.
- **EUTCNANO inner reorder.** Initial code did `Epoch::from_unix_duration
  (HD::from_total_nanoseconds(b))`, which saturates the intermediate
  HD when `b` is at `unix_min_nanos`. Reordered to compute
  `b + UNIX_REF.utc.total_ns()` first (i128 add, no saturation) and
  then construct the HD from the sum — guarantees no intermediate
  saturation when `b` is in the valid range.
- **`def_walk_helpers!` reused on `Epoch`.** The walk advances
  `e.to_tai_duration()` by 1 ns and reconstructs via
  `Epoch::from_tai_duration`. EUTCF064 uses the same shift function
  (TAI-anchored walk) but compares through `to_unix_seconds` —
  works because hifitime's UTC↔TAI conversion is monotonic.
- **No `#[ignore]`s introduced.** All 53 new hifi::epoch unit tests
  green plus 6 doctests.
- **f32 Epoch bridges still deferred.** F032ETAI / F032EUTC
  unimplemented; revisit if a downstream needs them.
- **MR iid prediction**: 64. Sprint 1 → 63 (also unpushed at the
  time of this writing).
