# Plan 46 — Sprint 5: hifi/calendar.rs (MonthName, Weekday)

## Goal

Add `src/hifi/calendar.rs` shipping **3 ConnL Conns** wrapping
hifitime's `MonthName` and `Weekday` enums:

- `MONTU008` : `Conn<Extended<MonthName>, Extended<u8>>` — 1=Jan…12=Dec
- `MONTNZ08` : `Conn<Extended<MonthName>, Extended<NonZeroU8>>` —
  natural NonZero rep
- `WKDYU008` : `Conn<Extended<Weekday>, Extended<u8>>` — 0=Mon…6=Sun (ISO)

Independent of Sprints 3/4 — only depends on Sprint 2's already-shipped
infrastructure. Runs in parallel with Sprint 4 in a separate worktree.

## Why ConnL not iso!

The master plan tagged all three as `Iso`. The hifitime source
disproves this:

- **`MonthName::From<u8>`** (`hifitime/src/month.rs:83-101`) defaults
  to January for any out-of-range u8 (including 0 and ≥13). So
  `from(0) = January = from(1)` — collision, not bijective.
- **`Weekday::From<u8>`** (`hifitime/src/weekday.rs:54-67`) uses
  `rem_euclid(7)`. So `from(7) = Monday = from(0)` — wrap, not
  bijective.

`iso!` requires `back ∘ forward = id` AND `forward ∘ back = id`. The
collisions break the second law. The right shape is `conn_l!` with
explicit saturation: out-of-domain integers map to `Extended::PosInf`
(or `NegInf` for u8=0 below the canonical Month range).

This makes the user-facing semantic **honest** about the partial
mapping: an unknown integer doesn't pretend to be January.

```rust
fn montu008_ceil(m: Extended<MonthName>) -> Extended<u8> {
    match m {
        Extended::NegInf => Extended::NegInf,
        Extended::Finite(month) => Extended::Finite(month as u8 + 1),
        Extended::PosInf => Extended::PosInf,
    }
}

fn montu008_inner(u: Extended<u8>) -> Extended<MonthName> {
    match u {
        Extended::NegInf => Extended::NegInf,
        Extended::Finite(0) => Extended::NegInf,
        Extended::Finite(n @ 1..=12) => Extended::Finite(MonthName::from(n)),
        Extended::Finite(_) => Extended::PosInf,
        Extended::PosInf => Extended::PosInf,
    }
}

crate::conn_l! {
    pub MONTU008 : Extended<MonthName> => Extended<u8> {
        ceil:  montu008_ceil,
        inner: montu008_inner,
    }
}
```

## Deferred

- **Iso variants for these enums.** Hifitime's `From<u8>` impls
  silently wrap/default, which breaks both adjoint laws of `iso!`.
  An iso requires a fallible-conversion variant of `From<u8>` (e.g.
  `TryFrom<u8>` returning `Option`), which hifitime doesn't ship.
  Compose `MONTU008` / `WKDYU008` with `MONTNZ08` instead if a
  particular caller wants integer-NonZero-month round trips.
- **`MonthName ↔ time::Month` cross-crate iso** (could be Sprint 6
  territory). Both crates' month enums have the same 12 variants;
  an iso between them is straightforward but out of scope for this
  sprint.

## Review

**Scope conformance:** All 8 tasks T1-T8 implemented as planned.

**Type signatures revised during implementation.** The plan and
master plan called for `Conn<Extended<X>, Extended<u8>>` shapes.
After observing kernel_l proptest failures (initial implementation
mapped out-of-range Extended<u8> inputs to Extended::PosInf, but
ceil(PosInf) returned Extended::PosInf which is > any Finite u8,
breaking `ceil(inner(b)) ≤ b`), revised to OFDTNANO-shape per the
master plan's Lesson 2: `Conn<Extended<X>, u8>` with bare integer
target and saturation arms (`ceil(NegInf) = 0` or smallest-valid;
`ceil(PosInf) = max_valid + 1`). All 30 lib tests pass under this
shape, including kernel_l.

**Build gates verified:** `cargo build / test / clippy / doc`
(strict warnings) all pass; `cargo build --no-default-features` is
clean (feature truly optional). 1376 lib tests, 25 hifi doctests.

**No `#[ignore]`d properties.**

**Recommendations:**
- The `arb_extended_u8` and `arb_extended_nonzero_u8` strategies I
  initially added to `src/prop/arb.rs` were removed when the type
  signatures shifted to bare-integer rung (no longer needed). The
  test file now uses local boundary-biased helpers
  (`arb_u8_calendar`, `arb_nz_u8_calendar`) since these are
  Sprint-5-specific and don't merit being public.
- `MONTNZ08`'s NegInf collapse onto `NonZeroU8(1)` is structural
  (NonZeroU8 has no sub-1 value to use as a sentinel). The Galois
  law tolerates the collapse via monotone-with-equality. Documented
