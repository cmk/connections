# Plan 04 — Integer + unsigned-integer conns (Word.hs + Int.hs port)

## Goal

Populate `src/conn/uint.rs` (renamed from `word.rs`) and `src/conn/int.rs`
with the within-sign and Word↔Int cross-sign integer Galois connections —
**28 named `Conn` constants total**, mirroring the within-ladder + cross-
ladder portions of Haskell's `Data.Connection.Word` and
`Data.Connection.Int`. Use Rust primitives (`u8`/`u16`/.../`i64`) directly
as conn type parameters; reserve newtype wrappers for the unit-bearing
fixed-point and sample tiers.

## Context

Plans 01-03 stabilised the trait stack (`Ple`, `Join`/`Meet`,
`Heyting`/`Coheyting`/`Symmetric`/`Boolean`), the Galois `Conn<A, B>` core,
the `compose_conn!` macro, the `Extended<T>` range-extension wrapper, and
the proptest pattern in `src/conn/fixed.rs` (~21 conns, ~150 properties).
What's missing: the integer side of the lattice atlas. `src/conn/word.rs`
and `src/conn/int.rs` are empty stubs with a single-line doc-comment.

This sprint fills both. After it lands every primitive integer width is a
node in a typed Galois ladder, with same-sign upcasts (single-sided
saturating) and cross-sign conversions (Int→Word saturate-at-zero;
Word→Int via `Extended<u_N>` source). Closes the third source file from
the Haskell repo (after `Data.Connection.Float` partially via plan 03 and
`Data.Connection.Fixed` via plan 02).

Out-of-scope, deferred for follow-up sprints: `*→Natural`/`*→Integer`
analogs (Rust has no built-in arbitrary-precision integer; needs a design
choice — `num-bigint` dep vs `i128`/`u128` stand-in), `usize`/`isize`
(platform-sized), `i128`/`u128`, `Bool↔Bounded` (`bndbin`).

### T2 — `uint.rs`: Word→Word ladder (6 single-sided conns)

Haskell reference: `Data.Connection.Word` — `w08w16`, `w08w32`, `w16w32`,

| Const     | Type             | ceil               | inner (saturating)          |
|-----------|------------------|--------------------|------------------------------|
| `U08U16`  | `Conn<u8,  u16>` | `x as u16`         | `x.min(u8::MAX  as u16) as u8`  |
| `U08U32`  | `Conn<u8,  u32>` | `x as u32`         | `x.min(u8::MAX  as u32) as u8`  |
| `U08U64`  | `Conn<u8,  u64>` | `x as u64`         | `x.min(u8::MAX  as u64) as u8`  |
| `U16U32`  | `Conn<u16, u32>` | `x as u32`         | `x.min(u16::MAX as u32) as u16` |
| `U16U64`  | `Conn<u16, u64>` | `x as u64`         | `x.min(u16::MAX as u64) as u16` |
| `U32U64`  | `Conn<u32, u64>` | `x as u64`         | `x.min(u32::MAX as u64) as u32` |

All built with `Conn::new_left(ceil, inner)` (see `src/conn.rs:124`).

**Macro to generate:**

```rust
macro_rules! uint_uint {
    ($NAME:ident, $A:ty, $B:ty) => {
        #[doc = concat!("`", stringify!($A), " → ", stringify!($B), "` saturating cast.")]
        pub const $NAME: $crate::conn::Conn<$A, $B> = {
            fn ceil(x: $A) -> $B { x as $B }
            fn inner(x: $B) -> $A {
                let cap = <$A>::MAX as $B;
                let clipped = if x > cap { cap } else { x };
                clipped as $A
            }
            $crate::conn::Conn::new_left(ceil, inner)
        };
    };
}
```

Six expansions at the top of `uint.rs`. The `if` pattern instead of `min()`
keeps the body `const fn`–compatible (today `Conn::new_left` is `const`,
`u64::min` is not stably const, so we hand-roll the comparison).

### T3 — `uint.rs`: Int→Word ladder (10 single-sided saturating conns)

Haskell reference: `Data.Connection.Word` — `i08w08`, `i08w16`, `i16w16`,
`i08w32`, `i16w32`, `i32w32`, `i08w64`, `i16w64`, `i32w64`, `i64w64`. All
`CastL`. **Source signed bit-width ≤ target unsigned bit-width** — Haskell
omits the cases where source is wider (the `as` narrowing would wrap
under the saturation arithmetic Haskell uses); we mirror.

Naming: `I08U08`, `I08U16`, `I16U16`, `I08U32`, `I16U32`, `I32U32`,
`I08U64`, `I16U64`, `I32U64`, `I64U64` (10 conns).

| Direction | ceil(a)               | inner(b) (saturating to source max) |
|-----------|-----------------------|--------------------------------------|
| `iN → uM` | `(a.max(0)) as uM`    | `b.min(iN::MAX as uM) as iN`         |

The lawfulness check: ceil maps the source's negative half to 0 and the
non-negative half through `as uM` (lossless because `bits(iN) - 1 ≤
bits(uM)` is implied by `bits(iN) ≤ bits(uM)` — i.e. positive `iN` values
fit in `uM`). Inner is the right adjoint: it returns the largest `iN`
whose ceil ≤ b. For b ≥ iN::MAX, that's iN::MAX. For 0 ≤ b < iN::MAX,
that's b cast back. For all of these, `b ≥ 0` is guaranteed by `uM`'s type.

**Macro:**

```rust
macro_rules! int_uint {
    ($NAME:ident, $A:ty, $B:ty) => {
        #[doc = concat!("`", stringify!($A), " → ", stringify!($B), "`: saturate at 0; inner saturates at source max.")]
        pub const $NAME: $crate::conn::Conn<$A, $B> = {
            fn ceil(x: $A) -> $B {
                if x < 0 { 0 } else { x as $B }
            }
            fn inner(x: $B) -> $A {
                let cap = <$A>::MAX as $B;
                let clipped = if x > cap { cap } else { x };
                clipped as $A
            }
            $crate::conn::Conn::new_left(ceil, inner)
        };
    };
}
```

Ten expansions.

```rust
macro_rules! int_int {
    ($NAME:ident, $A:ty, $B:ty) => {
        #[doc = concat!("`", stringify!($A), " → ", stringify!($B), "` saturating cast.")]
        pub const $NAME: $crate::conn::Conn<$A, $B> = {
            fn ceil(x: $A) -> $B { x as $B }
            fn inner(x: $B) -> $A {
                let lo = <$A>::MIN as $B;
                let hi = <$A>::MAX as $B;

                clipped as $A
            }

        };
    };
}
```

### T5 — `int.rs`: Uint→Int ladder (6 full-triple conns over `Extended<u_N>`)

Haskell reference: `Data.Connection.Int` — `w08i16`, `w08i32`, `w16i32`,
`w08i64`, `w16i64`, `w32i64`. **Source unsigned strictly narrower than
target signed**, so the source range fits in the target's positive half.
Type signature: `Conn<Extended<u_N>, i_M>`. Full adjoint triple (ceil ≠
floor at the infinity inputs).

Following Haskell's `conn :: Cast (Extended a) b`:

```rust
pub const U08I16: Conn<Extended<u8>, i16> = {
    fn ceil(x: Extended<u8>) -> i16 {
        match x {
            Extended::NegInf    => i16::MIN,
            Extended::Finite(a) => a as i16,
            Extended::PosInf    => u8::MAX as i16 + 1,  // "above"
        }
    }
    fn inner(x: i16) -> Extended<u8> {
        let above = u8::MAX as i16 + 1;
        if x < 0      { Extended::NegInf }
        else if x >= above { Extended::PosInf }
        else          { Extended::Finite(x as u8) }
    }
    fn floor(x: Extended<u8>) -> i16 {
        match x {
            Extended::NegInf    => -1,                  // "below"
            Extended::Finite(a) => a as i16,
            Extended::PosInf    => i16::MAX,
        }
    }
    Conn::new(ceil, inner, floor)
};
```

The six conns are: `U08I16`, `U08I32`, `U08I64`, `U16I32`, `U16I64`,
`U32I64`. **Macro:**

```rust
macro_rules! uint_int {
    ($NAME:ident, $A:ty, $B:ty) => {
        #[doc = concat!("`Extended<", stringify!($A), "> → ", stringify!($B),
                        "` adjoint triple. Source range fits in target's positive half.")]
        pub const $NAME: $crate::conn::Conn<$crate::extended::Extended<$A>, $B> = {
            const ABOVE: $B = <$A>::MAX as $B + 1;
            const BELOW: $B = -1;
            fn ceil(x: $crate::extended::Extended<$A>) -> $B {
                match x {
                    $crate::extended::Extended::NegInf    => <$B>::MIN,
                    $crate::extended::Extended::Finite(a) => a as $B,
                    $crate::extended::Extended::PosInf    => ABOVE,
                }
            }
            fn inner(x: $B) -> $crate::extended::Extended<$A> {
                if x < 0           { $crate::extended::Extended::NegInf }
                else if x >= ABOVE { $crate::extended::Extended::PosInf }
                else               { $crate::extended::Extended::Finite(x as $A) }
            }
            fn floor(x: $crate::extended::Extended<$A>) -> $B {
                match x {
                    $crate::extended::Extended::NegInf    => BELOW,
                    $crate::extended::Extended::Finite(a) => a as $B,
                    $crate::extended::Extended::PosInf    => <$B>::MAX,
                }
            }
            $crate::conn::Conn::new(ceil, inner, floor)
        };
    };
}
```

Six expansions. The `BELOW`/`ABOVE` constants encode the boundary
semantics — a target value of `-1` is "just below the source's range",
a value of `u_N::MAX + 1` is "just above". Inner detects these and
saturates to ±∞.

## Review

### Design deviations

- **`I??I??` family unified with `U??I??`** under one `ext_int!` macro
  producing `Conn<Extended<$A>, $B>`. The plan's original single-sided
  `Conn<i_N, i_M>` with a saturating-clamp inner cannot satisfy the
  Galois law for target values below `i_N::MIN`: there is no source
  value that maps below `i_N::MIN` under `ceil`, so no `a` can witness
  `a ≤ inner(b)` for `b` strictly below the source range. Wrapping the
  source in `Extended` gives `NegInf` as that witness. Mirrored
  Haskell: `i08i16 :: Cast k (Extended Int8) Int16`. Both Int→Int and
  Word→Int now share an identical macro body; only the source's `MIN`
  and `MAX` constants differ.

- **`floor_le_ceil` proptest dropped** for the full-triple Extended-
  source family. Inner is many-to-one (every `b ≤ BELOW` maps to
  `NegInf`; every `b ≥ ABOVE` maps to `PosInf`), so at the infinity
  inputs `ceil(NegInf) = target::MIN < BELOW = floor(NegInf)`. The
  property simply doesn't hold here. Replaced with `kernel_upper`
  (`ceil(inner(b)) ≤ b`) and `kernel_lower` (`b ≤ floor(inner(b))`) —
  the closure laws of the left and right adjoints respectively.
  Together with `galois_upper`, `galois_lower`, and the three
  monotonicity laws these fully characterise the adjoint triple.
  **This is permanent, not deferred** — the property is structurally
  inapplicable whenever `inner` is non-injective (collapses target
  values to a common Extended sentinel), which is intrinsic to any
  range-extending source. There is no re-enablement plan and future
  conns following this shape (e.g. `*128I64` once added) should use

### Recommendations

- The `ext_int!` macro is fully general; future sprints can add
  `usize`/`isize`/`i128`/`u128` source-target pairs by appending macro
  calls. The boundary arithmetic `(<$A>::MIN as $B) - 1` /
  `(<$A>::MAX as $B) + 1` must fit in `$B`, guaranteed by
  `bits($A) < bits($B)` for signed sources and
  `bits($A) ≤ bits($B) − 1` for unsigned-into-signed.
- `compose_conn!` interoperates with the new conns at `const` scope
  (the macro already accepts any `const Conn<A, B>` boundary types).
  A formal smoke test for an integer-conn composition could be added
  in a follow-up if desired.
