# Plan 2026-05-09-01 вҖ” FloatвҶ’Extended<intN> and FloatвҶ’FixedI/U<Frac> Conns

## Context

The Haskell library `connections-haskell/src/Data/Connection/Float.hs`
ships 22 `Cast k Float (Extended <int>)` connections that the Rust
port has not yet implemented. The user has asked for the Rust
equivalents, plus an extension into Q-format target rungs (`F<src>Q<frac>`
for `FixedI{8,16,32,64,128}<U<frac>>` and `FixedUвҖҰ<U<frac>>`).

Two existing Rust families bracket this work:

- `src/float/f064.rs`, `src/float/f032.rs` (and `src/float/f016.rs`
  behind `feature = "f16"`) ship the float-float narrowing Conns
  (`F064F032`, `F064F016`, `F032F016`).
- `src/fixed/i{008,016,032,064,128}.rs` and the matching `u*.rs` files
  ship `Extended<intN>вҶ’intM` widening (`ext_int!`) and intra-Q-format
  rung Conns (`fix_fix_iN!`/`fix_fix_uN!`).

Nothing currently bridges a Float source to either an integer rung or
a Q-format rung. This plan closes both gaps.

The intended outcome: every `f32w/f64w/f32i/f64i` from the Haskell
file has a Rust analogue in `src/float/`, and `FloatвҶ’FixedI/U<Frac>`
Conns land at the same rung cadence the existing `fix_fix_*!` ladders
use, hosted in `src/fixed/`.

## Goal

One sentence: ship 24 `FloatвҶ’Extended<intN>` Conns plus ~186
`FloatвҶ’FixedI/U<Frac>` Conns, all law-batteried, with f16 variants
behind the `f16` cargo feature вҖ” implementing roughly **half**
directly and deriving the rest via `compose!` chains through the
existing `F064F032` / `F032F016` / `F064F016` widening adjoints.

## Compose-where-trivial strategy

Per the user's choice, ~105 of the 210 Conns are emitted as
compositions of primary-source Conns through the existing
float-float narrowing Conns' upper adjoints. This halves direct
implementation work and test surface while keeping the public name
set identical to the all-direct version. Trade-off: each composed
Conn adds one fn-pointer hop at runtime.

Composability matrix (per family Г— source):

| Family            | f32 source | f64 source                                 | f16 source                                                  |
|-------------------|------------|--------------------------------------------|-------------------------------------------------------------|
| Phase 1 Ext<intN> | direct (8) | 4 direct (вүҘ32-bit targets) + 4 composed (вүӨ16-bit) | all 8 composed via swap(F032F016) вҲҳ F032<X>           |
| Phase 2 Q-format  | direct (62)| 31 direct (i32/u32/i64/u64/i128/u128 hosts) + 31 composed (i8/u8/i16/u16 hosts) | all 62 composed via swap(F032F016) вҲҳ F032Q<frac>       |

**Direct count**: 8 + 4 + 62 + 31 = **105**.
**Composed count**: 4 + 8 + 31 + 62 = **105**.

## Naming options (collision surface)

The 8-char Conn-name rule (CLAUDE.md В§Conn-name format) plus the
"`Q<frac>` prefix encodes frac-bits, host bit-width comes from module
path" rule force cross-module name collisions when the same `<frac>`
exists under multiple hosts. Three options to handle them:

| Option | Approach | Tradeoff |
|--------|----------|----------|
| **A (recommended)** | Embrace the collision per CLAUDE.md precedent. `F032Q008` lives in `src/fixed/i008.rs` (FixedI8<U8>), `src/fixed/u008.rs` (FixedU8<U8>), `src/fixed/i016.rs` (FixedI16<U8>), `src/fixed/u016.rs`, `src/fixed/i032.rs`, `src/fixed/u032.rs`, `src/fixed/i064.rs`, `src/fixed/u064.rs`. Resolved by qualified import: `use connections::fixed::i032::F032Q008 as F32_to_Q16_16;`. | Matches existing pattern (`fixed::i8::Q008Q000` and `fixed::i64::Q008Q000` already coexist). No name-budget pressure. Downstream users must qualify if they import multiple hosts at once. |
| **B** | Encode host bit-width in the name. E.g. rename `F032Q008` under FixedI16 to `F032QI16`. | Breaks the 8-char-with-3-digit-Q-side rule. Requires a brand-new shape (`F032QI16` is 8 chars but inconsistent with every other Conn). Rejected. |
| **C** | Ship only the *self-frac* rungs (`Q008` under FixedI8, `Q016` under FixedI16, вҖҰ) вҖ” i.e. `<frac> = host_bits`. No cross-host frac collisions. | Drops ~80% of the Q-format target surface. Loses every `Q0.<n<width>>` and intermediate-frac rung. |

**Decision: Option A** (confirmed by user). Cross-module collisions
are accepted; downstream code resolves with qualified imports
(`use connections::fixed::i032::F032Q008 as F32_to_Q16_16;`).
Matches the existing `fixed::i8::Q008Q000` вҶ” `fixed::i64::Q008Q000`
precedent. Plan body assumes Option A throughout.

Mantissa thresholds:
- f16 mantissa: 11 bits (incl. implicit) вҶ’ full triple iff host вүӨ 11 bits вҶ’ only i8/u8.
- f32 mantissa: 24 bits вҶ’ full triple iff host вүӨ 24 bits вҶ’ i8/u8, i16/u16. (i32/u32 spans 32 bits, which exceeds 24.)
- f64 mantissa: 53 bits вҶ’ full triple iff host вүӨ 53 bits вҶ’ i8/u8, i16/u16, i32/u32. (i64/u64 spans 64 bits.)

For Q-format, the relevant precision is the host bit-width, not the
frac value. So `F064Q008` in `i032` is full triple (i32 fits in f64
mantissa) regardless of whether frac is 0 or 24.

### T0 вҖ” Compose-primitive audit (read-only, no edits)

**Why first:** T2/T3/T7 macro design and T5/T6/T9/T10 compose tasks
all hinge on whether the existing `compose!` / `compose_l!` /
`compose_k!` macros (in `src/conn.rs`) can chain a Conn through the
*upper adjoint* of an existing narrowing Conn вҖ” i.e. whether

a. A `swap!` / `dual!` macro that flips a `Conn<A, B, K>` into
   `Conn<B, A, dual_kind(K)>` using the existing upper-adjoint
   side as the new lower adjoint.
b. A `compose_via_upper!` macro that dispatches through the parent's
   upper rather than lower adjoint.
c. Direct implementation of the f16 source Conns (skipping
   composition).

**Action:** Read `src/conn.rs` for the compose macro family. Quote
the macro signatures and behaviour to the plan's Review section.
Decide between (a)/(b)/(c). If (a) or (b), add the new macro as the
last sub-step of T2 (since it's small). If (c), drop the f16-source
composition and revert T6/T10 to direct implementation (raises
direct count from 105 to ~177, drops total Conn count by 0).

**Output:** A concrete answer in the plan's Review section

### T2 вҖ” `float_ext_int!` macro (full-triple Conns)

Emits a `pub const $NAME: Conn<ExtendedFloat<$float>, Extended<$int>>`
that's a full Galois adjoint triple (impls both `ConnL` and `ConnR`)
via `crate::conn_k!`.

Signature:
```rust
float_ext_int!($NAME, $float, $int);
// e.g. float_ext_int!(F032U008, f32, u8);
```

Body sketch (mirrors Haskell `fxxext` at Float.hs:393-405, plus
`ExtendedFloat::Bot/Top` pass-through):

```rust
fn ceil(x: ExtendedFloat<$float>) -> Extended<$int> {
    use ExtendedFloat::*;
    match x {
        Bot => Extended::NegInf,
        Top => Extended::PosInf,
        Extend(v) => {
            if v.is_nan() { return Extended::PosInf; }   // NaN: ceil is +вҲһ
            if v == <$float>::NEG_INFINITY { return Extended::NegInf; }
            if v == <$float>::INFINITY     { return Extended::PosInf; }
            let lo = <$int>::MIN as $float;
            let hi = <$int>::MAX as $float;
            if v > hi { Extended::PosInf }
            else if v < lo { Extended::Finite(<$int>::MIN) }
            else { Extended::Finite(v.ceil() as $int) }   // ceil of v as int
        }
    }
}

fn inner(b: Extended<$int>) -> ExtendedFloat<$float> {
    use Extended::*;
    match b {
        NegInf => ExtendedFloat::Extend(<$float>::NEG_INFINITY),
        PosInf => ExtendedFloat::Extend(<$float>::INFINITY),
        Finite(i) => ExtendedFloat::Extend(i as $float),
    }
}

fn floor(x: ExtendedFloat<$float>) -> Extended<$int> {
    // mirror of ceil with: NaN вҶ’ NegInf, swap hi/lo branches, use v.floor().
}
```

Note `inner`'s `NegInf вҶ’ Extend(-вҲһ)` choice matches Haskell's
`extended (-1/0) (1/0) fromIntegral` (Float.hs:398). It does **not**
emit `ExtendedFloat::Bot` вҖ” Bot/Top are pass-through, not synthesised
from the rung side. (The Haskell version doesn't have an `ExtendedFloat`
sentinel layer, only the float's native ВұвҲһ.)

Place the macro in `src/float.rs` next to `def_walk_helpers!`. Mirror
the `#[cfg_attr(feature = "macros", macro_export)]` pattern from
`ext_int!` in `src/fixed.rs:233-269` so downstream crates can use it.

### T3 вҖ” `float_ext_int_l!` macro (L-only Conns)

Same shape as T2 but `Conn::new_l(ceil, inner)` only, no triple
emission. Used when target precision exceeds source mantissa
(f32вҶ’u32/u64/i32/i64; f64вҶ’u64/i64; f16вҶ’u16/u32/u64/i16/i32/i64).

Body identical to T2's `ceil`/`inner` but does NOT include `floor`.
The L-Galois law `ceil(a) вүӨ b вҹә a вүӨ inner(b)` still holds because the
casts saturate consistently; only `order_reflecting` (the triple
witness) fails when two distinct large `int` values both round to the
same float.

### T7 вҖ” `float_fixed!` macro

Emits `pub const $NAME: Conn<ExtendedFloat<$float>,
Extended<$Fixed<$frac>>>` (or, for the L-only case, a `ConnL`). The
target type comes from `fixed`-crate `FixedI8<U8>`, etc.

The Q-format rung is wrapped in `Extended<_>` (matching the Haskell
target shape) so that out-of-range floats saturate to NegInf/PosInf.
Adjoint logic is structurally identical to T2/T3 but the `as` casts
now go through `FixedI<host>::from_num` (saturating Q-format
constructor вҖ” see `fixed`-crate docs) and `FixedI<host>::to_num` for
the reverse:

```rust
// pseudo-body, full-triple form
fn ceil(x: ExtendedFloat<$float>) -> Extended<$Fixed<$frac>> {
    use ExtendedFloat::*;
    match x {
        Bot => Extended::NegInf,
        Top => Extended::PosInf,
        Extend(v) => {
            if v.is_nan() { return Extended::PosInf; }
            if v == <$float>::INFINITY     { return Extended::PosInf; }
            if v == <$float>::NEG_INFINITY { return Extended::NegInf; }
            let max = $Fixed::<$frac>::MAX.to_num::<$float>();
            let min = $Fixed::<$frac>::MIN.to_num::<$float>();
            if v > max { Extended::PosInf }
            else if v < min { Extended::Finite($Fixed::<$frac>::MIN) }
            else {
                // ceil of v rounded to the nearest higher Q-format rung.
                // fixed crate exposes `from_num` (round-to-nearest); we need
                // round-up. Use scaled-int trick: q = ceil(v * 2^frac) / 2^frac.
                let scale = (1u128 << <$frac as Unsigned>::U32) as $float;
                let raw = (v * scale).ceil();
                let bits = raw as <$Fixed::<$frac> as Fixed>::Bits;
                Extended::Finite($Fixed::<$frac>::from_bits(bits))
            }
        }
    }
}
```

Implementation reality check: the `fixed` crate (1.x) has
`from_num_saturating` / `checked_from_num` and trait
`FromFixed`/`ToFixed`. Use `wrapping_to_fixed` /
`saturating_from_num` rather than the bare `as` casts where the API
gives a saturating variant; fall back to the scale-and-cast trick for
ceil/floor when no direct fixed-crate ceil exists. T7 first draft
should land the macro with a single host (FixedI8) to confirm the
math, then generalise.

L-only variant `float_fixed_l!` mirrors this with `Conn::new_l`.

### T0 outcome

The audit picked **option (c) for f16 sources** and **option (a)
unmodified for f64-narrow targets**. Specifically:

- For f64 вҶ’ вүӨ16-bit-target Conns, `compose_k!(F064F032, F032<X>)`
  works directly because both child Conns narrow in the same
  direction (high precision вҶ’ low). No new primitive needed.
- For f16 вҶ’ any-int-target Conns, composing through `F032F016`'s
  upper adjoint would have required a new `swap!` or `dual!`
  macro that flips a triple's adjoint sides. Direct
  implementation is ~20 lines per Conn Г— 8 Conns; the swap
  primitive is meaningful design surface and was judged
  out-of-scope. The original 105/105 direct/composed split moved
  to ~177 direct in the abstract; final shipped Phase 1 surface
  (24 Conns) matches the plan exactly.

### Design deviations

1. **`inner(NegInf)` maps to `Bot` (not `Extend(-вҲһ)`).** The N5
   ordering puts `Bot < Extend(-вҲһ)` strictly, so emitting
   `Extend(-вҲһ)` here breaks the Galois law at the
   `(Bot, NegInf)` and `(Top, PosInf)` boundaries. Symmetrically
   `inner(PosInf) = Top`. Documented inline at
   `__float_ext_int_inner_body!`.

2. **Saturation on `Extend(ВұвҲһ)` flows through `v > hi` / `v < lo`
   rather than explicit `==` checks**, except for an explicit
   `v == INFINITY вҶ’ PosInf` short-circuit added because for f16
   sources `<int>::MAX as f16` saturates to `+вҲһ`, so `v > hi`
   misclassifies `+вҲһ` as "in range". (Negative-infinity classification
   stays correct via `v < lo` because `<int>::MIN as f16` saturates
   to `-вҲһ`, and `-вҲһ < -вҲһ` is `false`, which is the right result вҖ”
   the unified branch then computes `Finite(MIN)`.)

3. **`inner` always uses `RoundDownFromI128::from_i128_rd`**, even
   for full-triple Conns where the cast is already exact. The
   round-down path is a no-op when `b as $float` already equals
   `b`, so the cost is only an `is_finite` check and an `i128`
   compare per inner call. Worth the simplicity of one inner
   path across both adjoint shapes.
