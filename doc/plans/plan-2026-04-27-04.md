# Plan 16 — Cast Sprint B: two-sided helpers

## Goal

Port the remaining `Data.Connection.Cast` two-sided helpers
(`interval`, `midpoint`, `round` / `round1` / `round2`, `truncate` /
`truncate1` / `truncate2`, `median`) into `src/conn/cast.rs`. Pure
addition; existing public surface unchanged.

## Context

Plan 09 (`doc/plans/plan-2026-04-26-08.md`) outlined four Cast
sprints. Sprint A (L+R accessors and lifters) shipped in MR !22.
This sprint covers Sprint B per Plan 09's roadmap, with two
adjustments:

1. The `Pcompare` trait is dropped — Plan 11 (`drop-Ple`) collapsed
   the lawful framework onto `Eq + PartialOrd`, so `pcompare` is
   `PartialOrd::partial_cmp` directly. No new traits.
2. Arithmetic bounds (`Add` / `Sub` / `Div` / `From<u8>`) live
   inline at each helper rather than behind a `Numeric` trait.

The public function shapes follow the upstream Haskell `Cast.hs`
verbatim; the only deviation is `midpoint`'s body, which uses
`lo + (hi - lo) / 2` instead of Haskell's `lo / 2 + hi / 2`. The
offset form is exact on both integer and float source types
(Haskell's formula loses 1 bit on odd-summed integers because
Haskell's `Fractional` constraint excludes them by construction).
Rust's bounds permit integer sources, so the offset form is the
right fit.

## Tasks

### T1 — Helpers in `src/conn/cast.rs`

Append below the L/R block (after `minimize`, before `mod tests`).
Module-level rustdoc gains a "Two-sided helpers" subsection;
Sprint A's outdated `Pcompare` reference is removed.

| Fn | Signature | Body |
|----|-----------|------|
| `interval` | `<A, B>(c, x) -> Option<Ordering>` where `A: Copy + PartialOrd + Sub<Output=A>` | `(x - lower1(c, id, x)).partial_cmp(&(upper1(c, id, x) - x))` |
| `midpoint` | `<A, B>(c, x) -> A` where `A: Copy + Add<Output=A> + Sub<Output=A> + Div<Output=A> + From<u8>` | `let lo = lower1(...); let hi = upper1(...); lo + (hi - lo) / 2` |
| `truncate` | `<A, B>(c, x) -> B` where `A: Copy + PartialOrd + From<u8>` | `if x >= 0 { c.floor(x) } else { c.ceil(x) }` |
| `truncate1` | `<A, B, F>(c, f, x) -> B` where `F: FnOnce(A) -> A` | `truncate(c, f(c.inner(x)))` |
| `truncate2` | binary lifter | `truncate(c, f(c.inner(x), c.inner(y)))` |
| `round` | `<A, B>(c, x) -> B` where `A: Copy + PartialOrd + Sub<Output=A> + From<u8>` | `match interval(c, x) { Some(Greater) => c.ceil(x), Some(Less) => c.floor(x), _ => truncate(c, x) }` |
| `round1` | unary lifter | `round(c, f(c.inner(x)))` |
| `round2` | binary lifter | `round(c, f(c.inner(x), c.inner(y)))` |
| `median` | `<A>(c: &Conn<(A,A), A>, x, y, z) -> A` where `A: Copy` | `(x ⊔ y) ⊓ (y ⊔ z) ⊓ (z ⊔ x)` via `maximize` / `minimize` |

`median`'s `Conn<(A, A), A>` shape comes directly from Haskell
(`forall k. Cast k (a, a) a`), and matches the existing
`ORDERED_PAIR` test fixture. The Birkhoff body is the standard
median-via-lattice formulation.

`round1` / `truncate1` and their binary cousins flip direction from
Sprint A's `upper1` / `lower1`: input is `B`, output is `B`, with
the closure applied in `A`-space via `c.inner(x)`. This matches
Haskell `round1 c f x = round c $ f (inner c x)`.

Each public fn carries a 4–6 line doctest using
`Conn::<i32, i32>::identity()` (or, for `median`, the 3-line
`max/min` Conn definition). Doctests run under `cargo test --doc`.

### End-to-end scenario

```rust
use connections::{ceiling, floor, interval, midpoint, round, truncate, median};
use connections::conn::Conn;

let c: Conn<i32, i32> = Conn::identity();
assert_eq!(interval(&c, 7), Some(core::cmp::Ordering::Equal));
assert_eq!(midpoint(&c, 7), 7);
assert_eq!(round(&c, 7), 7);
assert_eq!(truncate(&c, -5), -5);

fn ceil(p: (i32, i32)) -> i32 { p.0.max(p.1) }
fn floor_p(p: (i32, i32)) -> i32 { p.0.min(p.1) }
fn inner(x: i32) -> (i32, i32) { (x, x) }
let m: Conn<(i32, i32), i32> = Conn::new(ceil, inner, floor_p);
assert_eq!(median(&m, 1, 2, 3), 2);
```

## Review

- **Design deviation 1 (midpoint formula).** Haskell uses
  `lo / 2 + hi / 2` under `Fractional a`. Rust extends the bound
  set to `Add + Sub + Div + From<u8>` (allowing integer sources),
  and uses `lo + (hi - lo) / 2` to keep the result exact for
  integer inputs whose `lo == hi` is odd. The Haskell formula
  evaluated to `6` for `lo == hi == 7` under `i32`, breaking the
  identity-Conn doctest; the offset form returns `7`.
- **Design deviation 2 (round arithmetic bounds).** Haskell `round`
  carries `(Num a, Preorder a)`. Rust splits this into the minimal
  `Copy + PartialOrd + Sub<Output = A> + From<u8>` — the body uses
  `interval` (needs `Sub`) and `truncate` (needs `From<u8>` for the
  zero literal). No `Add` / `Div` are needed because `round` does
  not call `midpoint` directly.
- **Plan-mode plan deviation.** The earlier draft (in
  `~/.claude/plans/ptal-at-the-deferred-adaptive-llama.md`)
  speculated several signatures that turned out to differ from
  Haskell verbatim (notably `interval` returning `(B, B)` instead
  of `Option<Ordering>`, and `median` taking `Conn<A, B>` instead
  of `Conn<(A, A), A>`). Web-fetching the upstream `Cast.hs`
  resolved those before any code shipped. The implemented
  signatures match Haskell.
- **`#[ignore]`d properties.** None. Every new proptest runs.
