## Tasks

### T1 — `src/interval.rs` (new module)

Port `Data.Order.Interval` from the Haskell crate to Rust:

```rust
pub enum Interval<A> {
    Empty,
    Bounded { lo: A, hi: A },
}
```

Methods: `empty`, `singleton` (requires `A: Clone`), `new` (preorder
sort + collapse-to-`Empty` on incomparable; needs `A: PartialOrd`),
`contains`, `endpts`, `imap` (monotonic map; non-monotonic `f`
collapses to `Empty` because `Interval::new` re-sorts).

Trait impls: `PartialEq` (over `A: PartialEq`), `PartialOrd`
(containment preorder: `Empty ≤ all`; `Bounded { lo₁, hi₁ } ≤
Bounded { lo₂, hi₂ } ⟺ lo₂ ≤ lo₁ && hi₁ ≤ hi₂`), `Debug`, `Clone`,
`Copy` for `A: Copy`.

Re-exported from crate root as `pub use interval::Interval;`.

### T2 — Replace `pub fn interval` body

In `src/conn.rs:471`, replace the existing `Option<Ordering>` body
with the bracket function:

```rust
pub fn interval<T, A, B>(t: &T, x: A) -> Interval<A>
where T: ConnK<A, B>, A: Copy + PartialOrd, B: Copy {
    let lo = t.conn_r().lower1(|b| b, x);
    let hi = t.conn_l().upper1(|b| b, x);
    Interval::new(lo, hi)
}
```

Same name, completely new return shape. Bound delta: drops
`Sub<Output = A>` (the old body needed it for the `Ordering`
comparison).

### T4 — Rewrite `round` to consume `Interval<A>`

Today's `round` (`src/conn.rs:660-664`) ties **toward zero**
(`Equal | None → truncate(t, x)`). Preserve that:

```rust
pub fn round<T, A, B>(t: &T, x: A) -> B
where T: ConnK<A, B>,
      A: Copy + PartialOrd + Sub<Output = A> + From<u8>,
      B: Copy
{
    match interval(t, x) {
        Interval::Bounded { lo, hi } => {
            match (x - lo).partial_cmp(&(hi - x)) {
                Some(Ordering::Greater) => t.conn_l().ceil(x),
                Some(Ordering::Less)    => t.conn_r().floor(x),
                _                       => truncate(t, x),
            }
        }
        Interval::Empty => truncate(t, x),
    }
}
```

Bound delta: drops `Add + Div` (was needed by `midpoint`); keeps
`Sub + From<u8> + PartialOrd`. Strict relaxation — every Conn that
satisfied the old bounds also satisfies the new.

### T5 — `filter_l` / `filter_r` inherent methods

```rust
impl<A: Copy, B: Copy> Conn<A, B, L> {
    pub fn filter_l(self, a: A, b: B) -> bool where B: PartialOrd {
        self.ceil(a) <= b
    }
}
impl<A: Copy, B: Copy> Conn<A, B, R> {
    pub fn filter_r(self, a: A, b: B) -> bool where B: PartialOrd {
        b <= self.floor(a)
    }
}
```

Co-located with `Conn<_, _, L>::ceil` (`src/conn.rs:148-169`) and
`Conn<_, _, R>::floor` (`src/conn.rs:286-292`). Each gets a doctest.

## Deferred

- **`Maybe<A> ↔ Interval<A>` Galois connections.** The Haskell crate
  sketches three triples in commented-out code at
  `Data/Order/Interval.hs:93-115` — `Nothing → Empty`,
  `Just a → singleton a`; collapse-to-join (`x ⋁ y`) and
  collapse-to-meet (`x ⋀ y`) for the back direction. Porting needs
  a `Lattice` trait on `A` (join/meet); this crate doesn't yet have
  one in scope for `Interval`. Deferred.
- **`Lattice<Interval<A>>` impl.** Same prerequisite.
- **`Interval<A>` arithmetic instances (Minkowski-style interval
  arithmetic).** Out of scope; this crate isn't an interval-
  arithmetic library.
- **README mirror.** No new headline `Conn` const introduced; the
  README mirror rule from `CLAUDE.md` does not apply.
- **Renaming `filter_l` / `filter_r` to `in_filter` / `in_ideal`**
  for prose flow. Names match Haskell precedent; rename is a
  no-op-for-callers diff if needed later.
