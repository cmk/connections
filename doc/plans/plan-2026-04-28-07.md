# Plan 29 — Phantom-kind soundness for `Conn`

## Goal

Re-parameterize `Conn<A, B>` as `Conn<A, B, K: Kind = T>` with a sealed
`Kind` trait + marker types `L`, `R`, `T`, so the property-test layer
enforces at compile time which laws apply to which Conn shape. Surface
only `Conn 'L` and `Conn 'T` in the public constructor API; provide
`swap_left` / `swap_right` for the slot-swap relabeling so users who need to
reverse a connection go through that conversion explicitly. Mirror the
parent Haskell library's `Cast (k :: Side) a b` design.

## Motivation

The current Rust port collapses `Cast 'L` / `Cast 'R` / `Cast 'T` (full
triple) into a single `Conn<A, B>` discriminated only by which
constructor was used (`new_left` / `new_right` / `new` / `new_iso`).
The type system doesn't know which kind a given Conn is, so:

- A user can call `conn_galois_l(&new_right_conn, ...)` and get a
  proptest **runtime failure** instead of a compile error.
- Plan 27 had to hand-strip the `_r` family of property tests from
  every converted Conn — exactly the per-Conn audit the kind-tag
  prevents.
- The `Conn::new(ceiling, inner, floor)` constructor accepts any three
  fns; only the property tests catch a bogus full-triple, which is
  itself a tooling problem.

The Haskell parent uses DataKinds to make this static: `Cast 'L a b`
and `Cast 'R a b` are distinct types; per-kind property fns
(`adjointL :: Cast 'L a b -> ...`, `adjointR :: Cast 'R a b -> ...`)
refuse the wrong kind at compile time.

## Math design notes

Three structural equivalences shape the API surface:

1. **`Conn 'L a b ≡ Conn 'R b a`** — slot-swap (no Ord flip): the
   one-sided `(g: a→b, h: b→a)` pair satisfies the same Galois law
   under both labelings. This is what `swapL` / `swapR` return.
2. **`Conn 'L a b ≡ Conn 'L (Down b) (Down a)`** — order-duality:
   reversing both ≤'s preserves the iff. (Rust has no generic Down
   wrapper, so this lives in the math docs, not the API.)
3. Combining (1) and (2): `Conn 'R a b ≡ Conn 'L (Down b) (Down a)`.

Per (1), every right-Galois Conn is the swap of a left-Galois Conn.
The Haskell parent treats L as the canonical form.

**API consequence:** keep `Conn::new_right` in the public surface,
but do not provide Right-Galois connections in the library.

## Tasks

### T1 — `Kind` trait + markers in `src/conn.rs`

Add a sealed `Kind` trait with three marker types `L`, `R`, `T`.

```rust
mod kind {
    mod sealed { pub trait Sealed {} }
    pub trait Kind: sealed::Sealed {}

    pub struct L;
    pub struct R;
    pub struct T;

    impl sealed::Sealed for L {}
    impl sealed::Sealed for R {}
    impl sealed::Sealed for T {}
    impl Kind for L {}
    impl Kind for R {}
    impl Kind for T {}
}
```

### T2 — Re-parameterize `Conn<A, B>` to `Conn<A, B, K: Kind = T>`

Add a `PhantomData<K>` field. Default `K = T` keeps existing
`Conn<A, B>` annotations source-compatible (they resolve to
`Conn<A, B, T>`).

**Note on parameter ordering:** Rust requires defaulted type
parameters to be trailing. `Conn<K, A, B>` with `K = T` default does
not compile because `A` and `B` would have to default too (which
doesn't make sense for general types). Workaround: keep `K` last
internally, expose Haskell-shape ordering via type aliases:

```rust
pub type ConnL<A, B> = Conn<A, B, L>;
pub type ConnR<A, B> = Conn<A, B, R>;
```

Type aliases are first-class — `ConnL<i64, i32>` is a real, usable
type that appears in error messages and rustdoc with its alias name.

### T3 — Move constructors into per-kind impl blocks

```rust
impl<A, B> Conn<A, B, L> {
    pub const fn new_left(ceiling: fn(A) -> B, inner: fn(B) -> A) -> Self { ... }
    pub const fn fwd_left(rev: Conn<Reverse<B>, Reverse<A>, L>} -> Self { ... }
}

impl<A, B> Conn<A, B, R> {
    pub const fn new_right(inner: fn(B) -> A, floor: fn(A) -> B} -> Self { ... }
    pub const fn fwd_right(rev: Conn<Reverse<B>, Reverse<A>, R>} -> Self { ... }
}

impl<A, B> Conn<A, B, T> {
    pub const fn new(ceiling: fn(A) -> B, inner: fn(B) -> A, floor: fn(A) -> B) -> Self { ... }
    pub const fn new_iso(forward: fn(A) -> B, back: fn(B) -> A) -> Self { ... }
}
```

### T4 — `swap_left`, `rev_left`, `swap_right`, `rev_right` for slot-swap

Per the `Conn 'L a b ≡ Conn 'R b a` equivalence:

```rust
impl<A, B> Conn<A, B, L> {
    pub const fn swap_left(self) -> Conn<B, A, R> {
        // ceiling: A→B becomes inner of the swapped R-Conn.
        // inner: B→A becomes floor of the swapped R-Conn.
        // (`floor = ceiling` structurally on the R side; constructor
        //  wires it as a fn pointer.)
    }
    pub const fn rev_left(self) -> Conn<Reverse<B>, Reverse<A>, L> { ... }
}

impl<A, B> Conn<A, B, R> {
    pub const fn swap_right(self) -> Conn<B, A, L> { ... }
    pub const fn rev_right(self) -> Conn<Reverse<B>, Reverse<A>, R> { ... }
}
```

These are zero-cost type-level relabelings.

### T5 — Re-bound property fns in `src/prop/conn.rs` by kind

```rust
pub fn conn_galois_l<A, B>(c: &Conn<A, B, L>, ...) -> bool { ... }
pub fn conn_galois_r<A, B>(c: &Conn<A, B, R>, ...) -> bool { ... }
pub fn conn_floor_le_ceil<A, B, K: Kind>(c: &Conn<A, B, K>, ...) -> bool { ... }
pub fn conn_idempotent<A, B, K: Kind>(c: &Conn<A, B, K>, ...) -> bool { ... }
// ...
```

Closure / kernel / monotone fns split per-kind (Haskell precedent —
see `Data.Connection.Property`'s `closedL` / `closedR` /
`monotonicL` / `monotonicR` / `kernelL` / `kernelR` /
`idempotentL` / `idempotentR`).

After this lands, `conn_galois_l(&n_kind_conn, ...)` compiles (T
implies L), `conn_galois_l(&r_kind_conn, ...)` is a compile error,
and Plan 27's per-Conn test cleanup becomes structurally impossible
to forget.

### T6 — Bridge fns from `T` to `L` / `R`

Full triples satisfy both L and R laws, so `Conn<_, _, T>` should be
usable wherever `Conn<_, _, L>` or `Conn<_, _, R>` is expected:

```rust
impl<A, B> Conn<A, B, T> {
    pub const fn as_left(&self) -> &Conn<A, B, L> { ... }
    pub const fn as_right(&self) -> &Conn<A, B, R> { ... }
}
```

(Or use a trait `ImpliesL` with blanket impls for `L` and `T`, and
similarly `ImpliesR`. The `as_*` viewers are simpler and Haskell's
`forall k. Cast k a b` analog is rarely needed at the property-test
layer.)

### T7 — Audit existing Conn consts + tests

Every existing `pub const`: should compile unchanged because of the
`K = T` default. Any property-test call that passes a `K = L` Conn
to `conn_galois_r` (i.e. anywhere Plan 27 hand-stripped a test) gets
deleted by the type system.

Check for places where users currently call `.swap_right()` or want one —
none in this crate today (pure-Rust port doesn't yet exercise the
swap), but the Haskell parent's `Time.hs` patterns might be useful
references.
