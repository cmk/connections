# Plan 03 — Bi-Heyting trait methods + property tests

## Goal

Flesh out the deferred `Heyting` and `Coheyting` trait signatures with
provided (default) methods for derived operations, rename
`Coheyting::sub` → `coimp`, and port the property test functions from
`Data.Lattice.Property` so downstream crates can instantiate them with
their own proptest strategies.

## Context

The traits in `src/lattice.rs` have been signature-only since Plan 01.
The downstream `agogo` crate (Plan 16) implemented the bi-Heyting
algebra on its `Grid` lattice using standalone functions. This sprint
promotes the derived operations (`neg`, `mid`, `coneg`, `comid`) into
the trait definitions so any implementer gets them for free, and ships
generic property test functions that any downstream can reuse.

## Tasks

### T1 — Trait method additions + rename

In `src/lattice.rs`:

1. Rename `Coheyting::sub` → `Coheyting::coimp`.
2. Add provided methods to `Heyting`:
   - `fn neg(&self) -> Self { self.imp(&Self::bot()) }`
   - `fn mid(&self) -> Self { self.join(&self.neg()) }`
3. Add provided methods to `Coheyting`:
   - `fn coneg(&self) -> Self { Self::top().coimp(self) }`
   - `fn comid(&self) -> Self { self.meet(&self.coneg()) }`
4. Port doc comments from `Data.Lattice` (Haskell `//`, `\\`, `neg`,
   `non`, `middle`, `boundary` docstrings).

### T2 — Property test functions

In `src/property.rs`, add generic functions parameterized over trait
bounds. Each takes values by reference and returns `bool`. Downstream
crates instantiate them in `proptest!` blocks with their own strategies.
