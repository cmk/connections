# Plan 03 — Hoist the full Conn API onto `ConnL`/`ConnR`/`ConnK` methods

## Goal
Give markers and `Conn` values one uniform, method-call surface: move
the per-side accessors onto `ConnL`/`ConnR` and the two-sided helpers
onto `ConnK`, all as default methods, so `MARKER.ceil(x)`,
`MARKER.upper1(f, x)`, `MARKER.round(x)`, `MARKER.interval(x)`,
`MARKER.median(x, y, z)` work directly — no `view_l`/`view_r` step and no
free-function call in normal use.

## Motivation
Today every accessor is **inherent on `Conn` values** and the two-sided
helpers are **free functions**. Marker structs (`conn_k!` / `iso!` /
`compose_k!` / `lift_k!` / `nz_int_ext!`) carry no inherent accessors, so
using one means projecting first — `NDURU064.view_l().ceil(x)` in-crate,
or the swapped-frame workaround `NDURU064.swap_r().ceil(x)` downstream
(`gitjig-lib/src/time/uhlc.rs:78-84,258`, forced because the inherent
`view_l`/`view_r` are `pub(crate)`).

Plan-02 already promoted the **free** `view_l`/`view_r` fns to `pub`, but
the **marker inherent** `view_l`/`view_r` methods stay `pub(crate)`
(conn.rs:504-505, 527-528, 562 all call them "crate-private"), so the
downstream workaround remains.

The fix is to hoist the whole surface onto the capability traits as
**default methods** over the one required primitive (`swap_l`/`swap_r`),
and let `Conn` values join the traits via blanket impls. The inherent
`const fn` accessors stay on values as the base the defaults delegate to,
so `const` composition (`compose_*!`/`lift_*!`) is untouched.

`ConnK` is **kept** (name unchanged) and finally earns its keep: the
two-sided helpers become its default methods, so any triple gets them by
method call. It stays a blanket-impl'd super-trait, so no marker needs
hand-written impls.

## Trait ownership of the API
| Trait | Required | Provided (default) methods |
|-------|----------|----------------------------|
| `ConnL` | `swap_l` | `view_l`, `ceil`, `upper`, `ceil1`, `ceil2`, `upper1`, `upper2` |
| `ConnR` | `swap_r` | `view_r`, `floor`, `lower`, `floor1`, `floor2`, `lower1`, `lower2` |
| `ConnK` | — (blanket) | `interval`, `truncate`, `truncate1`, `truncate2`, `round`, `round1`, `round2`, `median`\* |

\* `median` needs the diagonal shape `A = (B, B)`; added with a
`where Self: ConnL<A = (<Self as ConnL>::B, <Self as ConnL>::B)>`
constraint, so it's only callable on the diagonal markers it already
required (`N5Float`-shaped). If that bound proves awkward it stays a free
fn only — decided at implementation, noted in Review.

## What changes vs. what stays

**Changes**
- `ConnL`/`ConnR` gain the per-side API as provided methods; `ConnK`
  gains the two-sided helpers as provided methods.
- Blanket impls `impl ConnL for Conn<A,B,L>` / `impl ConnR for
  Conn<A,B,R>` so values join the traits.
- Marker inherent `view_l`/`view_r`: `pub(crate)` → `$vis` (public), so
  they don't shadow the trait default with a privacy error.

**Stays (invariant — do not touch)**
- Inherent `const fn` `swap_l`/`swap_r`/`ceil`/`floor`/`upper`/`lower`/
  `*1`/`*2` on `Conn<_,_,L>` / `Conn<_,_,R>` values — the const base that
  the compose/lift macros and every `const` position depend on. Value
  receivers keep resolving to these (inherent shadows trait), so they
  stay const.
- `swap_l`/`swap_r` semantics: single required trait method, returns a
  `Conn` **value** (struct→const bridge). `compose_k!`/`lift_k!` inherent
  const swap/view unchanged.
- `ConnK` name, its `ConnL + ConnR<A=..,B=..>` super-trait bound, its
  blanket impl, its prelude entry. Conn **values** stay one-sided (a
  `Conn<A,B,L>` is `ConnL` only), so they never accidentally become
  `ConnK` — the two-sided methods remain triple-only, which is correct.
- Free `view_l`/`view_r` fns and the eight two-sided free fns: retained
  as forwarders (chosen over removal — bounded diff; full removal is
  Deferred).

## Tasks

### T1 — Blanket `ConnL`/`ConnR` impls for `Conn` values
`src/conn.rs`:
```rust
impl<A: Copy, B: Copy> ConnL for Conn<A, B, L> {
    type A = A; type B = B;
    fn swap_l(&self) -> Conn<B, A, R> { Conn::swap_l(*self) }   // inherent const swap
}
impl<A: Copy, B: Copy> ConnR for Conn<A, B, R> {
    type A = A; type B = B;
    fn swap_r(&self) -> Conn<B, A, L> { Conn::swap_r(*self) }
}
```
Coherent (distinct from every marker struct). Makes Example 6's stale
"blanket impl ConnL for Conn<A,B,L>" comment true.

### T2 — Per-side default methods on `ConnL`/`ConnR`
`src/conn.rs`. Keep `swap_l`/`swap_r` required; add provided methods
delegating through `view_l`/`view_r` to the inherent accessor on the
produced value:
```rust
pub trait ConnL {
    type A: Copy; type B: Copy;
    fn swap_l(&self) -> Conn<Self::B, Self::A, R>;
    fn view_l(&self) -> Conn<Self::A, Self::B, L> { self.swap_l().swap_r() }
    fn ceil(&self, a: Self::A) -> Self::B { self.view_l().ceil(a) }
    fn upper(&self, b: Self::B) -> Self::A { self.view_l().upper(b) }
    fn ceil1<H: FnOnce(Self::A)->Self::A>(&self, h: H, b: Self::B) -> Self::B { self.view_l().ceil1(h, b) }
    fn ceil2<H: FnOnce(Self::A,Self::A)->Self::A>(&self, h: H, b1: Self::B, b2: Self::B) -> Self::B { self.view_l().ceil2(h, b1, b2) }
    fn upper1<H: FnOnce(Self::B)->Self::B>(&self, h: H, a: Self::A) -> Self::A { self.view_l().upper1(h, a) }
    fn upper2<H: FnOnce(Self::B,Self::B)->Self::B>(&self, h: H, a1: Self::A, a2: Self::A) -> Self::A { self.view_l().upper2(h, a1, a2) }
}
```
`ConnR` mirrors it (`view_r`, `floor`, `lower`, `floor1/2`, `lower1/2`).
Signatures copy the inherent ones exactly (conn.rs:237/247/276/286,
339/349/379/389 at plan time — confirm by grep). Two layers result:
**value** receiver → inherent const (shadows default); **marker/generic**
→ default, one projection, non-const.

### T3 — Two-sided default methods on `ConnK`
`src/conn.rs`. Lift the eight helper bodies into `ConnK` default methods
(the trait currently has an empty body, conn.rs:553):
```rust
pub trait ConnK: ConnL + ConnR<A = <Self as ConnL>::A, B = <Self as ConnL>::B> {
    fn interval(&self, x: <Self as ConnL>::A) -> Interval<<Self as ConnL>::A> { /* current interval() body over self */ }
    fn truncate (&self, x: <Self as ConnL>::A) -> <Self as ConnL>::B { … }
    fn truncate1<H>(&self, h: H, x: <Self as ConnL>::B) -> <Self as ConnL>::B where H: FnOnce(<Self as ConnL>::A)->…  { … }
    fn truncate2<H>(…);  fn round(&self, x) -> …;  fn round1<H>(…);  fn round2<H>(…);
    // median: constrained (see Trait-ownership note)
}
```
Each body is the current free-fn body with `t` → `self` and
`view_l(t)`/`view_r(t)` → `self.view_l()`/`self.view_r()`. The blanket
`impl<T> ConnK for T` stays, so every marker gets all eight by method
call for free.

### T4 — Make marker inherent `view_l`/`view_r` `pub`, and refresh stale docs
Five marker macros emit `pub(crate) const fn view_l`/`view_r`
(conn.rs and extended.rs / core.rs `*_k!` / `iso!` / `nz_int_ext!`
emitters — grep `pub(crate) const fn view_l` for exact sites). A private
inherent method shadows the new trait default → downstream privacy error.
Change `pub(crate)` → `$vis` (matching the sibling `swap_l`/`swap_r`).
`MARKER.view_l()` then resolves to the inherent **const** method publicly;
value/generic receivers fall to the trait default.

Doc-prose cleanup (gardener — these go stale the moment the inherent view
is public and the traits carry the API):
- `ConnL`/`ConnR` trait docs (conn.rs ~500-537): drop "the marker's
  inherent `const fn view_l()` is crate-private, so downstream cannot call
  it"; keep the const-position guidance ("in `const` position spell the
  view as `t.swap_l().swap_r()`") since trait methods still aren't const.
- Free-`view_l` doc (conn.rs ~558-563): drop "the crate-private inherent
  `M.view_l()`" phrasing.

### T5 — Free fns become forwarders
`view_l`/`view_r` (conn.rs:564/572) and the eight two-sided fns
(`interval` 621, `truncate`, `truncate1`, `truncate2`, `round`, `round1`,
`round2`, `median` — grep for exact lines) become one-line forwarders to

### T6 — Simplify call sites
Drop the projection step in normal-usage doctests/tests now that markers
answer directly: `MARKER.view_l().ceil(x)` → `MARKER.ceil(x)`,
`view_l(&M).ceil(x)` → `M.ceil(x)`, `round(&M, x)` → `M.round(x)`, etc.
Touches `EXAMPLES.md`, `time/duration.rs`, `uhlc/ntp64.rs`, `prop.rs`,
and the `median`/`round`/`interval` doctests. **Keep** `view_l`/`view_r`

macro positions (plan-02 keep-set) untouched. Fix `median`'s bogus
`T: ConnK<(A, A), A>` doc pseudo-syntax (`ConnK` takes no type params).

## Review

### Delivered vs. planned
- **Delegation direction inverted (deviation from T3/T5).** The plan had
  `ConnK` methods *hold* the two-sided bodies and the free fns forward.
  Implemented the reverse: the eight `ConnK` methods are thin forwarders
  to the retained free fns (`round(self, x)` etc.), which keep the real
  bodies and their rich doctests. Rationale: forwarding avoids
  fully-qualified `<Self as ConnL>::A::from(0)` associated-type noise in
  the bodies and keeps the free-fn doctests intact. User-facing API and
  the `_eq_free_fn` verification are identical either way; the eventual
  free-fn deletion (Deferred) will relocate the bodies then. The per-side
  `ConnL`/`ConnR` accessors *do* carry real (trivial) default bodies —
  no free `ceil`/`floor` fn exists to forward to.
- **`median` kept as a method** — the diagonal `A=(B,B)` `where` clause
  compiled cleanly; no fallback to free-fn-only needed.
- **T6 scoped down.** Simplified the user-facing demos (EXAMPLES.md
  Example 3/7, the `TDURSECS` module doctest) to the direct method form.
  Left the internal `#[cfg(test)]` cosmetic `.view_l().ceil()` swaps
  (e.g. `time/duration.rs:260-268`, `uhlc/ntp64.rs:63-66`) for the
  Deferred free-fn-deletion sweep — they compile and pass unchanged.

### Implementation findings
- **Infinite-recursion trap found & fixed (all 4 emitters).** Each
  marker's `impl ConnL::swap_l(&self)` body was `self.view_l().swap_l()`.
  Adding a trait `view_l` default changed method resolution on the
  concrete `&self` receiver: `self.view_l()` now binds the trait default
  (`&self` matches `&$name` before the inherent by-value `view_l` does)
  → trait `view_l` calls `swap_l` → stack overflow (caught by
  `ipv6ipv4_laws::bracket_contains_x`). Fixed by having the trait `swap`
  impls call the inherent const swap by path (`$name::swap_l(*self)`),
  which prefers the inherent. The inherent const swap bodies (by-value
  `self`) were already correct and untouched.
- **Free fns relaxed to `?Sized`.** The eight two-sided free fns had an
  implicit `T: Sized` bound that the forwarding methods (`?Sized` `Self`)
  couldn't satisfy. Relaxed to `?Sized`, matching `view_l`/`view_r` — a
  strict generalization.
- **`compile_fail/round_on_l_conn` re-blessed (T1 consequence).**
  `Conn<i32,i32,L>` now impls `ConnL` (blanket), so the old "not `ConnL`"
  error is gone; `round` still rejects the L-value for lacking `ConnR`
  (one error instead of two, with a cleaner "implemented for
  `Conn<A,B,R>`" help). The guard still holds.
