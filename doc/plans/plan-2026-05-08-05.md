# Plan 2026-05-08-05 ‚Äî Replace 1-ns walk with chunk-bottom solver

## Goal

Eliminate the unbounded 1-ns backward/forward walk in float‚ÜíDuration
and float‚ÜíEpoch `Conn`s by replacing it with a binary-search solver in
`i128`-nanosecond space.

## Context

`def_walk_helpers!` (`src/float.rs:743-830`) emits four `loop` bodies;
the two live ones (`ascend_to_ceil`, `descend_to_ceil`) are bounded ‚â§ 2
steps when the destination is a narrow float (Kani-proven for `f32` and
`f16`) but **unbounded in practice** when the destination is a 1-ns
type. At `f64::MAX` the f64 plateau spans ~2 √ó 10¬π¬≤ ns; the walk does
not terminate. The current external rim fast-paths (`if v <= min_secs
{ return MIN }`) only mask the absolute saturation rim.

The fix: solve the cut directly. The plateau bottom is the smallest
representable-in-`$dst` value whose widening matches `widen(start)`;
that cut sits in `i128` ns space and is monotonely searchable in
`O(log plateau_width)` ‚âà 43 iterations vs. ~2 √ó 10¬π¬≤ before.

## Tasks

### T1 ‚Äî Extend `def_walk_helpers!` with `solve_to_ceil`

Add a 6-arg overload to the macro at `src/float.rs:743-830`. The new
form takes `$to_ns: fn($dst) -> i128` and `$from_ns: fn(i128) -> $dst`
in addition to the existing four args, and emits `solve_to_ceil`
alongside the existing walk helpers (which stay).

Body:

```rust
pub(super) fn solve_to_ceil(start: $dst, x: $src) -> ($dst, u32) {
    let n_start = $to_ns(start);
    let mut lo: i128 = n_start.saturating_sub(1 << 42);
    let mut hi: i128 = n_start.saturating_add(1 << 42);
    let mut steps: u32 = 0;
    while lo < hi {
        let mid = lo + (hi - lo) / 2;
        if $widen($from_ns(mid)) >= x {
            hi = mid;
        } else {
            lo = mid + 1;
        }
        steps += 1;
    }
    ($from_ns(lo), steps)
}
```

The four-arg invocations (float-narrowing) keep emitting only the walk
helpers ‚Äî unchanged.
