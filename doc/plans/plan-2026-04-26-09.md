# Plan 10 — `conn::time` module (Sprint Time)

## Context

The crate ships connection ladders for fixed-point, integer, float, and
audio-sample-rate types but has no time-domain module. Downstream users
working with the [`time`](https://crates.io/crates/time) crate's
calendar / clock / duration types currently have no lawful, rounding-aware
casts between them and integer rungs.

This sprint adds `connections::conn::time`, deriving the API entirely
from the `time` crate's public types. The Haskell reference module
`Data.Connection.Time` is consulted only as background; the Rust API is
**not** a port of it.

## Goal

Five public `Conn` constants in a new `src/conn/time.rs` module, named
under the new 8-character standard (`ABCDXWYZ` shape: 4 letters per
side), each with proptest coverage from `property::laws` plus
runnable doctest examples.

| Name       | Type                                       | Role               |
|------------|--------------------------------------------|--------------------|
| `DATEJDAY` | `Conn<Extended<Date>, i32>`                | exact bijection    |
| `TIMENANO` | `Conn<Extended<Time>, i64>`                | exact bijection    |
| `TIMESECS` | `Conn<Extended<Time>, i64>`                | rounding (subsec)  |
| `DURNSECS` | `Conn<Duration, Extended<i64>>`            | rounding (signed)  |
| `PDTMDATE` | `Conn<PrimitiveDateTime, Extended<Date>>`  | rounding (subday)  |

## Naming-convention note

This is the first module to use the 4-letter-each-side `ABCDXWYZ`
shape — `CLAUDE.md` lists it as "reserved — no current uses". The
domain types here are typed (`Date`, `Time`, `Duration`,
`PrimitiveDateTime`) rather than tier-coded, so an all-letter
identifier reads better than a digit-bearing one. The two halves
both pick `ABCD` independently of each other; the broader 4+4 rule
is unchanged.

## Tasks

### T1 — Scaffold

- `Cargo.toml`: add `time = "0.3"` to `[dependencies]`.
- `src/conn/mod.rs`: declare `pub mod time;` alongside the other
  domain modules; add a one-bullet description.
- `src/conn/time.rs`: module-level `//!` rustdoc explaining the new
  8-char naming shape, with a runnable doctest stub.

### T2 — `Ple` impls + arb strategies + preorder laws

- `Ple` impls for `Date`, `Time`, `Duration`, `PrimitiveDateTime`,
  delegating to their existing total `Ord`.
- Strategies in `src/property/arb.rs`: `arb_date`, `arb_time`,
  `arb_duration`, `arb_primitive_dt`, `arb_extended_date`,
  `arb_extended_time`, `arb_extended_i64`, `arb_jd_in_range`,
  `arb_ns_in_range`, `arb_secs_in_range`. Each biased toward
  MIN/MAX/MIDNIGHT/ZERO boundary cases.
- `mod tests` block in `time.rs` with `*_preorder` proptest mods
  driving `lattice_{reflexive, transitive, antisymmetric, bot, top}`.

### T3 — `DATEJDAY`

- `Conn<Extended<Date>, i32>`. Exact bijection on the finite
  portion; saturates at `Extended::NegInf` / `PosInf` for `i32`
  values outside `[Date::MIN.to_julian_day(), Date::MAX.to_julian_day()]`.
- Spot checks: Unix epoch (jd 2_440_588), Date::MIN/MAX round-trip,
  saturation extremes.
- Proptest: `galois_l/r`, `closure_l/r`, `kernel_l/r`, `monotone_l/r`,
  `idempotent`, `roundtrip_ceil/floor` (over `arb_jd_in_range`).
- Drop `floor_le_ceil` — same precedent as `conn::int::U008I016`:
  saturation makes `inner` non-injective and the law fails at the
  arms.

### T4 — `TIMENANO`

- `Conn<Extended<Time>, i64>`. Same shape as DATEJDAY: exact
  bijection on `[0, 86_400 × 10⁹)`, saturating outside.
- Spot checks: midnight↔0, end-of-day, noon, all four arms.
- Same proptest battery as T3.

### T5 — `TIMESECS`

- `Conn<Extended<Time>, i64>`. First proper-Galois rounding conn:
  `ceil` rounds up if `subsec_ns > 0`, `floor` truncates.
- Spot checks: midnight, exact-second, sub-second-rounding,
  end-of-day with full subsec, saturation arms.
- Proptest: full battery + `ulp_bound` on the Finite portion
  (rung extractor is identity on `i64`).

### T6 — `DURNSECS`

- `Conn<Duration, Extended<i64>>`. Source is plain `Duration`
  (already i64-bounded); rung is `Extended<i64>` because ceil at
  `Duration::MAX` and floor at `Duration::MIN` overflow `i64`.
- Sign-aware sub-second rounding via `checked_add` / `checked_sub`
  fall-through to `±Inf`.
- Spot checks: `Duration::ZERO`, positive subsec, negative subsec,
  both extremes, inner saturation.
- Proptest: full battery + `ulp_bound` (extractor flattens
  `±Inf → i64::{MIN, MAX}`) + roundtrip on `Finite` rung values.

### T7 — `PDTMDATE`

- `Conn<PrimitiveDateTime, Extended<Date>>`. Drops sub-day time:
  `floor` keeps the date, `ceil` rolls to `next_day` if past
  midnight (saturating to `PosInf` at `Date::MAX`).
- Spot checks: midnight (ceil = floor), one ns past midnight (ceil
  rolls), both extremes, `Date::MAX` with sub-day time.
- Proptest: full battery + `ulp_bound` (extractor collapses
  `NegInf → MIN_JD`, `PosInf → MAX_JD + 1`); `roundtrip_ceil`
  excludes `Date::MIN` (collides with `NegInf` arm).

### T8 — README + cross-linked example

- Add a `time` row to the "Connection families" table in `README.md`.
- Add a `DURNSECS` block to the Quick tour, mirrored verbatim into
  `conn::time`'s module-level doctest so `cargo test --doc` keeps
  the two in sync.
- Update the Layout tree to mention `conn/time.rs`.
- Update the "runtime dependencies" prose to include `time`.

## Out of scope (for this sprint)

- Float-seconds bridges (`f64`/`f32` ↔ `Duration`) — would compose
  through `ExtendedFloat<f64>` / `Extended<Nano>`; defer to a
  follow-up that integrates the float-ratio infrastructure.
- Enum-tag connections (`Weekday`/`Month` ↔ `u8`) — natural names
  like `WDAYU008` don't satisfy any of the four allowed 8-char
  patterns. Skip rather than invent awkward fits.
- `OffsetDateTime` / `UtcOffset` — no compelling adjoint pair until
  we decide on a canonical representation.
