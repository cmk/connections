# Plan 11 — README & docs polish for 0.1 crates.io release

## Goal

Land a clean, publishable 0.1.0: every public item documented to at
least ONELINER tier, README reordered with a friendly on-ramp and an
inline ASCII Galois diagram, and CI deploying rustdoc to GitLab Pages
on `main`.

### T2 — Reorder README sections

Promote `What is a connection?` from h3 to h2. Add a new
`## How connections work` section walking through three
`Ordering ↔ bool` examples with verification tables, mirroring the
Haskell `connections` README's pedagogical progression:

1. one-sided `Conn::new_left` (analog of `ordbin :: Cast 'L`) — 2-col table
2. flipped one-sided (analog of `binord`) — 2-col table, motivates the chain
3. full triple `Conn::new` (analog of `Cast k`) — 3-col table, with a closing
   note that `round`/`truncate`/`midpoint`/`interval`/`median` work on every
   `Conn<A, B>` regardless of construction; on one-sided ones they degenerate
   trivially because `floor == ceil`.

Add `## Installation` (cargo add, MSRV, `f16` / `testing` flags). Final
order: preamble → What are Galois connections? → How connections work →
Why this crate → Installation → The core type → Quick tour → What's
lawful → What's shipped vs deferred → Layout → Relationship to Haskell
→ Testing → Links.

## Review

### Plan-vs-implementation deltas

- **T4 doc-coverage scope correction.** The audit cited at plan time
  reported all 78 int-submodule consts as `NONE`-tier; in fact the
  `ext_int!`/`int_int_narrow!`/`uint_int_sat!`/`int_uint_narrow!`
  macros in `src/int.rs` already emit `#[doc = concat!(...)]` per
  generated const. T4 was reduced to the 180 fixed-submodule consts
  (10 macros, none of which had per-const docs). Outcome: every
  macro-generated `pub const Conn<…>` in both `int/` and `fixed/`
  now carries a one-liner.

- **T7 const name in plan was wrong.** The plan listed
  `fixed::i8::I008I004` as a doctest target. The Conn that hosts the
  Q-format reduction example in that module is `I004I000` (Q3.4 →
  Q7.0). The doctest was added under the correct name.

- **T7 `round`/`truncate`/`median` doctests not added.** The plan
  listed these as candidates "where prose alone is hand-wavy."
  Reviewing the existing source revealed that all three free
  functions in `src/conn.rs` already carry working `/// # Examples`
  blocks (cargo test --doc was already running them at the start of
  the sprint). Adding more would duplicate without adding clarity.
  Dropped, with no test gap created.

- **`/img` excluded from published crate.** Not in the original plan;
  added when `cargo package --no-verify` showed `img/example.png`
  shipping inside the .crate despite no longer being referenced from
  the README. The PNG remains in the repo; the publish bundle
  drops 48 KB.

- **Q-notation fix during sprint review.** The first iteration of
  the `fixed/i8.rs` doctest labeled `FixedI8<U4>` as Q0.4 and
  `FixedI8<U0>` as Q4.0. Tier-1 reviewer flagged: the correct
  designations are Q3.4 and Q7.0. Fixed in the same commit round.

- **Plan filename collision.** A parallel sprint (Plan 22, CI cost
  reduction + walk-perf) landed `doc/plans/plan-2026-04-27-11.md`
  on main while this branch was open. On rebase, this plan moved to
  `plan-2026-04-27-12.md`; the `Plan 11 — README & docs polish`
  internal title is unchanged.
