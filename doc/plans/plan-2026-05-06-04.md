# Plan 41 — `hifi` cargo feature + `hifi/duration.rs` (Sprint 1 of hifitime integration)

## Goal

Land the `hifi` cargo feature scaffolding and the four
`hifitime::Duration`-only Conns: `HDURNANO`, `HDURSECS`, `F064HDUR`,
`F032HDUR`. No Epoch, no cross-crate Duration bridges, no calendar
enums — those come in later sprints. The driving choice for the four
shapes is **single-side `Extended<…>`** (per Chris's correction of the
master plan): two-sided wrapping is an antipattern; STDR* is the
cautionary precedent we don't extend.

### T2 — Arb strategies (in `src/prop/arb.rs`, cfg-gated)

- `arb_hifi_duration()` — full canonical range via
  `Duration::from_total_nanoseconds(any::<i128>())` plus explicit
  `Just`s for `ZERO`, `MIN`, `MAX`, `EPSILON`, `MIN_NEGATIVE`,
  ±1 ns, ±1 s, and ±(1 s − 1 ns).
  - Why all-via-`from_total_nanoseconds`: hifitime's PartialOrd on
    Duration is field-wise on `(centuries, nanoseconds)`. For
    canonical Durations (`nanoseconds < NANOSECONDS_PER_CENTURY`)
    this agrees with semantic order, but `from_parts` accepts
    non-canonical inputs and breaks it. `from_total_nanoseconds`
    always normalizes.
- `arb_extended_hifi_duration()` — `prop_oneof![1 NegInf, 1 PosInf,
  2 Finite(arb)]` mirror of `arb_extended_duration`.
- `arb_hifi_duration_bounded_f64()` — bound to ±10⁹ s (~31 years)
  so float ULP-walk budgets stay within ~120 steps per call. Same
  rationale as `arb_duration_bounded_f64` (existing).
- `arb_extended_hifi_duration_bounded_f64()` /
  `arb_extended_hifi_duration_bounded_f32()` — Extended-wrapped
  bounded variants for the float-bridge proptests.
- `arb_hifi_total_nanos_in_range()` — `i128` strategy bounded to
  `[HDuration::MIN.total_nanoseconds(), HDuration::MAX.total_nanoseconds()]`
  for the `HDURNANO.roundtrip_ceil` proptest (mirrors
  `arb_unix_nanos_in_range`).
- All gated `#[cfg(all(feature = "testing", feature = "hifi"))]`.

### T3 — `HDURNANO`

`Conn<Extended<hifitime::Duration>, i128>` — ConnL.

Shape mirrors `OFDTNANO` (source-side `Extended`, plain `i128` rung):

```rust
fn hdurnano_ceil(d: Extended<HD>) -> i128 {
    match d {
        Extended::NegInf => HD::MIN.total_nanoseconds().saturating_sub(1),
        Extended::Finite(d) => d.total_nanoseconds(),
        Extended::PosInf => HD::MAX.total_nanoseconds().saturating_add(1),
    }
}

fn hdurnano_inner(n: i128) -> Extended<HD> {
    let min_n = HD::MIN.total_nanoseconds();
    let max_n = HD::MAX.total_nanoseconds();
    if n < min_n {
        Extended::NegInf
    } else if n > max_n {
        Extended::PosInf
    } else {
        Extended::Finite(HD::from_total_nanoseconds(n))
    }
}
```

`MAX.total_nanoseconds()` is `i16::MAX × NANOSECONDS_PER_CENTURY +
NANOSECONDS_PER_CENTURY = (1 << 15) × NANOSECONDS_PER_CENTURY ≈ 1.03 ×
10²³`, well inside `i128`'s ≈ 1.7 × 10³⁸ envelope, so the `±1`
saturation marker arithmetic can't overflow.

### T4 — `HDURSECS`

`Conn<Extended<hifitime::Duration>, i64>` — ConnL.

Shape mirrors `OFDTSECS`. `HD::MAX.to_seconds().abs()` ≈ 1.03 × 10¹⁴
s, well inside `i64`'s ±9.22 × 10¹⁸, so no rung-side overflow at the
extremes; saturation arms handle synthetic source ±∞.

```rust
fn hdursecs_ceil(d: Extended<HD>) -> i64 { ... ceil sub-second up ... }
fn hdursecs_inner(s: i64) -> Extended<HD> {
    let min_s = HD::MIN.to_seconds() as i64;  // safe — see range note above
    let max_s = HD::MAX.to_seconds() as i64;
    if s < min_s { NegInf } else if s > max_s { PosInf } else { Finite(HD::from_seconds(s as f64)) }
}
```

Note: `HD::from_seconds(f64)` is the simplest constructor; the sub-
second precision concern doesn't apply since we feed integer seconds.

### T5 — `F064HDUR` / `F032HDUR`

`Conn<F064, Extended<hifitime::Duration>>` and the f32 mirror — ConnL.

Direct port of `F064DURN` / `F032DURN`. Need:
- `shift_hifi_duration(n: i32, d: HD) -> HD` — saturating ±1ns step

- `hifi_duration_to_f32(d: HD) -> f32` — `d.to_seconds() as f32`
  (hifitime exposes `to_seconds() -> f64` only; the `as f32` cast is

  and the f32 sibling.
- `f064hdur_ceil` / `f032hdur_ceil` follow `f064durn_ceil` exactly,
  with `HD::MAX.to_seconds()` / `HD::MIN.to_seconds()` as
  `max_secs` / `min_secs` and `HD::from_seconds(v)` as the estimate.

## Review (sprint close)

- **`default-features = false` worked.** hifitime's `[features]`
  section gates serde / web-time / snafu under the `std` feature; the
  no-features build has no transitive surprises beyond
  `num-traits`/`lexical-core`/`libm`/`snafu` (the snafu core is
  `default-features = false` in hifitime). Compile clean against
  hifitime 4.3.0.
- **Walk-step budget** for `F064HDUR` / `F032HDUR` was within the
  64-cases / 200-shrink envelope used by `F064DURN` / `F032DURN`.
  The `arb_hifi_duration_bounded_f64` strategy pins `|total_ns| ≤
  10¹⁸` (≈ 31 yr) and `_f32` to `±10 s`, matching the time-crate
  precedent. No proptest cases needed extension.
- **No `#[ignore]`s** introduced.
- **Naming-table mention.** Skipped a row in `src/lib.rs`: that
  table only enumerates the 1-letter-prefix codes (`F` / `I` / `U`).
  The 4-letter `HDUR` / `NANO` / `SECS` mnemonics are documented in
  `src/hifi.rs` (the module's own docs), matching how
  `src/time.rs` handles its `DATE` / `TIME` / `DURN` / `STDR` /
  etc. mnemonics.
- **Asymmetric saturation bug surfaced and fixed during T3.** Initial
  `hdurnano_ceil(NegInf)` returned `min_n - 1`; this broke `galois_l`
  for `b ∈ [i128::MIN, min_n - 2]` because `inner(b) = NegInf` for the
  whole range while `ceil(NegInf) = min_n - 1` only satisfied `≤ b`
  for `b ≥ min_n - 1`. Correct shape (matching `OFDTNANO`):
  `ceil(NegInf) = i128::MIN`, `ceil(PosInf) = max_n + 1`. The
  asymmetry is forced — `inner(b) = NegInf` on a left-open ray, so
  `ceil(NegInf)` must be small enough that `≤ b` is always true.
  Same fix on `HDURSECS`. Doc-comment updated to spell out the
  derivation.
- **No two-sided `Extended`.** All four Conns ship with `Extended<…>`
  on at most one side, per the master plan rule. The precedent is
  set; Sprint 2's Epoch Conns will follow.
