# Plan 26 — Disentangle core primitives from `fixed/`

## Goal

Move every Conn whose endpoints are pure Rust core/std primitive types out
of `src/fixed/` and `src/float/` into a new `src/core/` module, leaving
`src/fixed/` to host only Conns where at least one endpoint touches the
`fixed` crate's Q-format wrappers.

## Context

Today the crate's top-level domain split is `float/` (IEEE-binary) vs
`fixed/` (everything finite-precision-with-integer-storage). The `fixed/`
module mixes three host crates inside one tree:

- **Rust core/std primitives** — `iN`/`uN` (the §1 sections of every
  `fixed/{i,u}*.rs`) and `core::num::NonZero<T>` (the §3 sections);
  `BOOLBE01`/`BOOLLE01` also currently live in `fixed/u008.rs`.
- **The `fixed` crate** — `FixedI<N><Frac>`/`FixedU<N><Frac>` Q-format
  wrappers (§2 sections) and the `I*Q000` / `U*Q000` cross-crate isos (§4).
- **Bridges** — `float_fixed!` instantiations linking IEEE floats to
  Q-format wrappers.

The host-crate boundary is invisible from the path:
`connections::fixed::i008::I008I016` is pure-core, but
`connections::fixed::i008::I008Q000` reaches into the `fixed` crate. The
only top-level `mod` for "this crate plus `core`/`std`" content was
`float/` for IEEE; non-IEEE core primitives piled into `fixed/` by
accident of source-side placement. This sprint introduces `src/core/` so
that:

- `connections::core::*` — host-crate is `core`/`std` only.
- `connections::fixed::*` — at least one endpoint touches the `fixed` crate.

The crate's invariant ("only top-level domain split is `float/` vs
`fixed/`") becomes "only top-level split is `core/` vs `fixed/`"; the
IEEE world folds into `core/` because every IEEE primitive ships in
core. CLAUDE.md's §"Module organization" updates accordingly.

## Review

- **`#[ignore]`s introduced:** none. All relocated `law_battery!` and
  `proptest!` blocks continue to pass at their new homes; no test was
  marked ignored during the migration.
- **Design deviation: `ExtendedFloat<T>` stayed in `src/float.rs`.**
  My initial sketch contemplated folding `ExtendedFloat` (and the
  ULP / `BitsToF64Rd` / `RoundDownFromF64` / `def_walk_helpers!` /
  `float_ext_int!` infrastructure) into `src/core.rs` so that
  `src/float.rs` and `src/float/` could be deleted entirely. On
  inspection `src/float.rs` is 2,284 lines of IEEE-only support code;
  flattening it into `src/core.rs` would have ballooned that file
  unhelpfully. Final shape: `src/float.rs` keeps the wrapper type and
  helpers (analogous to `src/extended.rs` for `Extended<T>`), and
  only the per-IEEE-width Conn files (`f016`/`f032`/`f064`) moved
  into `core/`. The `crate::float` path therefore continues to exist
  alongside `crate::core` and `crate::fixed`.
- **`super::` macro re-exports in `src/fixed.rs` were dropped after
  T3.** During T2 I added a transitional `pub(crate) use crate::core::{...}`
  block in `src/fixed.rs` so the per-host-int `fixed/X.rs` files
  could keep compiling against `use super::{ext_int, …}` while T3
  rewrote them to `use crate::core::{…}` directly. By the end of T3
  no consumer needed those re-exports, so they're gone in the final
  diff — the only thing `src/fixed.rs` still hosts is the
  `float_fixed!` / `float_fixed_l!` / `__float_fix_*_body!` macro
  family.
- **`bool` is a Rust primitive type name, not a reserved keyword.**
  `pub mod bool;` works fine — same shape as the existing `pub mod char;`
  in the old layout.
- **Recommendations for follow-up:** none required for this sprint.
  Possible future tightening:
  - The cross-host-int test references in `core/{i,u}*.rs` still
    carry the full glob-import set out of caution; once the
    `props_for_pair!` shape stabilises, the imports could be slimmed
    to only the cross-references actually used in spot tests.
  - `src/float.rs` is now structurally a "type + macros" module with
    no Conn submodules. If the float infrastructure ever shrinks,
    folding it into `src/core.rs` becomes a cheap follow-up; not
    worth the diff today.
