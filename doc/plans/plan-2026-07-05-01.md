# Plan 01 — `usize` Conn family

## Goal
Add a platform-independent `usize` cast family — `SIZEU032` (`usize → u32`,
saturating narrow) and `SIZEU064` (`usize → u64`, saturating embed) — in a new
`core::size` submodule, lawful on 16/32/64-bit targets via saturating `TryFrom`.

## Dependency Graph
T1 → T2 → T3

## Tasks

### T1 — `core::size` submodule with the two Conns
- **Problem.** The crate has no `usize` endpoint. `usize` is pointer-width
  (16/32/64-bit depending on target), so the fixed-width `as`-based std-int
  macros in `core.rs` cannot express a platform-independent cast: e.g.
  `uint_uint_narrow!(_, usize, u32)`'s `inner` is `x as usize`, which
  *truncates* a `u32 > usize::MAX` on a 16-bit target (`70000 as u16 == 4464`)
  instead of saturating. The value must be **clamped**, which requires
  `TryFrom`, not `as`.
- **Solution.** New file `src/core/size.rs`, registered `pub mod size;` in
  `src/core.rs`. Two `pub const Conn<usize, _>` (default kind `L`, one-sided
  left-Galois via `Conn::new_l`):
  - `SIZEU032 : Conn<usize, u32>`
    - `ceil(x) = u32::try_from(x).unwrap_or(u32::MAX)` — saturating narrow
      (lossless widen on a 16-bit target where `usize < u32`).
    - `inner(y) = if y == u32::MAX { usize::MAX } else { usize::try_from(y).unwrap_or(usize::MAX) }`
      — lossless widen with the **FINE_MAX fixup** (`inner(u32::MAX) =
      usize::MAX`). The fixup is what makes left-Galois hold on a 64-bit target,
      where source values above `u32::MAX` all `ceil` to `u32::MAX`: for
      `b = u32::MAX` the law's LHS `ceil(a) ≤ b` is always true, so the RHS
      `a ≤ inner(b)` must hold for every `a`, forcing `inner(u32::MAX) =
      usize::MAX`. Mirrors `uint_uint_narrow!`.
  - `SIZEU064 : Conn<usize, u64>`
    - `ceil(x) = u64::try_from(x).unwrap_or(u64::MAX)` — lossless embed on every
      real target (`usize ≤ 64` bits); the `unwrap_or` saturates only on a
      hypothetical `> 64`-bit `usize`.
    - `inner(y) = usize::try_from(y).unwrap_or(usize::MAX)` — saturating narrow;
      **no** FINE_MAX fixup (a widening, like `uint_uint!` — the saturation cap
      already lands `inner(u64::MAX) = usize::MAX` on 16/32-bit, and it is the
      identity on 64-bit).
- **Why `TryFrom` in a `const`.** The Conn `const` stores fn *pointers*; the
  `fn ceil` / `fn inner` bodies run at call time, so non-`const` `try_from` is
  fine (identical shape to `uint_uint!`, which stores `fn` items as pointers).

### T2 — Property + spot tests
- Inline `#[cfg(test)] mod tests` in `src/core/size.rs`: saturation spot-checks
  (`ceil(usize::MAX)`, the `inner(MAX)` fixup, small-value round-trips,
  `SIZEU064` embed identity on the low range).
- Integration battery `tests/core/size.rs` registered in `tests/core.rs`,
  reusing the `single_sided_props!` shape from `tests/core/u032.rs`
  (`galois_upper`, `ceil_monotone`, `inner_monotone`, `kernel`, `idempotent`)
  for both `SIZEU032` and `SIZEU064`. Strategies bias toward boundaries
  (`0`, `usize::MAX`, `u32::MAX`, `u64::MAX`).

## Verification

### Properties (must pass)
| Property | Module | Invariant |
|----------|--------|-----------|
| `galois_upper` (SIZEU032) | `tests::core::size` | `ceil(a) ≤ b ⟺ a ≤ upper(b)` for `usize`/`u32` |
| `galois_upper` (SIZEU064) | `tests::core::size` | `ceil(a) ≤ b ⟺ a ≤ upper(b)` for `usize`/`u64` |
| `ceil_monotone` (both) | `tests::core::size` | `a1 ≤ a2 ⟹ ceil(a1) ≤ ceil(a2)` |
| `inner_monotone` (both) | `tests::core::size` | `b1 ≤ b2 ⟹ upper(b1) ≤ upper(b2)` |
| `kernel` (both) | `tests::core::size` | `ceil(upper(b)) ≤ b` |
| `idempotent` (both) | `tests::core::size` | `upper∘ceil` idempotent |

### Spot checks
- `SIZEU032.ceil(usize::MAX) == u32::MAX`; `SIZEU032.upper(u32::MAX) == usize::MAX`
  (fixup); `SIZEU032.upper(0) == 0`; a mid value round-trips.
- `SIZEU064.upper(u64::MAX) == usize::MAX`; `SIZEU064.ceil(v) == v as u64` and
  `SIZEU064.upper(v as u64) == v` for `v < usize::MAX`.

### Build gates
- `cargo build`, `cargo test`, `cargo clippy --all-targets -- -D warnings`,
  `cargo doc` with `RUSTDOCFLAGS=-D warnings` — all clean.

## Deferred
- **`u8`/`u16`/`u128` rungs of the family** (`SIZEU008`, `SIZEU016`,
  `SIZEU128`) — the same `ceil = TryFrom-saturate`, `inner = widen (+ fixup on
  narrowings)` template covers them; left out to match the exact request.
- **`isize` family** — signed counterpart via an `Extended<isize>` source
  (would need the `ext_int!` treatment, not just `TryFrom`).
- **kani harnesses** — CBMC models one concrete pointer width per run, so a
  `usize` proof is target-arch-specific rather than "full bit-width domain."
  Left out; the README SMT claim is caveated instead of contradicted.
