# Plan 48 — Tighten the review prompts so reviewers stop ratifying author framing

## Goal

Update the local `/sprint-review` agent prompt and the GitLab
`claude-review` bot's `SYSTEM` prompt so they audit type-system claims
before reading the author's plan, and add a Pattern 9 calibration
example anchored on the MR !70 NaN/`iso!` failure mode.

## Context

MR !70 shipped three float Conns (`F016OBYT`, `F032OBYT`, `F064OBYT`)
declared `iso!` — claiming the Galois law `(ceil(a) ≤ b) == (a ≤
upper(b))` over `<=`. The byte encoding violates that law on
NaN-decoding bytes, but the proptest battery had been quietly weakened
to `*_total` (using `f32::total_cmp`), so the suite was green. Both
Claude reviewers (the local `/sprint-review` agent and the GitLab bot)
saw the diff with calibration and missed the soundness bug; Codex
caught it in round 2.

The systematic failure: the author's plan §Review noted "we use
`total_cmp` because the classical predicate fails on NaN," and both
reviewers accepted that as a *test deviation*. The plan pre-framed the
diff and the reviewers ratified the framing.

This sprint addresses the three axes the user identified:

1. **De-frame the diff.** Two-pass review in `/sprint-review`: cold
   review first (no plan), then plan-aware review. The plan can no
   longer prime the analysis.
2. **Enumerate trait/contract claims.** Both prompts now require an
   explicit trait-claim audit (quote the contract, quote the impl,
   state consistency) before any other review section.
3. **Test-exception escalation.** Both prompts treat relaxation
   suffixes (`*_total`, `*_relaxed`, `*_partial`, `*_weak`) as
   load-bearing signals: if the type wasn't also weakened, the
   mismatch is `must-fix`.

A fourth axis — *don't rely on these reviewers as a soundness gate* —
gets a one-paragraph reality check in CLAUDE.md so future planning
doesn't over-trust them.

## Tasks

### T1 — `.claude/commands/sprint-review.md`

Restructure Step 4's reviewer prompt template from one block to two
explicit passes:

- **Pass 1 (cold, no plan)** opens with **P1.A — Trait-claim audit**.
  For every new/changed `pub const` of `Conn<_, _>` type, every `iso!`
  / `conn_l!` / `conn_r!` / `compose!` / `triple!` invocation, and
  every non-derived impl of a law-bearing trait, write down (1) the
  contract, (2) the impl, (3) consistency. Then **P1.B** (the
  existing standard review: commit hygiene, code quality, test
  coverage, risks).
- **Pass 2 (plan-aware)** opens with **P2.A — Test-exception
  escalation**: scan the plan for relaxation language ("weaker
  predicate", "exclude", "saturate instead", …) and ask whether the
  type still claims the unrelaxed law. Promote Pass 1 audit findings
  to `must-fix` if the plan note matches. Then **P2.B — Plan
  conformance** (existing).
- Update Step 4's prelude bullet list to the new ordering: diff,
  commit log, conventions, calibration, Pass 1 instructions, plan,
  Pass 2 instructions.
- Output format adds: "Include the trait-claim audit verbatim — the
  audit log is the artifact that proves you did it."

### T2 — `scripts/claude_review.py`

Update the `SYSTEM` constant:

- Add a `## Trait-claim audit (do this first)` section near the top
  enumerating the macros / impls in scope and the (1)-(2) check.
- Add a paragraph treating relaxation-suffix test renames as
  load-bearing signals.
- Replace the existing "don't speculate about alternate `ceil` /
  `floor`" bullet with a revised version that keeps the legitimate
  guidance (closure shape is determined by `inner` and orders) but
  drops the chilling effect on type-claim audits. The new wording:
  "if the closures only satisfy [the law] on a subset, flag the type
  declaration, not the closures."

### T3 — `doc/reviews/review-calibration.md`

Append Pattern 9 to the existing eight-pattern file, modeled on the
MR !70 shape: `iso!` claim + `*_iso_total` proptest battery, with the
review comment noting the type-system violation and proposing
weakening the declaration (NonNan newtype / totalOrder byte-newtype /
ExtendedFloat with custom ordering) or dropping the float Conns.

### T4 — CLAUDE.md (in-repo, `connections/CLAUDE.md`)

under §Automated MR reviewer that names MR !70 as the precedent and

Pattern 9 calibration), with the explicit "type-system soundness on

## Deferred

- **Tooling enforcement.** A linter that scans for `*_total` /
  `*_relaxed` test renames and grep-asserts a matching type-level
  weakening exists is a longer-term option but premature now — the
  false-positive rate is hard to tune without empirical data from
  the next 2–3 sprints.
- **Plan-template change.** Forcing the §Review section to declare
  type-level relaxations alongside their corresponding declaration
  changes is a structural fix on the author side; lower-leverage
  than fixing the reviewer prompts. Revisit if author-side errors
  resurface.
- **CI gating on the bot.** Currently `allow_failure: true`. Keep
  it — turning the AI into a merge blocker would amplify the
  "ratifies framing" failure mode this sprint exists to mitigate.
- **Calibration smoke test against MR !70.** Manually re-running
  `claude_review.py` against MR !70's pre-merge diff to verify the
  prompt change binds is the obvious next step, but it requires
  either resurrecting the pre-fix branch or piping the diff in
  locally. Track as a follow-up; the trait-claim audit instruction
  is concrete enough that the smoke test is a confirmation, not a
  gate.
