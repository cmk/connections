# Plan 02 — `core::usize` rename + portable `core::isize` + kani doc fix

## Goal
Rename the merged `core::size` (`SIZE****`) family to `core::usize`
(`USZE****`) — impl unchanged — and add a **fully portable** `core::isize`
(`ISZE****`) family covering only the three rungs whose cast direction is
fixed across all pointer widths. Also fix the pre-existing kani `## Layout`
doc drift.

## Design decision — stay portable, defer the direction-flipping rungs

`usize`'s casts are portable for free (unsigned widening shares the `0`
bottom, so plain `TryFrom` is lawful on 16/32/64-bit — PR #16's shipped
design). `isize`'s casts are portable for free on exactly the rungs whose
**direction is fixed** across pointer widths:

| Rung   | Direction (all real targets)      | Macro              | Portable? |
|--------|-----------------------------------|--------------------|-----------|
| `→i8`  | always narrowing (`isize ≥ 16>8`) | `int_int_narrow!`  | yes       |
| `→i16` | narrow or iso, never widens       | `int_int_narrow!`  | yes       |
| `→i32` | **flips** (narrow/iso/**widen**)  | —                  | **no**    |
| `→i64` | **flips** (iso/**widen**)         | —                  | **no**    |
| `→i128`| always widening (`i128` widest)   | `ext_int!`         | yes       |

`→i32` / `→i64` flip direction with target width, so no single stock-macro
const covers them: `int_int_narrow!` truncates on the widening target,
`ext_int!` overflows computing `BELOW` on the iso/narrowing target. Making
them portable needs a bespoke cross-regime `Extended` hybrid; assuming a
64-bit target would sidestep that. **This PR ships neither — it omits
`ISZEI032`/`ISZEI064`**, which keeps every shipped const portable and bakes
in no 64-bit assumption, leaving the strategy for those two rungs open.

The `→i128` widening is the one rung needing an `Extended` source:
`isize::MIN` embeds *above* `i128::MIN`, leaving unreachable negatives, so a
plain `Conn<isize, i128>` breaks left-Galois at the bottom. `Extended<isize>`
absorbs them — exactly as `ext_int!(I064I128, i64, i128)` does.

### Spot checks
- `USZEU032.ceil(usize::MAX) == u32::MAX` (64-bit host); `USZEU032.upper(
  u32::MAX) == usize::MAX`.
- `ISZEI008.ceil(isize::MAX) == i8::MAX`; `ISZEI008.ceil(isize::MIN) ==
  i8::MIN`; `ISZEI008.upper(i8::MAX) == isize::MAX`.
- `ISZEI128.ceil(Extended::PosInf) == (isize::MAX as i128) + 1`;
  `ISZEI128.ceil(Extended::NegInf) == i128::MIN`.
