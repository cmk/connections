# Plan 13 — `conn::float` modularization + half-based f16/bf16 Conns

## Goal

Reshape `src/conn/float.rs` into the `conn::fixed.rs + conn::fixed/`
parent + subdirectory pattern, then add four direct precision-narrowing
Conns to half-precision (`F032F016`, `F032B016`, `F064F016`, `F064B016`)
via the `half` crate. Renames `ExtendedFloat::Finite` → `Extend` and
moves the existing `F064F032` into the new submodule layout.

## Context

Plan 11 (drop-Ple) shipped `pub const F064F032: Conn<ExtendedFloat<f64>,
ExtendedFloat<f32>>`, `impl Eq for ExtendedFloat<T>`, and the
`Eq + PartialOrd` lawful framework. With that foundation, the float
module is ready to grow:

- The half-precision deferral in plan 11 (f128/f16 native unstable) is
  unnecessary: the [`half`](https://docs.rs/half) crate provides stable
  software impls of IEEE binary16 and Google bfloat16 with const
  constructors and bit-faithful NaN handling. Adopting `half` lets us
  ship f16 / bf16 Conns now, on the pinned 1.85 toolchain.
- `Finite(NaN)` is a category error — NaN isn't finite. The variant
  should be named after its lattice-theoretic role (the *extension*
  slot between `Bot` and `Top`).
- Single-file `float.rs` was getting unwieldy; the `conn::fixed.rs +
  conn::fixed/` pattern (parent doc + per-tier submodules) maps
  cleanly onto float once we have multiple source tiers (f64, f32, …).

When native `f16`/`f128` eventually stabilize, `pub type F016 =
ExtendedFloat<half::f16>` swaps its inner type without changing public
Conn names — `F064F016` etc. all stay.

## Tasks

### T0 — `task: Add half crate dep`

Add `half = { version = "2", default-features = false }` to
`Cargo.toml`. Both half-precision types (`half::f16`, `half::bf16`) are
`#[repr(transparent)]` over `u16` with const constructors and RNE
conversions. MSRV 1.81 ≤ pinned 1.85.

### T1 — `refactor: Rename ExtendedFloat::Finite → Extend`

Pure rename. Touches the enum def + `impl_float_ext!` macro patterns
+ pattern matches in `decimal.rs` F064FD0N consts + arb strategies +
prose in lib.rs/design.md/lattice.rs. `Extended<T>::Finite` is left
alone — its wrapped values really are finite. Crate is unpublished;
internal-only churn.

### T2 — `task: Modularize conn::float into parent + subdirectory`

Mirror `conn::fixed.rs + conn::fixed/`. Move `F064F032` + ULP-walk
helpers + tests into new `src/conn/float/f64.rs`. Refactor walk
helpers to return `(f32, u32)` so iteration count is observable for
the new `ulp_steps_bounded` proptest. `shift32` becomes `pub(crate)`
so submodules can call it.

Per Q3 of the plan: lazy submodule file creation — `f16.rs` /
`b16.rs` placeholders are NOT created until they have content.

### T3 — `task: Extend impl_float_ext! to half types + F016/B016 aliases + shift16 family`

In `src/conn/float.rs` parent:
- `impl_float_ext!(half::f16);` `impl_float_ext!(half::bf16);` after
  the existing f32/f64 instantiations.
- `pub type F016 = ExtendedFloat<half::f16>;`
- `pub type B016 = ExtendedFloat<half::bf16>;` — both with
  software-emulated caveats per Q2.
- `pub(crate) fn signed16/unsigned16/clamp16_f16/clamp16_bf16/shift16_f16/shift16_bf16`
  alongside the existing 32-bit family. signed16 / unsigned16 are
  shared (both half types are u16-backed sign-magnitude); the clamp
  helpers differ only in ±∞ bit constants.
- Spot tests: NaN reflexivity for F016 and B016.

### T4 — `test: Add arb strategies for half-precision types`

In `src/property/arb.rs`:
- `arb_f16` / `arb_bf16` — boundary-heavy + bounded-uniform per the
  proptest_shrink_tuning memory. Boundary set: `{NAN, ±INFINITY, ±0,
  MIN_POSITIVE, MIN_POSITIVE_SUBNORMAL, MAX, MIN}`. Uniform slot is a
  bounded f32 range mapped through `from_f32` (RTNE).
- `extended_float_f16` / `extended_float_bf16` — 1:1:8 over Bot/Top/Extend.

### T5 — `feat: Add F032F016 (f32 → IEEE binary16)`

### End-to-end scenario

```rust
use connections::conn::float::f64::F064B016;
use connections::conn::float::ExtendedFloat::Extend;

let pi = Extend(std::f64::consts::PI);
let pi_up   = F064B016.ceil(pi);
let pi_down = F064B016.floor(pi);
assert!(F064B016.inner(pi_down) <= pi);
assert!(pi <= F064B016.inner(pi_up));

let huge = Extend(f64::MAX);
assert_eq!(F064B016.ceil(huge), Extend(half::bf16::INFINITY));
```

This is the canonical doctest mirrored into both `F064B016`'s rustdoc
and the README "f64 → bfloat16" section.

## Deferred

- **Cross-tier-into-decimal Conns** for the new precisions:
  `F016FD00..F016FD12`, `B016FD00..B016FD12`, `F032FD00..F032FD12`.
  These belong with the FD ladder in `src/conn/fixed/decimal.rs`. Each
  source tier is its own follow-up sprint.
- **Native `f16` / `f128` migration.** When the toolchain pin advances
  past stabilization, swap `pub type F016 = ExtendedFloat<half::f16>`
  → `ExtendedFloat<f16>` and add `F128`/`F128F064`/`F128F032` then. No
  public Conn-name changes.
- **`F016B016` / `B016F016`.** half-rs has no direct f16↔bf16
  conversion; both round-trip through f32. Composition
  (`compose!(F016F032_inv, F032B016)`) covers this when needed.
- **Pcompare / round / truncate / midpoint / median for f16, bf16.**
  Once Sprint B (`Pcompare`) lands for f32/f64, extend to half
  precisions in a tiny follow-up.
- **`Extended<T>::Finite → Extend`.** Per Q1 of the plan, only
  `ExtendedFloat` is renamed. `Extended<T>` keeps `Finite` because
  its wrapped values really are finite (no NaN to worry about).

## Review

### Design deviations from the plan

- **Plan number was 12, not 11.** The float-half plan-mode plan
  predicted plan number 11 because remove-ple was queued; remove-ple
  ended up titled "Plan 11" in `plan-2026-04-26-10.md`, so this plan
  bumps to 12 in `plan-2026-04-26-11.md`. MR iid bumped from the
  predicted 19 to 20 because !19 (sprint/time-restructure) was opened
  during this sprint's work.
- **`pub type F032 = ExtendedFloat<f32>` and `pub type F064 =
  ExtendedFloat<f64>` aliases added in T5** (not T3 as plan-mode plan
  suggested). They became necessary when `F032`/`F064` were
  referenced from per-source-tier submodules.
- **shift16 family marked `#[allow(dead_code)]`** when first added in
  T3, then the attribute was removed implicitly when the first
  consumer (T5: F032F016) landed. The early adds had test coverage
  exercising them so they weren't actually dead.

### `#[ignore]`d properties

None. Every property test in the new files runs.

### Recommendations

- A future sprint should consolidate the ULP-walk machinery into
  a `walk!` macro or a generic helper. The four-walk-helpers-per-Conn
  pattern is heavy boilerplate (≈ 80 lines / Conn × 5 Conns = 400
  lines of near-duplicated code).
- The "headline doctest" mirroring discipline now applies to two
  modules (time and float). Consider adding a `scripts/check_readme_mirror.sh`
  that diffs the README's mirrored block against the source-of-truth
  rustdoc and fails CI if they drift.
