# Plan 07 — `fixed`-crate-native binary Galois conns (i08/i16/i32/i64)

## Goal

Lift the `fixed` crate's `FixedI*<Frac>` lattice into 66 named `Conn`
constants under `conn::fixed::{i08,i16,i32,i64}`, each total and
satisfying the five Galois axioms for every input. Drive the test
suite through the public `property::laws::*` predicates.

## Tasks

### T1 — Add `doc/design.md` § "Triple-only properties and the role of injectivity"

Distinguish the five Galois axioms (which hold for every connection)
from the triple-only `floor(a) ≤ ceil(a)` "rounding sandwich" (which
requires injective `inner`). The new binary modules have a saturation
plateau on `inner`, so they fail `floor_le_ceil` at the boundary by
design — this section justifies that and pins the test contract:
predicates `conn_galois_l/r`, `conn_closure_l/r`, `conn_kernel_l/r`,
`conn_monotone_l/r`, `conn_idempotent` are universal; `conn_floor_le_ceil`
is asserted only for documented-injective Conns.

### T2 — Binary sub-modules + Ple impls + laws-driven tests

For each inner width `W ∈ {8, 16, 32, 64}`, add a sub-module
`conn::fixed::iWW` containing one `Conn<FixedIW<FineFrac>,
FixedIW<CoarseFrac>>` per ordered pair of Frac levels with `FineFrac >
CoarseFrac`.

Frac level subset per width:

| Width | Levels | Pair count |
|-------|--------|------------|
| i8    | `{U0, U1, U2, U3, U4, U6, U8}` | 21 |
| i16   | `{U0, U2, U4, U8, U12, U16}` | 15 |
| i32   | `{U0, U4, U8, U16, U24, U32}` | 15 |
| i64   | `{U0, U8, U16, U32, U48, U64}` | 15 |

Const naming: `F<dd>F<dd>` (zero-padded), e.g. `conn::fixed::i16::F08F04`.

Per-Conn implementation:

- `inner(c)` widens `c.to_bits()` through one wider integer
  (`i16/i32/i64/i128` for `i08/i16/i32/i64`), multiplies by
  `RATIO = 1 << (FineFrac − CoarseFrac)`, **saturates** the result to
  the Fine inner-bit range, and constructs the Fine value via
  `from_bits`.
- `ceil(f)` performs `bits.div_euclid(RATIO)` + remainder rounding on
  the widened Fine bit pattern. Boundary fixup: if `f.bits ==
  Fine_inner::MIN`, return `Coarse::MIN` (every `c` on the lower
  inner-saturation plateau has `inner(c) = Fine::MIN`, so
  `ceil(Fine::MIN)` must be the smallest such `c`).
- `floor(f)` mirrors `ceil` with the symmetric upper-boundary fixup
  `f.bits == Fine_inner::MAX ⇒ Coarse::MAX`.

Add blanket `Ple` impls in `src/conn/fixed.rs`:

```rust
impl<F: ::fixed::types::extra::LeEqU8>  Ple for ::fixed::FixedI8<F>  { … }
impl<F: ::fixed::types::extra::LeEqU16> Ple for ::fixed::FixedI16<F> { … }
impl<F: ::fixed::types::extra::LeEqU32> Ple for ::fixed::FixedI32<F> { … }
impl<F: ::fixed::types::extra::LeEqU64> Ple for ::fixed::FixedI64<F> { … }
```

Each test mod imports `use crate::property::laws;` and the
`props_for_pair!` macro emits one `proptest!` per law that calls the
predicate. No inline assertion patterns — every property goes through
the public API.

## Verification

### Properties (must pass)

Per `(W, FineFrac, CoarseFrac)` triple, **nine** universally-
quantified laws over the **full** primitive bit range (`any::<i_W>()`).
Every property is a call to a `crate::property::laws::*` predicate.

| Property | Module | Predicate |
|----------|--------|-----------|
| `galois_l` | `conn::fixed::iWW` | `laws::conn_galois_l` |
| `galois_r` | `conn::fixed::iWW` | `laws::conn_galois_r` |
| `monotone_l` | `conn::fixed::iWW` | `laws::conn_monotone_l` |
| `monotone_r` | `conn::fixed::iWW` | `laws::conn_monotone_r` |
| `closure_l` | `conn::fixed::iWW` | `laws::conn_closure_l` |
| `closure_r` | `conn::fixed::iWW` | `laws::conn_closure_r` |
| `kernel_l` | `conn::fixed::iWW` | `laws::conn_kernel_l` |
| `kernel_r` | `conn::fixed::iWW` | `laws::conn_kernel_r` |
| `idempotent` | `conn::fixed::iWW` | `laws::conn_idempotent` |

66 conns × 9 properties = **594 generated proptests**. i64 caps
proptest cases to 64 (`ProptestConfig::with_cases(64)`) to bound CI
time.

`laws::conn_floor_le_ceil` is intentionally NOT asserted: it requires
injective `inner`, which fails at the saturation plateau (see
`doc/design.md` § Triple-only properties).

### Spot checks

Per inner-width module, hand-computed boundary checks:

| Test | Asserts |
|------|---------|
| `iWW::tests::spot_*_on_grid` | grid-aligned values round-trip exactly |
| `iWW::tests::spot_*_off_grid_{positive,negative}` | bit-math rounding direction is `−∞` for `floor`, `+∞` for `ceil` |
| `iWW::tests::spot_*_saturation_boundary` (i16) | `Fine::MAX` ceils up, floor uses fixup; symmetric on `Fine::MIN` |
| `iWW::tests::spot_*_degenerate` | high-SHIFT pairs (e.g. `F16F00` for i16) have only `Coarse(0)` round-trip; ±1 saturates inner |
| `iWW::tests::spot_boundary_fixups` | `ceil(Fine::MIN) = Coarse::MIN`; `floor(Fine::MAX) = Coarse::MAX` |

## Deferred

- **Family B (width widening at fixed Frac)** — `Conn<FixedI8<F>,
  FixedI16<F>>` etc. Needs `Extended<Narrow>` on the Coarse side for
  a full lawful triple. Pattern from `int.rs` copies cleanly when
  needed.
- **Family C (sign change at fixed width+frac)** — `Conn<FixedI16<F>,
  FixedU16<F>>`. Same `Extended<U…>` machinery as `int.rs::U08I16`.
- **Family D (Fixed → primitive int)** — degenerate case of `F<frac>F00`
  plus a trivial newtype unwrap.
- **Family E (Fixed ↔ float)** — substantive next sprint.
- **Family F (cross-radix decimal ↔ binary)** — defer indefinitely.
- **`I128` / `U128`** — out of source-of-truth Haskell port.
- **Unsigned `FixedU*` mirror** (`conn::fixed::u08`/`u16`/`u32`/`u64`)
  — macro generalises trivially over signedness; bundle into a
  follow-up sprint.
- **Decimal-module totality alignment** — `conn::fixed::decimal::inner`
  still uses naked `i64` multiplication that panics in debug for
  out-of-range Coarse input. Bringing it to the new totality-everywhere
  standard is its own follow-up.
