`note-2026-04-27-01.md` (the 0.1.0 publish-readiness audit) by
shipping 45 new `pub const` Conns across four families:

- §1 I→I narrowing (10 Conns)
- §2 U→U narrowing (10 Conns)

## Locked design decisions

### D1. Per-primitive `conn::std::<dest>` layout

Every Conn lives in the module of its right-side (destination)
primitive. Existing widening Conns are redistributed; new
narrowing/non-widening Conns join their destination's submodule.
This makes "what Conns target u32?" a one-look question
(`conn::std::u32`) and pre-empts the W1 audit recommendation
(`conn::primitive::{int,uint}`) by going one level deeper.

### D2. Single-sided shape, with right-Galois for §3

§1, §2, §4 are left-Galois single-sided (`Conn::new_left`). §3 is
right-Galois single-sided (`Conn::new_right` — added in T1).

The direction is forced by where the saturation plateau lives:

- §1/§2/§4: source-side plateau on `ceil` (e.g. `i32` values
  outside `i8`'s range collapse to `i8::MIN`/`i8::MAX`). The
  source-side fiber is compatible with `galois_l`'s bi-implication.
- §3: target-side plateau on `inner` (`i_M`'s negative half clips
  to `0_u_N`). The target-side fiber breaks `galois_l` but
  satisfies `galois_r`.

### D3. FINE_MAX boundary fixup in `inner` for §1/§2/§4

The same fixup pattern as `conn::fixed::u<width>` (MR !20). At
the source-side high plateau, multiple sources collapse onto
`Coarse::MAX` via `ceil`; for `galois_l` to hold, `inner(MAX)`
must reach `Source::MAX` so `a ≤ inner(b)` holds for every
plateau `a`. The low end works without fixup because
`Coarse::MIN as Source` lands strictly above the low plateau.

§3 needs no fixup: `inner`'s image is exactly `[0, i_M::MAX as
u_N]`, and `floor`'s collapse at the high plateau aligns the
target-side fiber with the source identity at `i_M::MAX`.

### D4. Integration tests for the new layout

Per-primitive proptest batteries in
`tests/conn_std_<primitive>_galois.rs` (10 files), mirroring the
precedent set by `tests/conn_fixed_u<width>_galois.rs` in MR !20.
Existing widening Conns' proptests are migrated alongside the
move; spot tests stay inline. Indirectly closes audit W2 for the
std-int family.

## Review

- Galois lawfulness for §3 (right-Galois single-sided) was the
  late-breaking design clarification — caught in the planning Q&A
  before any code landed. The corrected derivation is encoded in
  the test battery via `single_sided_right_props!`; the original
  spec doc's "single-sided + clip-negs-to-0" prescription for §3
  passes with the right-Galois battery (and fails the left-Galois
  one at every `(0_u_M, neg_i_N)` corner).
- Test count: lib went from 1781 (pre-restructure) → 1584. Net
  difference is the proptest batteries that moved out of `mod
  tests` blocks into integration files (210+ tests relocated).
  Adding the new 45 Conns brought the total integration test
  count to ~1490.
- No `#[ignore]`d properties.
