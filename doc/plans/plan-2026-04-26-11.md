# Plan 12 — `conn::time` per-domain restructure

(Sprint Time-1A — pure structural cleanup, no semantic changes.)

## Context

`src/conn/time.rs` is currently a single 1144-line file holding five
`Conn` constants, four `*_preorder` proptest mods, and five
Conn-specific test mods. The user's brief: "should be structured
similar to `conn/fixed`". `conn::fixed` keeps a slim parent
(`fixed.rs` — module docs + `pub mod` decls) and pushes each
implementation family into its own sub-module
(`fixed/decimal.rs`, `fixed/i08.rs`, …, `fixed/i128.rs`). Time
needs the same treatment.

A sibling concern — `src/conn/mod.rs` should have been
`src/conn.rs` (modern Rust 2018+ flat-file form) — was already
addressed upstream in MR !18 along with the Ple→Eq+PartialOrd
shift. This MR therefore contains only the time-module split.

## Sub-module split

Group by source domain, mirroring `conn::fixed`'s split-by-backing-
primitive:

| File                          | Contents                                  |
|-------------------------------|-------------------------------------------|
| `src/conn/time.rs`            | parent: docs, `pub mod` decls, `pub use` re-exports |
| `src/conn/time/date.rs`       | `DATEJDAY` + Date preorder tests         |
| `src/conn/time/clock.rs`      | `TIMENANO` + `TIMESECS` + Time preorder  |
| `src/conn/time/duration.rs`   | `DURNSECS` + Duration preorder tests     |
| `src/conn/time/datetime.rs`   | `PDTMDATE` + PrimitiveDateTime preorder  |

The parent file re-exports each Conn at the family-flat path
(`pub use date::DATEJDAY;` etc.) so callers continue to write
`connections::conn::time::DURNSECS` without learning the
sub-module layout.

After MR !18's Ple removal, the parent file no longer carries any
trait impls — `Date`, `Time`, `Duration`, and `PrimitiveDateTime`
all derive `Eq + PartialOrd` upstream and flow through the law
machinery directly. The parent is therefore a slim docs + glue
file (~85 lines).

## Tasks

### T1 — Split `src/conn/time.rs`

Mechanical extraction of each Conn + its tests into a sub-module.
Each sub-module's `#[cfg(test)] mod tests` now contains:

- One `*_preorder` proptest mod for the type that sub-module's Conn
  uses (e.g. `clock.rs` owns the `Time` preorder tests). Five
  predicates each: `lattice_reflexive`, `lattice_transitive`,
  `lattice_antisymmetric` (2-arg form, post-MR !18),
  `lattice_bot`, `lattice_top`.
- Spot checks for each Conn defined in the file.
- The full Galois law battery for each Conn.

The single big `mod tests` wrapper in the original file is gone —
each sub-module has its own. The five inner `mod {datejday,
timenano, timesecs, durnsecs, pdtmdate}` wrappers are dropped for
single-Conn sub-modules and retained only in `clock.rs` (which
hosts both `TIMENANO` and `TIMESECS` and needs the disambiguation).

### T2 — README Layout tree

Update `## Layout` to show `conn/time/` as a directory (not
`conn/time.rs`). Pure mechanical text edit.

## Review

- Pure refactor. Reviewer's job is to confirm "no semantic drift"
  rather than to vet new behavior. The diff shows ~1144 deletions
  in `src/conn/time.rs` and ~1199 additions across the four new
  `conn/time/*.rs` files; the residual ~85 lines in `time.rs` are
  the module docs + `pub mod` / `pub use` glue.
- 1626 lib tests + 18 doctests — same totals as pre-restructure.
  Test names change (e.g. `conn::time::tests::datejday::galois_l`
  becomes `conn::time::date::tests::galois_l`) but the count and
  coverage are identical.
- The new `pub use` re-exports preserve the public path
  `connections::conn::time::DURNSECS` etc. — no caller migration
  needed.
- Rebase note: the original plan listed a `Rename src/conn/mod.rs
  → src/conn.rs` task (T1) and a Ple-removal forward-port. Both
  shipped in MR !18 ("Drop Ple; adopt Eq + PartialOrd") while this
  branch was open; this MR's content is therefore reduced to the
  pure split.
