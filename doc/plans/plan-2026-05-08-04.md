# Plan 2026-05-08-04 — Eliminate `Conn<Extended<A>, Extended<B>>` antipattern

## Goal

Strip the `Extended<…> => Extended<…>` shape from the three Conns
that ship it by hand (`U032CHAR`, `SDURU064`, `SDURU128`), restructure
their up-stream producers (`F064SDUR`, `F032SDUR`) so they no longer
synthesize `Extended<StdDuration>`, and add a pre-commit guardrail
that prevents the antipattern from re-entering the source tree
except through the `lift_l!`/`lift_r!`/`compose_l!`/`compose_r!`
derivation paths.

The design choice for option (2): **strip Extended from both sides
of every offending Conn, accept synthetic-top promotion in `inner`
to preserve Galois L, and demote the two former `conn_k!` Conns
(`U032CHAR`, `SDURU064`) to `conn_l!`** — the full triple is not
recoverable once the source is plain (Galois R fails at the
top boundary by saturation).

## Dependency Graph

```
T1a (U032CHAR) ──┐
                 ├──► T2 (pre-commit guardrail)
T1b (SDUR fam) ──┘
```

T1a and T1b are independent (different host modules; no shared
constants). T2 depends on both because its allowlist tests must
verify a clean tree.

## Tasks

### T1a — Strip `U032CHAR` to `Conn<u32, char>` (ConnL)

**Problem.** `pub U032CHAR : Extended<u32> => Extended<char>` carries
synthetic `NegInf`/`PosInf` on the source side that pass through
unchanged to the target. The source `Extended` is dead weight: the
Conn has no upstream producer that emits `Extended<u32>` (verified
by grep — `U032CHAR` is orphaned).

**Solution.** Restripe both sides:

```rust
pub U032CHAR : u32 => char {
    ceil:  ceil_u32_char,    // saturating; surrogate gap → '\u{E000}'; u > 0x10FFFF → '\u{10FFFF}'
    inner: inner_char_u32,   // synthetic-promote: '\u{10FFFF}' → u32::MAX; otherwise c as u32
}
```

- ConnL only — Galois R fails at `a = u32::MAX, b = '\u{10FFFF}'`
  with the synthetic-promoted `inner` (the `floor` adjoint can't
  satisfy both directions at the top). Drop `floor_u32_char` and
  the `floor` test cases.
- `inner('\u{10FFFF}')` becomes `u32::MAX` rather than `0x10FFFF`.
  Document this as the synthetic-top arm.
- Keep the surrogate-gap and overflow handling in `ceil`.

**Tests.** Update `src/char.rs`:
- Replace `Extended<u32>` / `Extended<char>` references in spot-check
  tests with bare `u32` / `char`.
- Drop `surrogate_gap_floor`, `beyond_max_floor`, `extremes_pass_through`
  (no synthetic markers remain on either side).
- Drop the `floor_le_ceil` proptest (ConnK-only law).
- Add a `synthetic_top` test pinning `inner('\u{10FFFF}') == u32::MAX`.
- `law_battery!` switches from `arb_extended_u32` / `arb_extended_char`
  to plain `any::<u32>()` / `any::<char>()`.

**API surface.** `U032CHAR` now `Conn<u32, char, L>` (was `Conn<Extended<u32>, Extended<char>>`).
Doctests rewritten to drop the `Extended::Finite(_)` wrapping on inputs.
Module doc updated: ladder note already references `compose_l!` chaining.

### T1b — Restructure SDUR family

Four Conns shift in lockstep so composition still type-checks:

| Conn        | Old shape                                       | New shape                       | Flavor change |
|-------------|-------------------------------------------------|---------------------------------|---------------|
| `F064SDUR`  | `Conn<F064, Extended<StdDuration>, L>`          | `Conn<F064, StdDuration, L>`    | none (still ConnL) |
| `F032SDUR`  | `Conn<F032, Extended<StdDuration>, L>`          | `Conn<F032, StdDuration, L>`    | none |
| `SDURU064`  | `Conn<Extended<StdDuration>, Extended<u64>>`    | `Conn<StdDuration, u64, L>`     | **ConnK → ConnL** (drop `floor`) |
| `SDURU128`  | `Conn<Extended<StdDuration>, Extended<u128>, L>`| `Conn<StdDuration, u128, L>`    | none (still ConnL) |

**Common design — synthetic-top promotion in `inner`.** Because the
target now lacks a synthetic `PosInf`, the `inner` adjoint must map
the maximum element of the rung to the source's synthetic top to
preserve Galois L:

- `F???SDUR.inner(StdDuration::MAX) = ExtendedFloat::Top` (was `Extend(MAX.as_secs_f??())`)
- `SDURU064.inner(u64::MAX) = StdDuration::MAX` (was `from_secs(u64::MAX)`, which lacks the subsec part)
- `SDURU128.inner(n > MAX.as_nanos()) = StdDuration::MAX` (already saturating; preserved)

`inner` is therefore lossy at the top boundary in each case. This is
the price of a clean codomain. The Galois L kernel/closure laws still
hold across the full domain.

**Common design — saturation in `ceil`.** Without `Extended::PosInf`
on the target, ceil saturates at the rung's max value:

- `F???SDUR.ceil(Top) = StdDuration::MAX`; `ceil(NaN) = StdDuration::MAX`; `ceil(POS_INFINITY) = StdDuration::MAX`; `ceil(Bot) = StdDuration::ZERO`; `ceil(NEG_INFINITY) = StdDuration::ZERO`.
- `SDURU064.ceil(d)` where `d.as_secs() == u64::MAX && d.subsec_nanos() > 0 → u64::MAX` (saturating add → was PosInf).
- `SDURU128.ceil` already exact across the entire `StdDuration` domain.

**Tests.** `src/time/duration.rs`:
- Replace `Extended<StdDuration>` / `Extended<u64>` / `Extended<u128>` in
  spot checks with bare types.
- Drop the `synthetic_arms` tests for the SDUR family (no `NegInf`/`PosInf`
  on either side).
- Add `synthetic_top` tests for each: `inner(MAX) == synthetic top of source`.
- `law_battery!` switches `arb_extended_std_duration` → `arb_std_duration`,
  `arb_extended_u64` → `any::<u64>()`, `arb_extended_u128` → `any::<u128>()`,
  and the float-side strategies switch from `arb_extended_std_duration_bounded_f??`
  → `arb_std_duration_bounded_f??`.
- Drop the `roundtrip_floor` and ConnR proptests for `SDURU064` (was ConnK,
  now ConnL).
- New strategies in `src/prop/arb.rs`: `arb_std_duration`,
  `arb_std_duration_bounded_f64`, `arb_std_duration_bounded_f32`. Old
  `arb_extended_std_duration*` stay (still used by tests that traffic
  in `Extended<StdDuration>` rungs internally — none after this
  sprint, so they go away too).

**Kani.** `src/kani_proofs/time_pure.rs` references `SDURU064` and
`SDURU128`. Update the proofs to operate on bare `StdDuration`/`u64`/`u128`
rungs. `prove_l!` / `prove_k!` macros may need light adaptation;

**Module docs.** `src/time.rs` and `src/time/duration.rs` table rows
referencing `Conn<Extended<StdDuration>, …>` get rewritten.

### T2 — Pre-commit guardrail (`scripts/check_no_ext_ext_conn.sh`)

**Problem.** Once the antipattern is gone we want to keep it gone,
without having to remember the rule on every Conn-add.

**Solution.** A `bash + ripgrep` scan over staged `.rs` files that
flags `Extended<…> => Extended<…>` inside `conn_l!`/`conn_k!`/`conn_r!`
macro bodies. Matches are blocked unless on the allowlist:

- File is `src/extended.rs` (the `lift_*!` machinery's home).
- Line is preceded by a literal `// allow(ext-ext-conn): <reason>`
  comment on the previous line.

The `lift_l!` / `lift_r!` / `compose_l!` / `compose_r!` macro outputs
DON'T match the pattern (their bodies use `Conn::new_l(...)` /
`Conn::new_r(...)` directly, not the `=>`-arrow declaration syntax).
So they need no special-case in the regex.

**Implementation.** `scripts/check_no_ext_ext_conn.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail
# Scan staged .rs files for `Extended<…> => Extended<…>` inside
# conn_l!/conn_k!/conn_r! macro bodies. Allowlist:
#   - src/extended.rs (home of lift_l!/lift_r!)
#   - lines preceded by `// allow(ext-ext-conn): <reason>`.

shopt -s nullglob
exit_status=0

while IFS= read -r file; do
    [[ "$file" == "src/extended.rs" ]] && continue
    [[ "$file" == *.rs ]] || continue

    # Multi-line aware grep: search for `Extended<...> => Extended<...>` on a single line.
    while IFS=: read -r lineno _; do
        prev=$(( lineno - 1 ))
        prev_line=$(sed -n "${prev}p" "$file")
        if [[ "$prev_line" =~ //[[:space:]]*allow\(ext-ext-conn\): ]]; then
            continue
        fi
        echo "error: $file:$lineno — Extended<…> => Extended<…> antipattern"
        echo "  use lift_l!/lift_r!/compose_l!/compose_r! to derive, or"
        echo "  add '// allow(ext-ext-conn): <reason>' on the previous line."
        exit_status=1
    done < <(grep -nE 'Extended<[^>]*>[[:space:]]*=>[[:space:]]*Extended<' "$file" || true)
done < <(git diff --cached --name-only --diff-filter=ACM)

exit $exit_status
```

**Wiring.** Append to `.githooks/pre-commit` AFTER `check-pii.sh`,
BEFORE `cargo test`:

```bash
scripts/check_no_ext_ext_conn.sh || exit 1
```

Layer 1 (`.claude/settings.json` PreToolUse) ALSO gets the hook so
the agent flow catches violations before staging.

**Tests for the script.** A small `tests/` shell test that:
- Asserts the script exits 0 on the current tree (post-T1 strip).
- Asserts it exits 1 when injected with a mock `conn_l! { ... Extended<X> => Extended<Y> { ... } }` snippet.
- Asserts the `// allow(ext-ext-conn):` escape hatch suppresses the error.

## Deferred

- **Pos-only Extended type.** A "pos-extended" type (`E<T>` with `Top`
  but no `Bot`) would let `SDURU064` regain Galois R at the top by
  keeping target Extended without source Extended. Out of scope —
  larger design change, low payoff.
- **Free-function `ceil_to_secs(d) -> u64`** for callers who want the
  saturating-ceil-to-u64 convenience that the demoted `SDURU064` no
  longer exposes via `floor`. Not added; if a downstream caller
  needs it we'll surface it then.
- **Renaming `inner` to `upper` / `lower`.** Plan 32 already settled
  on `inner` for both Galois directions; not revisiting.
