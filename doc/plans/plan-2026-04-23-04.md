# Plan 05 — File tree reorg and FloatExt → ExtendedFloat rename

## Goal

Bring `src/` in line with `doc/design.md` §"Module structure" — move
domain-tier files under `src/conn/`, fold `float_ext.rs` into
`src/conn/float.rs`, rename `FloatExt` → `ExtendedFloat` throughout,
collapse `order.rs` into a new `src/lattice.rs`, and create empty
stubs for `src/property.rs` (populated in Sprint C).

## Context

Sprint 2 of the three-sprint tidy described in the approved meta-plan
(`~/.claude/plans/this-library-is-accumulating-deep-wilkes.md`).
Mechanical rename + `mod` restructuring; no behavior change, test
counts held constant.

Target tree (per meta-plan with the user's "fold float_ext.rs into
float.rs, move everything tier-y under conn/" decision):

```
src/
├── lib.rs
├── conn.rs
├── extended.rs
├── lattice.rs          (new; absorbs Ple from order.rs)

    ├── word.rs         (empty stub; future sprint)
    ├── float.rs        (moved from top-level; absorbs float_ext.rs)
    ├── fixed.rs        (moved from top-level)
```

## Tasks

### T1 — Create `src/conn/` and move tier files

```
git mv src/float.rs  src/conn/float.rs
git mv src/fixed.rs  src/conn/fixed.rs
git mv src/sample.rs src/conn/sample.rs
```

`git mv` preserves history for blame. Do not edit content in this
task — only move. Intra-file imports (e.g. `use crate::conn::*`)
stay referencing crate-root items until T6 fixes up `lib.rs`.

### T2 — Fold `float_ext.rs` into `conn/float.rs`

Append the full contents of `src/float_ext.rs` (type definition,
`PartialOrd` impl, unit tests, any `pub use`) onto `src/conn/float.rs`.
Preserve the section ordering so the type definition comes before
the connections that reference it. Delete `src/float_ext.rs`.

The `git mv` pattern does not fit here (the fold merges two files
into one), so this is a straight `Edit` / `Write` + `git rm`. The
commit message should note history-preservation is lost for
`float_ext.rs` content; history for `float.rs` remains through
`git log --follow`.

### T3 — Rename `FloatExt` → `ExtendedFloat`

Crate-wide find-and-replace across `src/` and tests:

- `FloatExt` → `ExtendedFloat`
- `float_ext` identifier (if any) in module paths — already gone
  after T2.
- `Conn<FloatExt<f64>, FloatExt<f32>>` → `Conn<ExtendedFloat<f64>, ExtendedFloat<f32>>`

Use `rg 'FloatExt' src/` to locate, then `Edit --replace_all` on
each file. Verify `grep -r FloatExt src/` returns zero matches post-
task.

### T4 — Collapse `order.rs` into new `src/lattice.rs`

- Create `src/lattice.rs`.
- Move the `Ple` trait and its impls verbatim from `src/order.rs`
  into `src/lattice.rs`.
- Add stub signatures for `Join`, `Meet`, `Heyting`, `Coheyting`,
  `Symmetric`, `Boolean` traits matching the shapes in
  `doc/design.md` §"Lattice hierarchy". Implementation bodies are
  deferred — the signatures just need to compile. A doc-comment on
  each explains "implementations deferred; see Plan NN".
- Delete `src/order.rs`.
- Update every `use crate::order::Ple` → `use crate::lattice::Ple`.

### T5 — Create empty `src/property.rs` and stub `conn/int.rs`, `conn/word.rs`

Each file is a module declaration with a doc comment explaining the
intended population timeline. No code; they must compile as empty
modules.

`src/property.rs`:
```rust
//! Shared proptest strategies and property helpers.
//!
//! Populated by Plan 06 (proptest audit). Empty stub for now so
//! `pub mod property;` in `lib.rs` compiles.
```

`src/conn/int.rs` / `src/conn/word.rs`:
```rust
//! Integer ↔ integer connections.    // or word ↔ word
//!
//! Empty stub; connections to be added in a future sprint.
```

### T6 — Update `src/lib.rs`, `doc/design.md`, and verify build

`src/lib.rs` module declarations update to:

```rust
#![forbid(unsafe_code)]

pub mod conn;
pub mod extended;
pub mod lattice;
pub mod property;
```

Then `src/conn.rs` becomes the `conn` module with its own submodule
decls:

```rust
// ...existing Conn type + combinators...

pub mod int;
pub mod word;
pub mod float;
pub mod fixed;
pub mod sample;
```

Note: `src/conn.rs` AND `src/conn/` — Rust 2018+ supports this
combined style (the file defines the `conn` module and the directory
holds its submodules). This is idiomatic and does not require a
`src/conn/mod.rs`.

Update `doc/design.md` §"Module structure" to reflect the real tree.

Run full build+test gates:

```
cargo build
cargo test --workspace
cargo clippy --all-targets -- -D warnings
```

All must be green at the same 349 tests pass count as main.

## Review

### Deviations from the plan

- **Test count:** plan said "349 tests held constant"; actual count
  is **377** on both main and this branch. Main advanced between
  writing the meta-plan and starting Sprint B (Sprint D landed mid-
  flight and added tests). Invariant held — same count on main and
  branch, no regressions.
- **Lib.rs doc-link overhaul:** Plan called for mechanical rename
  only. Actual change had to also rewrite every module doc link in
  lib.rs (14 links: `fixed::*`, `float_ext::*`, `sample::*`) to the
  new `conn::*` prefix since there are no crate-root re-exports.
  No API surface change since the crate is pre-1.0 and the new
  paths are what downstream users will call.
- **Missed scope item:** plan T1 said "do not edit content in this
  task — only move." That held literally for `git mv`, but the import
  fixups in `conn/fixed.rs` (`crate::float_ext::FloatExt` →
  `crate::conn::float::FloatExt`) and `conn/sample.rs`
  (`crate::fixed::Pico` → `crate::conn::fixed::Pico`) had to land in
  the same commit as T1+T2 to keep `cargo test` green. CLAUDE.md's
  "every commit green" constraint dominated the plan's fine-grained
  task split.
