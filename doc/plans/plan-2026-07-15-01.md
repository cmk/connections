## T1 тАФ Three-surface export prelude + view-fn cleanup (delivered)

**What shipped.** `pub mod prelude` (`src/lib.rs`) is now three analogous
polarity surfaces, each a curated re-export bundle with the same shape
(capability trait + kind marker + `conn_*!` / `compose_*!` / `lift_*!`
macros), brought into scope together *and* independently importable:

| surface | trait | marker | free fns | macros |
|---------|-------|--------|----------|--------|
| `prelude::left` | `ConnL` | `L` | тАФ | `conn_l!` `compose_l!` `lift_l!` |
| `prelude::right` | `ConnR` | `R` | тАФ | `conn_r!` `compose_r!` `lift_r!` |
| `prelude::bilateral` | `ConnK` | тАФ | `round` `truncate` `median` `interval` (+ `1`/`2`) | `conn_k!` `compose_k!` `lift_k!` |

A shared, polarity-agnostic core sits at the prelude root (`Conn`,
`Extended`, `compose!`, `iso!`). `use connections::prelude::*` unions all
three surfaces + the core (via `pub use left::*` / `right::*` /
`bilateral::*`); `use connections::prelude::left::*` (etc.) brings a
single polarity. Single-polarity globs do **not** pull in the shared core
тАФ that lives at the root, not in the submodules.

**Also delivered:**

- **Deleted the free `view_l` / `view_r` functions** (`src/conn.rs`). The
  "retained for source back-compat and for the `&Conn`-receiver call
  sites" comment was theatre тАФ a prior agent avoided the call-site
  rewrites and wrote a justification. `t.view_l()` / `t.view_r()` (the
  `ConnL` / `ConnR` trait methods) do the identical thing. Rewrote ~22
  internal call sites (the `conn.rs` two-sided helpers and every
  `prop/conn.rs` law predicate), the `filter_l` / `filter_r` doctests,
  the `prop.rs` usage examples, stale doc prose in `conn.rs` /
  `extended.rs`, and the free-fn call sites in
  `tests/{fixed/nz_iso,trait_method_api,macros_feature_smoke}.rs`.
- Removed `view_l` / `view_r` from the `left` / `right` surfaces (those
  surfaces have no free fns now).

**Diagnosis note.** An independent audit confirmed **no method is
mis-scoped** between `ConnL` / `ConnR` / `ConnK` тАФ the trait split is
correct. The original symptom ("ConnK scope is messed up") traced to an
intermediate prelude that re-exported modules instead of traits, dropping
the trait methods from a `prelude::*` glob; the three-surface rewrite
supersedes it.

**Verification (green on `305ef36`).** Full `cargo test` (2664 + 1664 тАж
proptest cases across suites), doctests (81 passed, 2 ignored),
`clippy --all-targets -- -D warnings`, `cargo doc -D warnings`,

## Tasks (pending)

### T2 тАФ `#[inline]` on generated accessors; `macros` default (delivered)
**Premise correction.** The plan originally claimed the codegen macros
emit `swap_l` / `swap_r` without `#[inline]`. That was wrong for the
current code: the macro-emitted projection/trait methods
(`conn_l!` / `conn_r!` / `conn_k!` / `iso!` in `conn.rs`, the
`nz_*_ext!` families in `core.rs`, the `Extended` view/swap macro in
`extended.rs`) already carry `#[inline]`. The uneven attr counts in the
old premise counted *all* fns (tests, strategy fns, fn-pointer bodies),
not the missing-inline set.

**Delivered (inline).** Marked the public, non-generic forwarders
`#[inline]` so downstream crates can inline them across the crate
boundary тАФ without it a monomorphic `pub fn` like `N5::<f64>::sqrt`
won't inline into a consumer:
- `float.rs` (43): the `N5` arithmetic / assign / `From` / transcendental
  ops and the `N5` `PartialEq` / `PartialOrd`.
- `conn.rs` (7): the inherent `Conn` const fns `new_l` / `swap_l` /
  `new_r` / `swap_r` / `identity`, plus `Clone` / `PartialEq`.
- `extended.rs` (4): `Extended::fold` / `fold_with` and the `PartialOrd`
  / `Ord` impls.

**Excluded (deliberate).** The `fn ceil` / `fn inner` / `fn floor`
emitted inside `const $NAME: Conn = { тАж }` blocks (`core.rs`,
`fixed.rs`, and the `float_ext_int!` / `float_fixed!` families) are only
ever taken as `fn` pointers, so `#[inline]` is a no-op there тАФ left
unmarked. `Debug::fmt` (not a hot path) and the bodyless trait-method
declarations left unmarked.

**Delivered (default).** Added `default = ["macros"]` to `[features]`.
All `#[macro_export]` macros now land in the `connections::` root
namespace for every consumer, and the `macros` module plus the prelude
macro re-exports become unconditional. Acceptable at 0.1.0; narrowable
in 0.2 without breaking a 1.0 promise.

**Verification.** `cargo build` / `clippy --all-targets -- -D warnings`
/ `cargo test --doc` (109 pass / 2 ignored) green on the full feature
set; default-feature build + doctests (54 pass / 0 ignored) green;
`cargo tree -e features` confirms `macros` resolves by default.

**Surface.** `[features] default = ["macros"]`; 54 `#[inline]`
attributes (behavior-neutral).

## Review
- **T1 delivered vs planned.** The plan's T1 was "restore the prelude's
  trait re-exports." The delivered work went further: three uniform
  polarity surfaces (`left` / `right` / `bilateral`) + deletion of the
  `view_l` / `view_r` forwarders. Surface naming: `bilateral` (not
  `two_sided`) for the `ConnK` group.
- **Drift caught / fixed.** The "retained for back-compat" comment on the
  free view fns was theatre (an agent avoided ~22 call-site rewrites and
  wrote a justification) тАФ removed with the fns. Stale prelude doc
  comment rewritten. Redundant / broken intra-doc links introduced by the
  re-export churn fixed (`redundant_explicit_links`,
  `broken_intra_doc_links`).
- **T2 premise was wrong (corrected in-place).** The plan asserted the
  codegen macros emit `swap_l` / `swap_r` without `#[inline]`. False for
  the current tree тАФ the macro-emitted projection/trait methods already
  carry it. The real missing-inline set was the inherent `Conn` const
  fns and the hand-written `N5` / `Extended` impls. Rationale for
  inlining shifted from "no-op-in-const" hand-waving to the real reason:
  cross-crate inlining of public *non-generic* forwarders. See the T2
  section.
- **T2 deliberate exclusion.** The `fn ceil` / `fn inner` / `fn floor`
  emitted inside `const $NAME: Conn = { тАж }` blocks are only taken as
  `fn` pointers; `#[inline]` cannot affect an indirect call, so they were
  left unmarked (not an oversight). `core.rs` and `fixed.rs` therefore
  got zero `#[inline]` additions.
- **T4 `#[ignore]` justifications.** `EXAMPLES.md:308` тАФ the `f16` path
  is nightly-gated (`#![feature(f16)]`), so the fence cannot compile on
  the stable doctest run; keep `ignore` (the sole remaining ignore).
  `EXAMPLES.md:75` тАФ the `conn_l!` syntax fragment was ignored to keep
  `EXAMPLES.md` independent of the `macros` feature; **now enabled** as a
  real doctest, since T2 made `macros` default.
