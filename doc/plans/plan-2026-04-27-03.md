## Module organization rule (NEW ŌĆö codified in CLAUDE.md by T7a)

**One submodule of `conn::` per dependency crate.** Each Conn lives
in the module of its **winning side**, where the precedence order
is:

1. **Specific > general.** The Conn lives in the module of the
   more-specific endpoint. Specificity is roughly: domain types
   (`Duration`, `Date`, `OffsetDateTime`, `FixedI8`, `f16`, `bf16`)
   > generic numeric wrappers (`Extended<T>`, `ExtendedFloat<T>`,
   `FD<N>`) > std primitives. Subsumes "crate-deps > std".
2. **Right > left** (tie-breaker). Among Conns where both
   endpoints sit at the same specificity tier, the **right-hand
   side** wins (the coarser / smaller-resolution side per CLAUDE.md
   ┬¦Conn-name format).

Module structure:

```
conn::cast              # Conn-typed accessors / lifters (stays)
conn::std::*            # types from std (i8/ŌĆ”/i128, u8/ŌĆ”/u128, f32, f64)
conn::std::i64::decimal # custom decimal-SI ladder (i64-backed; nested
                        # under i64 since FD types are i64 newtypes
                        # and "i64 + decimal scaling" is a single
                        # semantic family)
conn::fixed::*          # types from `fixed` crate (FixedI8<U>, FixedU16<U>, ŌĆ”)
conn::half::*           # types from `half` crate (f16, bf16)
conn::time::*           # types from `time` crate (Date, Time, PrimitiveDateTime,
                        # Duration, OffsetDateTime ŌĆö clock.rs, date.rs,
                        # datetime.rs, duration.rs, offset.rs)
conn::float             # thin module hosting `ExtendedFloat<T>` only
                        # (the type is this-crate's wrapper, not bound
                        # to any one dep crate)
```

Filenames within each per-crate module follow the **type name** of the
host type ŌĆö `i8.rs`, `u128.rs`, `f64.rs`, `bf16.rs`, `date.rs`,
`duration.rs`, `offset.rs`. The 8-char zero-padded form (`I008I016`)
is for Conn const **identifiers**, not module paths.

Concrete placement consequences (representative):

| Conn         | Sides                              | Winning rule              | Module                          |
|--------------|------------------------------------|---------------------------|---------------------------------|
| `F064F032`   | `f64` / `f32` (both std)           | tie ŌåÆ right wins          | `conn::std::f32`                |
| `F064F016`   | `f64` (std) / `f16` (half)         | (1) `f16` more specific   | `conn::half::f16`               |
| `F064B016`   | `f64` (std) / `bf16` (half)        | (1) `bf16` more specific  | `conn::half::bf16`              |
| `F032F016`   | `f32` (std) / `f16` (half)         | (1) `f16` more specific   | `conn::half::f16`               |
| `F032B016`   | `f32` (std) / `bf16` (half)        | (1) `bf16` more specific  | `conn::half::bf16`              |
| `I008I016`   | `Extended<i8>` / `i16`             | tie ŌåÆ right wins          | `conn::std::i16`                |
| `U008I016`   | `Extended<u8>` / `i16`             | tie ŌåÆ right wins          | `conn::std::i16`                |

carries calendar / range semantics; `FD09` is a numeric rung that
happens to mean "nanoseconds"). The earlier draft of this plan
flagged DURNFD09 for relocation under a "right-wins-everywhere"
read; the specificity rule resolves it cleanly without a move.

This rule set is codified in CLAUDE.md by T7a.

### T15 ŌĆö Attribute hygiene: `#[must_use]` and `#[inline]`

**Problem:** API audit blockers. `Conn::ceil/inner/floor` are
missing `#[must_use]` (discarding their result is almost certainly
a bug) and `#[inline]` (each method is a single fn-pointer dispatch;
without `#[inline]`, LLVM cannot inline through `c.ceil(x)`).

**Approach:** In `src/conn.rs`:

```rust
impl<A, B> Conn<A, B> {
    #[inline]
    #[must_use]
    pub fn ceil(&self, a: A) -> B { (self.ceil)(a) }

    #[inline]
    #[must_use]
    pub fn inner(&self, b: B) -> A { (self.inner)(b) }

    #[inline]
    #[must_use]
    pub fn floor(&self, a: A) -> B { (self.floor)(a) }
}
```

Also add `#[must_use]` to the cast free-fns in `src/conn/cast.rs`.
`#[inline]` is already on the cast free-fns.
