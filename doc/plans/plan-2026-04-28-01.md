## Tasks

### T1 вҖ” `STDRU064` and `STDRU128` Conns (whole seconds + nanoseconds)

Add to `src/time/duration.rs`. Disambiguate the two `Duration` types
via `use std::time::Duration as StdDuration;` near the top.

- **`STDRU064`**: `Conn<StdDuration, Extended<u64>>`.
  - `floor(d) = Finite(d.as_secs())` for finite `d` with no subsec rest;
    `Finite(d.as_secs())` when `subsec_nanos == 0`; else floor of
    integer seconds (which is `as_secs()` for unsigned).
  - `ceil(d) = Finite(d.as_secs() + (subsec_nanos > 0) as u64)`,
    saturating to `PosInf` if `as_secs() == u64::MAX && subsec_nanos > 0`.
  - `inner(Finite(s)) = StdDuration::from_secs(s)`.
  - `inner(NegInf) = StdDuration::ZERO` (saturating; no negative
    representatives), `inner(PosInf) = StdDuration::MAX`.
- **`STDRU128`**: `Conn<StdDuration, Extended<u128>>`.
  - On Finite: `floor = ceil = d.as_nanos()` вҖ” bijection on the
    representable range. Degenerate-Galois (the bijection is its own
    Galois adjoint).
  - `inner(Finite(n))`: split `n` into `(secs = n / 10^9, subsec_nanos
    = n % 10^9)` and call `StdDuration::new(secs as u64, subsec_nanos
    as u32)`. Saturate to `StdDuration::MAX` if `secs > u64::MAX`.
  - `inner(NegInf) = ZERO`, `inner(PosInf) = MAX`.

Both Conns saturate cleanly because `StdDuration` has no signed
representation. The `Extended<u64>` rung mirrors `DURNSECS`'s
`Extended<i64>` shape so the contrast (signed vs unsigned overflow
arms) is visible in adjacent code.

### T2 вҖ” `F064STDR` and `F032STDR` (float seconds вҶ” Duration)

Add to `src/time/duration.rs`, immediately after T1's constants.

- Extend the existing `def_walk_helpers!` invocations: today they
  drive walks against `time::Duration` via `shift_duration` and
  `duration_to_f64` / `duration_to_f32`. Add std-Duration analogues:
  - `shift_std_duration(n: i32, d: StdDuration) -> StdDuration` вҖ”
    nanosecond shift saturating at `StdDuration::ZERO` and
    `StdDuration::MAX` (no negative arm).
  - `std_duration_to_f64(d: StdDuration) -> f64`,
    `std_duration_to_f32(d: StdDuration) -> f32` вҖ” use
    `d.as_secs_f64()` / `as_secs_f32()` (these are total).
- Two new `def_walk_helpers!` invocations: `f64_stdr_walks`,
  `f32_stdr_walks`.
- Two new `pub const`: `F064STDR: Conn<F064, Extended<StdDuration>>`,
  `F032STDR: Conn<F032, Extended<StdDuration>>`.
- Boundary handling: `inner(Bot)` of the float Conn produces
  `Extended<StdDuration>::NegInf`; `inner(NegInf)` of the rung side
  saturates to `StdDuration::ZERO`. Negative input floats (e.g.
  `-1.0_f64`) project to `NegInf` because the unsigned StdDuration
  has no representative below ZERO. This asymmetry vs `F064DURN`
  (which lands in signed `time::Duration`) needs an explicit doctest
  showing what `F064STDR.ceil(F064::from(-0.5))` yields.

## Review

### Design deviations from the plan

1. **Source-side `Extended` wrap on STDRU064 / STDRU128.** The plan
   specified `Conn<StdDuration, Extended<u64>>` and
   `Conn<StdDuration, Extended<u128>>`. As written those aren't
   Galois-lawful: with `inner(NegInf) = StdDuration::ZERO` (the only
   StdDuration вүӨ everything), Galois forces `ceil(StdDuration::ZERO) =
   NegInf` (the same trick `DURNSECS` uses at `Duration::MIN`). At
   `MIN` that's a deeply-saturated edge; at `ZERO` it's the most-common
   input and would be a real footgun. Wrapped both sides instead вҖ”
   `Conn<Extended<StdDuration>, Extended<u64>>` and
   `Conn<Extended<StdDuration>, Extended<u128>>` вҖ” matching the
   `OFDTNANO`/`OFDTSECS` shape. `ceil(Finite(ZERO))` is now naturally
   `Finite(0)`. The constant names are unchanged because the `STDR`
   half is the all-letter `ABCD` shape that elides the `Extended` wrap
   (same convention `OFDT` uses).
2. **F???STDR negative-input semantics.** The plan said negative
   floats project to `NegInf` for both `ceil` and `floor`. With
   `inner(NegInf rung) = ExtendedFloat::Bot` (the synthetic source
   bottom), Galois forces `ceil(Extend(v < 0)) = Finite(StdDuration
   ::ZERO)` and `floor(Extend(v < 0)) = NegInf` вҖ” asymmetric. The
   doctests, spot tests, and a dedicated proptest
   (`f64_stdr_negative_input_ceil_to_zero`) assert the corrected arms.
3. **STDRU128 law battery uses an in-range strategy.** The rung
   `Extended<u128>` extends past `StdDuration::MAX.as_nanos()`
   (вүҲ 1.84Г—10ВІвҒё vs `u128::MAX` вүҲ 3.4Г—10ВівҒё). On the bijection's image
   `STDRU128` is lawful, but `inner` saturates outside that image and
   the Galois law breaks at `(Finite(MAX), Finite(MAX.as_nanos() + 1))`
   (proptest hit this on the first run). Switched the
   `STDRU128.{galois_l/r,kernel_l/r,monotone_r}` strategy from
   `arb_extended_u128` to `arb_extended_stdr_nanos_in_range` (the same
   pattern `OFDTNANO` uses with `arb_unix_nanos_in_range`). The full
   `arb_extended_u128` strategy is kept in `prop::arb` for downstream
   use.

### Ignored properties

None вҖ” every proptest in the new battery is enabled.

### Recommendations

- A future `DURNSTDR` cross-crate Conn (deferred above) will need the
  same source-wrap analysis. The `OFDTNANO` saturation pattern is the
  template; reach for it when adding any unsigned-rung Conn whose
  source has only one open end.
- The `arb_extended_u64` / `arb_extended_u128` strategies added here
  belong to a small unsigned-Extended family that the rest of `prop/`
  should grow into uniformly when a downstream Conn wants them.
