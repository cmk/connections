### T4 — Document the `fixed::iN`/`fixed::uN` per-host alias layer

**Files**: `src/fixed.rs` (module-level docs), `src/lib.rs` (cross-link)

Add a `### Per-host type aliases` paragraph to `src/fixed.rs`'s
`//!` doc comment, clarifying that the `I###`/`U###` aliases in
`fixed::iN`/`fixed::uN` modules are a per-host wrapper layer
(letter = host signedness, digits = frac level), distinct from the
crate-level Conn-name convention (where `I###`/`U###` are reserved
for std-primitive sides and `Q###` for Q-format wrapper sides). The
two namespaces never collide because the per-host aliases are reached
through qualified module paths and Conn-name sides only appear inside
8-char Conn const identifiers.

Cross-link from `src/lib.rs`'s Naming section with one sentence.

No code changes — audit confirmed all 10 `fixed::*` files already
follow the documented pattern uniformly.

## Review

### Drift caught (gardener rule)

- `src/float.rs:907`, `src/float.rs:1046` — `crate::float::f016::shift16_f16`
  references a path that doesn't exist (`shift16_f16` lives in
  `crate::core::f016`, not `crate::float::f016`). The default-feature
  build skipped these lines via cfg gating; the `f16` feature build
  did not. **Fixed in this sprint** as a drive-by in the T3
  dedup commit (`debt: Dedup F016 alias; fix crate::float::f016 path
  typo`). Caught when validating that T3 didn't introduce a new
  broken path — the f16 build was already failing on `main`.

### Design deviations from the plan

- **T3 import shape.** The plan sketch had `core/f016.rs` switching
  to `use crate::float::F016;` for self-reference. After re-reading
  the source, `core/f016.rs` doesn't actually use `F016` as a type
  anywhere (only as a macro argument prefix for `pub F016U008`
  etc.), so the alias re-import would be a dead import warning. The
  in-file doc-link `[\`F016\`]` was retargeted to
  `[\`F016\`](crate::float::F016)` instead. `core/f032.rs` and
  `core/f064.rs` (which do use `F016` as a type) got the import added.
- **T4 lib.rs cross-link.** Sprint plan proposed
  `see [\`mod@fixed\`]` — the intra-doc-link broke the default-feature
  `cargo doc -D warnings` build because `mod fixed` is gated on the
  `fixed` feature. Switched to prose ("The `fixed` module docs
  (gated on the `fixed` feature) carry the full layering rationale.")
  with no link. Same pattern as the pre-existing F016 row in lib.rs's
  Naming table, kept prose for the identical reason.
- **No additional repetitions of the const/struct convention beyond
  the two budgeted placements** (the `> Note.` in `## L & R kind
  connections` and the `> Convention.` callout in `## 2-kinded
  connections`).
