# Plan 08 — Standardize Conn-name format + add 128-bit Conns

## Goal

Adopt a uniform 8-character `XnnnYmmm` shape for every public `Conn`
constant — 4 chars per side, smaller-resolution tier on the right —
so a name parses unambiguously without the digit-range tricks the
old `F##` scheme leaned on; then close the gap to 128-bit primitives
that were absent from every integer family.

## Locked design decisions (per the design discussion in plan file)

- **D1. Family letters** — `FD` (decimal fixed exp, 2L+2D), `F` (float
  width, 1L+3D), `I` (signed binary fixed frac AND signed int width,
  1L+3D), `U` (unsigned binary fixed frac AND unsigned int width,
  1L+3D), `S` (sample rate kHz, 1L+3D). Name collisions across modules
  are allowed and resolved by qualified-import: `use connections::conn::fixed::i08
  as fi8; fi8::I008I004`.
- **D2. Wrappers** — full conformity. `Uni`→`FD00`, …, `Pico`→`FD12`;
  `S44`→`S044`, `S48`→`S048`, `S88`→`S088`, `S96`→`S096` (S176/S192
  unchanged). Type aliases added per binary module: `pub type I008 =
  FixedI8<U8>;` etc.
- **D3. i128 binary-fixed widening** — `i128::checked_mul` + saturate.
  Bit-identical outputs to a hypothetical i256-widening pattern; same
  Galois-law machinery as the i08/i16/i32/i64 modules. The
  `I128I000` endpoint is intrinsically degenerate (RATIO = 2^128
  doesn't fit i128); special-cased via `i128::checked_shl(128)`
  returning `None`.

## Deferred

- **`conn::fixed::u08` / `u16` / `u32` / `u64` / `u128`** — unsigned
  binary-fixed conns. Naming scheme already accommodates them
  (`U008U004` etc.) but no `FixedU<N>`-backed conns exist today.
  Defer until there's a concrete user request.
- **Cross-sign binary-fixed conns** (`I008U004` between `FixedI8<U8>`
  and `FixedU8<U4>`): same deferral — needs the unsigned modules first.
- **`F032` cross-tier** — deferred for the plateau-collapse reason
  documented in `decimal.rs` (i64→f32 narrowing creates ~120k-element
  collapse classes that turn correction loops into O(plateau) walks).
- **`F128` cross-tier** — blocked on `f128` stabilisation in stable Rust.
- **Decimal ladder backing extension** — keeping `FD00`..`FD12` on i64
  rather than i128 backing. Out of scope for this sprint; would be a
  separate breaking change.
- **arb-strategy rename** (e.g. `pico_fine` → `fd12_fine`) — the
  strategies still work; their lowercase names reflect the
  parameter shape, not a tier code. Defer to a cosmetic-only sprint.
