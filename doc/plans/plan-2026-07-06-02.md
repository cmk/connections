# Plan 02 ŌĆö Retire the double-swap view idiom for `view_l`/`view_r`

## Goal
Finish the in-flight migration from the verbose `X.swap_l().swap_r()` /
`X.swap_r().swap_l()` view-extraction idiom to the named
`view_l(&X)` / `view_r(&X)` free functions. Promote `view_l`/`view_r`
from `pub(crate)` to `pub` (the module doc already calls `view_l(t)` "the
public spelling of a direct view"), then replace every eligible
occurrence across doctests, `tests/`, the `law_battery!` macro, and
internal code.

## Motivation
`crate::conn` already ships `view_l(t) = t.swap_l().swap_r()` and
`view_r(t) = t.swap_r().swap_l()`, and newer predicate code uses them
(`view_l(t).ceil(a)`). But ~103 older sites still spell the view as a
raw double swap. The two are fn-pointer-identical by the
`swap_involutive_*` laws, so this is a mechanical readability cleanup ŌĆö
`view_l(&F064F032)` beats `F064F032.swap_l().swap_r()`.

The blocker: `view_l`/`view_r` are `pub(crate)`, so doctests (external
crates), `tests/`, and the `#[macro_export] law_battery!` expansion
(runs downstream) can't call them. Promoting to `pub` unblocks the full
migration and makes the module doc's "public spelling" claim true.

## What migrates vs. what stays

Migrate (marker receiver, runtime position):
- **59** occurrences in doctests (`///` / `//!`).
- **13** occurrences in `tests/` (`macros_feature_smoke.rs`, `fixed/nz_iso.rs`).
- The `law_battery!` macro body in `src/prop/conn.rs` (galois / closure /
  kernel / monotone / idempotent / iso / roundtrip law checks) ŌåÆ
  `$crate::conn::view_l(&($c))` (qualified, for downstream hygiene).

Stays as a double swap (each for a concrete reason):
- `src/conn.rs` `view_l` / `view_r` **definitions** ŌĆö replacing them
  would be infinite recursion.
- `src/prop/conn.rs` `swap_involutive_l` / `swap_involutive_r` bodies
  (`c.swap_l().swap_r() == *c`) and the `swap_involution` law-battery
  test ŌĆö they *are* the involution law; the receiver is a `Conn`
  **value**, which doesn't impl `ConnL`/`ConnR` anyway.
- `src/conn.rs` swap-round-trip **unit tests** (`Conn` value receiver).
- **const positions**: `compose_l!` / `compose_r!` (`$t1` / `$t2`) and
  `lift_l!` / `lift_r!` (`$parent`). `view_l` is generic over the
  `ConnL` **trait** method, so it is not `const`; the inherent
  `const fn swap_l` / `swap_r` are, so const composition must keep them.
