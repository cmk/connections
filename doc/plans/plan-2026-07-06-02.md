# Plan 02 — Retire the double-swap view idiom for `view_l`/`view_r`

## Goal
Finish the in-flight migration from the verbose `X.swap_l().swap_r()` /
`X.swap_r().swap_l()` view-extraction idiom to the named
`view_l(&X)` / `view_r(&X)` free functions. Promote `view_l`/`view_r`
from `pub(crate)` to `pub` (the module doc already calls `view_l(t)` "the
public spelling of a direct view"), then replace every eligible
occurrence across doctests, `tests/`, the `law_battery!` macro, and
internal code.

## Motivation
`crate::conn` already ships `view_l(t) = t.swap_l().swap_r()` and
`view_r(t) = t.swap_r().swap_l()`, and newer predicate code uses them
(`view_l(t).ceil(a)`). But ~103 older sites still spell the view as a
raw double swap. The two are fn-pointer-identical by the
`swap_involutive_*` laws, so this is a mechanical readability cleanup —
`view_l(&F064F032)` beats `F064F032.swap_l().swap_r()`.

The blocker: `view_l`/`view_r` are `pub(crate)`, so doctests (external
crates), `tests/`, and the `#[macro_export] law_battery!` expansion
(runs downstream) can't call them. Promoting to `pub` unblocks the full
migration and makes the module doc's "public spelling" claim true.

## What migrates vs. what stays

Migrate (marker receiver, runtime position):
- **59** occurrences in doctests (`///` / `//!`).
- **13** occurrences in `tests/` (`macros_feature_smoke.rs`, `fixed/nz_iso.rs`).
- The `law_battery!` macro body in `src/prop/conn.rs` (galois / closure /
  kernel / monotone / idempotent / iso / roundtrip law checks) →
  `$crate::conn::view_l(&($c))` (qualified, for downstream hygiene).

Stays as a double swap (each for a concrete reason):
- `src/conn.rs` `view_l` / `view_r` **definitions** — replacing them
  would be infinite recursion.
- `src/prop/conn.rs` `swap_involutive_l` / `swap_involutive_r` bodies
  (`c.swap_l().swap_r() == *c`) and the `swap_involution` law-battery
  test — they *are* the involution law; the receiver is a `Conn`
  **value**, which doesn't impl `ConnL`/`ConnR` anyway.
- `src/conn.rs` swap-round-trip **unit tests** (`Conn` value receiver).
- **const positions**: `compose_l!` / `compose_r!` (`$t1` / `$t2`) and
  `lift_l!` / `lift_r!` (`$parent`). `view_l` is generic over the
  `ConnL` **trait** method, so it is not `const`; the inherent
  `const fn swap_l` / `swap_r` are, so const composition must keep them.

## Dependency Graph
T1 → T2 → T3 → T4 (visibility first, then call sites, then imports,

## Tasks

### T1 — Promote `view_l` / `view_r` to `pub`
`src/conn.rs`: `pub(crate) fn view_l` → `pub fn view_l` (and `view_r`).
Confirm no `unreachable_pub` / doc-link fallout.

### T2 — Migrate call sites
Deterministic textual pass keyed by explicit skip-line sets (definitions,
involution laws, const macro args, swap unit tests). Receiver forms:
bare ident (`F064F032`), `$`-ident (`$CONN`), parenthesized (`($c)`).
`law_battery!` uses the `$crate::conn::` prefix; everything else uses the
bare `view_l` / `view_r` (imports added in T3).

### T3 — Imports
- `tests/` files: add `use connections::conn::{view_l, view_r};` (only
  the ones used) at file top.
- Doctests: add `use connections::conn::{view_l, view_r};` to each block
  that now calls them (compiler-driven — `cargo test --doc` names each
  block missing the import).
- `law_battery!` needs no import (qualified path).

### T4 — Doc prose (gardener)
Update the `conn.rs` module doc and `ConnL` / `ConnR` trait docs that
describe the double swap as *the* public spelling now that `view_l` /
`view_r` are public. Keep the involution-law cross-reference.

## Review

- **Scope delivered.** `view_l` / `view_r` are `pub`; ~70 marker-receiver
  double swaps migrated (doctests + `tests/`). 26 doctest blocks gained a
  `use connections::conn::{…};`. The `law_battery!` macro migration was
  **reverted** (see codex finding below), so the keep-set is 31 code
  sites: 2 definitions, 2 involution-law bodies, 19 `law_battery!` sites,
  2 involution-law-battery test lines, 2 swap unit tests, 4 const macro
  args.

- **No `#[ignore]`d properties.** Full suite green
  (`--features fixed,time,hifi,uhlc,proptest,macros`): 1925 lib + 9 + 1
  integration + 90 doctests, 0 failed. Behaviour-preserving by the
  kept `swap_involutive_*` laws.

- **Bug found & fixed during migration:** `unsigned_nz_props` in
  `tests/fixed/nz_iso.rs` passes a **one-sided `Conn` value**
  (`U###N###`, only `galois_l` holds), where `X.swap_l().swap_r()` was
  an identity no-op — `view_l(&X)` fails (`Conn` value isn't `ConnL`).
  Reverted those two sites to the bare `$CONN`. Confirms the value-vs-
  marker distinction the plan called out; the compiler caught it (E0277).

- **Mechanical over-reach, corrected:** the textual pass also rewrote
  double-swap mentions inside **doc prose**, producing nonsense like
  `view_l(&c)` on a `Conn` value (in the `ConnL` "Laws" text and the
  `swap_involutive_*` docs). All prose reverted / rewritten by hand:
  `conn.rs` module doc + `ConnL`/`ConnR` docs, `extended.rs`,
  `prop.rs`, `prop/conn.rs`.

- **Codex finding (local review) — `law_battery!` regression, fixed:**
  the exported `law_battery!` is documented to accept raw one-sided
  `Conn` values (`conn_l!` / `conn_r!` consts with `l_only` / `r_only` /
  `iso_only`). Migrating its `($c).swap_?().swap_?()` to `view_?(&($c))`
  broke that (a `Conn` value isn't `ConnL` / `ConnR`) — the double swap
  is the *uniform* spelling (view on markers, identity on values).
  Reverted the 19 macro sites to the double swap; it is a keep-site.

- **swap_involution test kept raw (design):** the `swap_involution`
  law-battery test (`prop/conn.rs:678-680`) still spells the view as
  `($c).swap_l().swap_r()` rather than `view_l(&($c))`, deliberately —
  it exists to verify the involution that *justifies* `view_l`, so it
  must not be written in terms of `view_l`.

- **Drift caught (gardener rule):** `conn.rs` `ConnR` doc said the
  inherent view accessor was `const fn r()`; there is no such method
  (it is `view_r()`). Corrected.

- **Recommendations:** consider a `#[doc(hidden)]`-free prelude entry
  for `view_l` / `view_r` so doctests need not import them explicitly
  (out of scope here). See Deferred for the PR #18 stragglers.
