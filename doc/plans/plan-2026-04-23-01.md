# Plan 02 ŌĆö Fixed decimal tier + rate-typed sample tier

## Goal

Add two modules under `connections::`:

1. **`fixed`** ŌĆö port of the Haskell `Data.Connection.Fixed` ladder:
   decimal fixed-point newtypes `Uni / Deci / Centi / Milli / Micro /
   Nano / Pico` over `i64`, with `Conn<fine, coarse>` triples between
   every pair (adjacent and non-adjacent) mirroring Haskell's
   `f01f00`, `f02f00`, ŌĆ”, `f12f09` naming.

2. **`sample`** ŌĆö rate-typed sample-indexed time: `S44, S48, S88, S96,
   S176, S192`, each a distinct newtype backed by `fixed::FixedI64<U16>`
   (Q48.16 samples). Galois connections between all rate pairs (exact
   ├Ś2/├Ś4 for power-of-two families, rational rounding across families
   like S44ŌåöS48=147:160) and from each rate to `Pico`.

Motivation: the sibling `agogo` project is about to refactor its DSP
core to eliminate floating-point from stored values. It needs
rate-aware sample types and a decimal time ladder. Rather than build
them locally in agogo, land them upstream in `connections` where they
logically belong ŌĆö every future consumer of Galois-connected
fixed-point benefits.

**Types** ŌĆö all `#[repr(transparent)]` newtypes over `i64`, deriving
`Copy, Clone, Debug, PartialEq, Eq, PartialOrd, Ord, Hash, Default`:

```rust
pub struct Uni  (pub i64);     // resolution 10Ōü░
pub struct Deci (pub i64);     // resolution 10Ōü╗┬╣
pub struct Centi(pub i64);     // resolution 10Ōü╗┬▓
pub struct Milli(pub i64);     // resolution 10Ōü╗┬│
pub struct Micro(pub i64);     // resolution 10Ōü╗ŌüČ
pub struct Nano (pub i64);     // resolution 10Ōü╗Ōü╣
pub struct Pico (pub i64);     // resolution 10Ōü╗┬╣┬▓
```

The stored `i64` is the numerator; the denominator is implicit via
resolution. A `Pico(5)` represents `5 ├Ś 10Ōü╗┬╣┬▓` in abstract units.

```rust
pub trait HasResolution {
    /// The resolution denominator. `Pico::PREC = 10^12`,
    /// `Uni::PREC = 1`, etc.
    const PREC: i64;
}
```

**Connections**: one `pub const` per (fine, coarse) pair, named per
the Haskell `fXYfZW` convention. For each pair, three free `fn`
functions (so they fit `Conn`'s non-capturing `fn(A) -> B` shape):

```rust
fn f03f00_ceil (x: Milli) -> Uni {
    // ceil(x / 1000) using Euclidean (always-positive-remainder) division
    let q = x.0.div_euclid(1_000);
    if x.0.rem_euclid(1_000) != 0 { Uni(q + 1) } else { Uni(q) }
}
fn f03f00_inner(x: Uni)   -> Milli { Milli(x.0 * 1_000) }
fn f03f00_floor(x: Milli) -> Uni   { Uni(x.0.div_euclid(1_000)) }

pub const F03F00: Conn<Milli, Uni> = Conn::new(f03f00_ceil, f03f00_inner, f03f00_floor);
```

Note: the Rust port follows the documented `Conn` semantics
(`ceil ŌŖŻ inner ŌŖŻ floor` with `floor = standard floor-division`). This
matches the Haskell `ratfix` implementation of `h`; it differs from
Haskell `fixfix`'s `h` which returns `j - 1` on nonzero remainder ŌĆö
likely an idiosyncrasy of the Haskell library, not a different Galois
law. The Rust port standardises on the documented adjoint triple.

**Adjacent pairs** (one per "neighbour" in the decimal ladder):

`F01F00 DeciŌåÆUni, F02F01 CentiŌåÆDeci, F03F02 MilliŌåÆCenti,
F06F03 MicroŌåÆMilli, F09F06 NanoŌåÆMicro, F12F09 PicoŌåÆNano`.

**Non-adjacent pairs** per Haskell reference lines 85-158 ŌĆö direct
triples rather than compositions (to preserve `fn`-pointer shape):

- `F02F00: CentiŌåÆUni (├Ś100)`
- `F03F00, F03F01: MilliŌåÆ{Uni, Deci}`
- `F06F00..F06F02: MicroŌåÆ{Uni, Deci, Centi}`
- `F09F00..F09F03: NanoŌåÆ{Uni, Deci, Centi, Milli}`
- `F12F00..F12F06: PicoŌåÆ{Uni, Deci, Centi, Milli, Micro}`

Total: 6 adjacent + 15 non-adjacent = **21 `Conn` constants**, each
with three free fns = 63 functions. Generate by template (macro) to
avoid copy-paste.

**Macro** to DRY the triples:

```rust
macro_rules! fix_fix {
    ($name_ceil:ident, $name_inner:ident, $name_floor:ident,
     $const_name:ident, $fine:ident, $coarse:ident, $prec:expr) => {
        fn $name_ceil(x: $fine) -> $coarse {
            let q = x.0.div_euclid($prec);
            if x.0.rem_euclid($prec) != 0 { $coarse(q + 1) } else { $coarse(q) }
        }
        fn $name_inner(x: $coarse) -> $fine { $fine(x.0 * $prec) }
        fn $name_floor(x: $fine) -> $coarse { $coarse(x.0.div_euclid($prec)) }
        pub const $const_name: Conn<$fine, $coarse> =
            Conn::new($name_ceil, $name_inner, $name_floor);
    };
}
fix_fix!(f01f00_ceil, f01f00_inner, f01f00_floor, F01F00, Deci,  Uni,   10);
fix_fix!(f02f01_ceil, f02f01_inner, f02f01_floor, F02F01, Centi, Deci,  10);
// ... etc.
```

**Proptests** (in-module `#[cfg(test)] mod tests`):

For each connection `C: Conn<Fine, Coarse>`:

- `inner_then_ceil_is_id`: `ŌłĆ b. C.ceil(C.inner(b)) == b` (upper adjoint)
- `inner_then_floor_is_id`: `ŌłĆ b. C.floor(C.inner(b)) == b` (lower adjoint)
- `ceil_monotone`: `x Ōēż y ŌćÆ C.ceil(x) Ōēż C.ceil(y)`
- `floor_monotone`: `x Ōēż y ŌćÆ C.floor(x) Ōēż C.floor(y)`
- `floor_le_ceil`: `C.floor(x) Ōēż C.ceil(x)` and they differ by at
  most 1 ULP of `Coarse`
- `galois_upper`: `C.ceil(x) Ōēż b Ō¤║ x Ōēż C.inner(b)`
- `galois_lower`: `C.inner(b) Ōēż x Ō¤║ b Ōēż C.floor(x)`
- `negative_values`: all of the above for `i < 0` (regression-proofing
  against integer division's sign pitfalls)

Strategy ranges bounded so `inner` can't overflow: `|Coarse.0| <
i64::MAX / PREC`. Edge cases included via `prop_oneof!` with
frequency weights on `0, ┬▒1, ┬▒(PREC-1), ┬▒PREC, ┬▒(PREC+1)`.

Composition property across the full ladder:
- `composition_matches_direct`: `F12F00` applied to `Pico(x)` equals
  `F12F09` Ōłś `F09F06` Ōłś `F06F03` Ōłś `F03F00` applied to the same `x`
  (i.e. the direct non-adjacent triple agrees with the four-step
  climb up the ladder).

**Types** ŌĆö one newtype per standard audio sample rate, each wrapping

```rust
#[repr(transparent)] pub struct S44 (pub FixedI64<U16>);  // 44 100 Hz

#[repr(transparent)] pub struct S88 (pub FixedI64<U16>);  // 88 200 Hz
#[repr(transparent)] pub struct S96 (pub FixedI64<U16>);  // 96 000 Hz
#[repr(transparent)] pub struct S176(pub FixedI64<U16>);  // 176 400 Hz
#[repr(transparent)] pub struct S192(pub FixedI64<U16>);  // 192 000 Hz

pub trait SampleRate {
    const HZ: u32;
}

impl SampleRate for S44  { const HZ: u32 = 44_100;  }
// ... through S192
```

Derives per type: `Copy, Clone, Debug, PartialEq, Eq, PartialOrd,
Ord, Hash, Default`. Inherent methods:

```rust
impl S48 {
    pub const ONE_SAMPLE: Self = Self(FixedI64::<U16>::from_bits(1 << 16));
    pub const fn from_sample(n: i64) -> Self { Self(FixedI64::from_bits(n << 16)) }
    pub const fn from_bits(bits: i64) -> Self { Self(FixedI64::from_bits(bits)) }
    pub const fn to_bits(self) -> i64 { self.0.to_bits() }
    pub const fn sample(self) -> i64 { self.0.to_bits() >> 16 }       // arithmetic shift for signed
    pub const fn sub_q16(self) -> i16 { (self.0.to_bits() & 0xFFFF) as i16 }
}
```

Rational pairs (Galois-exact rounding):

```rust
// S44 ŌåÆ S48: s48_bits = s44_bits * 160 / 147 (rational ŌĆö lossy)
fn s44_to_s48_ceil(x: S44) -> S48 {
    let n: i128 = x.to_bits() as i128 * 160;
    // div_euclid for sign-correct "round toward -inf, add 1 if remainder nonzero"
    let q = n.div_euclid(147);
    let r = n.rem_euclid(147);
    S48(FixedI64::from_bits((if r != 0 { q + 1 } else { q }) as i64))
}
fn s48_to_s44_inner(x: S48) -> S44 {
    let n: i128 = x.to_bits() as i128 * 147;
    S44(FixedI64::from_bits(n.div_euclid(160) as i64))
}
fn s44_to_s48_floor(x: S44) -> S48 {
    let n: i128 = x.to_bits() as i128 * 160;
    S48(FixedI64::from_bits(n.div_euclid(147) as i64))
}
pub const S44_S48: Conn<S44, S48> = Conn::new(s44_to_s48_ceil, s48_to_s44_inner, s44_to_s48_floor);

// Also all other cross-family rationals: S88ŌåöS96, S88ŌåöS48, S176ŌåöS96,
// S176ŌåöS48, S192ŌåöS44, S192ŌåöS88, ŌĆ”
```

**Connections from each Sxx to `Pico`**:

```rust
// S48 ŌåÆ Pico: ps = sample_bits_q16 * (10^12 / 48_000) / 2^16
//                = sample_bits_q16 * 125_000_000 / (6 * 2^16)
//                = sample_bits_q16 * 125_000_000 / 393_216
fn s48_to_pico_ceil(x: S48) -> Pico {
    let n: i128 = x.to_bits() as i128 * 125_000_000;
    let d: i128 = 393_216;
    let q = n.div_euclid(d);
    let r = n.rem_euclid(d);
    Pico((if r != 0 { q + 1 } else { q }) as i64)
}
fn s48_to_pico_inner(x: Pico) -> S48 {
    // bits_q16 = ps * 393_216 / 125_000_000 (round nearest)
    let n: i128 = x.0 as i128 * 393_216;
    let d: i128 = 125_000_000;
    // Nearest-rounding for the embed direction keeps adjoint laws tight.
    S48(FixedI64::from_bits(((n + d / 2).div_euclid(d)) as i64))
    // Note: for negative n, need symmetric-round-half-to-even or similar.
    // See test `s48_pico_inner_round_nearest` for exact convention.
}
fn s48_to_pico_floor(x: S48) -> Pico {
    let n: i128 = x.to_bits() as i128 * 125_000_000;
    Pico(n.div_euclid(393_216) as i64)
}
pub const S48_PICO: Conn<S48, Pico> = Conn::new(s48_to_pico_ceil, s48_to_pico_inner, s48_to_pico_floor);

// 6 such Conns, one per rate.
```

Numerator constants (simplified 10┬╣┬▓/SR):

| Rate | 10┬╣┬▓ / SR | Simplified |
|------|-----------|------------|
| 44100 | 22 675 736.961ŌĆ” | 10^12 / 44_100 = 10^12 / (2┬▓ ├Ś 3┬▓ ├Ś 5┬▓ ├Ś 7┬▓) |
| 48000 | 20 833 333.33ŌĆ” | 10^12 / 48_000 = 125_000_000 / 6 |
| 88200 | 11 337 868.48ŌĆ” | ┬Į of 44100 factor |
| 96000 | 10 416 666.67ŌĆ” | ┬Į of 48000 factor = 62_500_000 / 6 |
| 176400 | 5 668 934.24ŌĆ” | ┬╝ of 44100 factor |
| 192000 | 5 208 333.33ŌĆ” | ┬╝ of 48000 factor |

To keep arithmetic in i128 without loss, use the pattern
`num = bits_q16 * A, den = B ├Ś 2^16` with A/B chosen from the simplified
ratio. For 48000: `A = 125_000_000, B = 6`. For 44100: `A = 10^12, B = 44_100`
or further simplify via gcd (`gcd(10^12, 44100) = 4`, giving
`A = 250_000_000_000, B = 11_025`). Keep the arithmetic explicit per-rate.
