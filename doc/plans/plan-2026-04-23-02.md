# Plan 03 — GitLab workflow port from template-rust

## Goal

Port the mature two-tier review workflow from
`~/Music/Software/Rust/template-rust` into this repo, translated
for GitLab (MRs, `glab`), so `/sprint-review` becomes a first-class
command with a post-push review round-trip and an audit trail in
`doc/reviews/`.

## Context

This sprint is step 1 of the three-sprint cleanup described in the
meta-plan `this-library-is-accumulating-deep-wilkes.md` (in Chris's
`~/.claude/plans/` directory). It runs first so Sprint B (file tree
reorg) and Sprint C (proptest audit) benefit from the improved
`/sprint-review`, autosquash, and PII gate.

Reference template: `~/Music/Software/Rust/template-rust/`.
That template is GitHub-based. The port swaps GitHub primitives for
GitLab equivalents:

| GitHub | GitLab |
|--------|--------|
| `gh` CLI | `glab` CLI |
| PR | MR (merge request) |
| `.github/workflows/ci.yml` | extend existing `.gitlab-ci.yml` |
| `.github/pull_request_template.md` | `.gitlab/merge_request_templates/default.md` |
| `.github/CODEOWNERS` | `CODEOWNERS` at repo root |

Current repo state to replace/extend:
- `.claude/skills/sprint-review.md` — legacy skill form; will be deleted.
- `.claude/settings.json` — pre-commit hook runs `cargo test` + `cargo clippy` only.
- `.gitlab-ci.yml` — single-stage: `cargo test` + `cargo clippy -D warnings`, caches `target/` and `.cargo/`.
- `doc/refs/review-calibration.md` — already exists; will move to `doc/reviews/review-calibration.md`.
- No `scripts/`, no `doc/reviews/`, no `doc/workflow.md`, no `rust-toolchain.toml`/`rustfmt.toml`/`deny.toml`.

## Tasks

### T1 — Port `scripts/` directory

Copy and adapt the seven helper scripts from template-rust. Most are
git-only and copy verbatim; the GitHub-facing ones are rewritten against
the GitLab API.

**Copy verbatim** (git-only, VCS-agnostic):
- `autosquash.sh` — collapses fixup commits via `git rebase --autosquash`.
- `check-pii.sh` — scans staged content for absolute paths and API
  tokens.

**Adapt `gh` → `glab`** (lightweight CLI wrappers):
- `review_path.sh` — replace `gh pr view --json number --jq .number`
  with `glab mr view -F json | jq .iid`. Produces zero-padded filename
  under `doc/reviews/`.
- `extract_pr_body.sh` → rename `extract_mr_body.sh`. Reads the
  `## Summary` section from `doc/reviews/review-NNNNN.md`; no CLI call,
  just text extraction. Rename tracks the MR terminology.
- `next_pr_number.sh` → rename `next_mr_number.sh`. Replace
  `gh api repos/... /pulls | jq '.[0].number'` with
  `glab api projects/:id/merge_requests?order_by=iid&sort=desc&per_page=1 | jq '.[0].iid'`.

**Rewrite against GitLab API** (Python, HTTP):
- `_gh.py` → rename `_glab.py`. Wraps GitLab REST API: base URL from
  `git remote get-url origin` (parse `https://gitlab.com/<group>/<repo>`
  or `git@gitlab.com:<group>/<repo>.git`), auth via `GITLAB_TOKEN` env
  or `glab auth status`. Expose `get()`, `post()`, `project_id()`.
- `pull_reviews.py` — fetch all discussions on the current MR via
  `GET /projects/:id/merge_requests/:iid/discussions`, deduplicate by
  discussion id (stable marker comment format:
  `<!-- glab-id: {discussion_id} -->` so re-runs are idempotent),
  append each thread chronologically to `doc/reviews/review-NNNNN.md`.
- `reply_review.py` — post a reply to a specific discussion via
  `POST /projects/:id/merge_requests/:iid/discussions/:discussion_id/notes`,
  mirror the reply into the review doc under the correct thread.

### T8 — Automated MR reviewer (added mid-sprint after MR !2 was opened)

GitLab has no direct equivalent of GitHub's `anthropics/claude-code-action`
for MR reviews; a CI job that calls the Claude API directly and posts
findings as MR discussion notes is the recommended pattern.

- `scripts/claude_review.py` — Python script using the `anthropic`
  SDK. Builds a prompt-cached request (system + CLAUDE.md +
  calibration examples cached; diff not cached), parses the JSON
  finding array, POSTs each to
  `/projects/:id/merge_requests/:iid/discussions` with the
  `position[*]` fields for inline anchoring. Falls back to a
  general-comment reference if GitLab rejects the inline position
  (line not in the diff window, file renamed, etc.).
- `.gitlab-ci.yml` — new `claude-review` job in the `audit` stage.
  Uses `python:3.12-slim`, installs `anthropic`, requires `GIT_DEPTH: 0`
  so the merge-base is available. `allow_failure: true` — advisory
  only.
- Gated by `rules:` on `ANTHROPIC_API_KEY && GITLAB_TOKEN` so the job
  is invisible until both CI variables are wired.
- Default model: `claude-sonnet-4-6`. Override via `CLAUDE_MODEL` env.

## Review

### Deviations from the plan

- **No explicit test of `scripts/autosquash.sh` against a synthetic
  fixup.** The plan listed this as a spot check. It is a non-breaking
  omission (the script is byte-identical to template-rust's, which is
  already exercised upstream). Verified shell syntax via `bash -n`.
  Re-enable by running `/sprint-review` on a branch with a staged
  fixup before the next merge.

- **No live validation of `glab api` calls.** Any path that hits a
  real GitLab project (`next_mr_number.sh`, `pull_reviews.py`,
  `reply_review.py`) was only syntax-checked. First real use will
  exercise the endpoints; expect to tune exact jq paths (`.iid` vs
  `.id`, `source_branch` vs `sourceBranch`) on the first MR.

- **`cargo fmt --check` reports pre-existing formatting drift in
  `src/conn.rs`**. The new pre-commit hook logs this as a warning and
  does not block. Reformatting is deferred to Sprint B (file tree
  reorg), which rewrites `src/` anyway.
