Land the publish-blocker and high-leverage-ergonomics findings from
the four-agent code-health audit, plus reorganise the integration
test tree to mirror `src/` and rename `kani_proofs` → `kani`. Target
artefact: a crate that `cargo publish --dry-run` accepts and renders

## Tasks

### T1 — Fix `#[doc(cfg)]` so docs.rs builds

Problem: `src/lib.rs:193,201` and `src/prop.rs:65` use
`#[cfg_attr(docsrs, doc(cfg(...)))]`. `doc(cfg)` is nightly-only
(`feature(doc_cfg)`). docs.rs builds with `--cfg docsrs` per
`Cargo.toml:111-113`; without the feature gate the build errors out
with E0658.

Solution: add `#![cfg_attr(docsrs, feature(doc_cfg))]` at the top of
`src/lib.rs` (alongside the existing `#![cfg_attr(feature = "f16",
feature(f16))]`). Add `#[cfg_attr(docsrs, doc(cfg(feature = "f16")))]`
mirrors to the `f16`-gated items in `src/core/f032.rs` and
`src/core/f064.rs` so they surface in the rendered docs with the
gating breadcrumb.

### T2 — Plug the 5 macro-export leaks

Problem: `__float_ext_int_ceil_body!`, `__float_ext_int_floor_body!`,
`__float_ext_int_inner_body!` in `src/float.rs` and
`__float_fixed_ceil_body!`, `__float_fixed_floor_body!` in
`src/fixed.rs` are unconditionally `#[macro_export]`. They are
`#[doc(hidden)]`, but `#[macro_export]` lifts them to the crate root
regardless of `pub mod` visibility. This contradicts `src/lib.rs:97-99`
("seven internal saturating-cast macros ship ONLY when `macros` is on").

Solution: replace `#[macro_export]` with
`#[cfg_attr(feature = "macros", macro_export)]` on those five
declarations, matching the parent-macro idiom already used at
`src/float.rs:1551,1603` and `src/fixed.rs:237,289`.

### T4 — Public `Debug` + `PartialEq` on `Conn`

Problem: `src/conn.rs:108` has a manual `Clone`/`Copy` impl pair but
no `Debug` or `PartialEq`. Function pointers support both. Today
`dbg!(c)`, `assert_eq!(c1, c2)`, and `HashMap<_, Conn<_,_>>`-keyed
registries all fail.

Solution: hand-impl `Debug` (prints `Conn<source-tyname → target-
tyname, K>` using `core::any::type_name`) and derive `PartialEq` +
`Eq` via fn-pointer address comparison. Cover with a spot test that
`assert_eq!(C, C)` holds for at least three real Conn consts across
families, and that `format!("{:?}", C)` is non-empty.

### T5 — `connections::prelude`

Problem: `src/lib.rs:223` re-exports `round`/`truncate`/`median`/
`interval` as bare names at the crate root. The neighbouring lint
comment at 215-222 already acknowledges the collision risk under
`use connections::*`. The idiomatic solve is a `prelude` module.

Solution: introduce `pub mod prelude` re-exporting the bare two-sided
helpers + the capability traits (`ConnL`, `ConnR`, `ConnK`). Retain
the existing root re-exports for one release to avoid a Catch-22
discovery story (`use connections::prelude::*` is the new path; the
root surface is deprecated). Note this in the crate doc.

### T6 — Rename `kani_proofs` → `kani`

Problem: per CLAUDE.md the SMT submodule is referenced as
`kani_proofs` in `src/lib.rs:244-245`. The shorter `kani` matches
the tool name and is the more idiomatic module name.

Solution: rename `src/kani_proofs.rs` → `src/kani.rs`,
`src/kani_proofs/` → `src/kani/`, and the `mod kani_proofs;`
declaration. Update CLAUDE.md cross-references.

### T7 — Mirror src layout under tests/

Problem: `tests/` is flat (38 files). User wants subdir mirror of
`src/`'s top-level split (`core/`, `fixed/`). `tests/common/` is the
existing shared-helper carve-out used only by the fixed_*_float_q
tests.

Solution: Cargo finds `tests/<name>.rs` as a binary and treats
`tests/<name>/` as its submodule directory.

- `tests/core.rs` — new aggregator binary; declares `mod iNNN;` for
  each moved file.
- `tests/core/`:
  - `i008.rs` ← `tests/int_i8_galois.rs`
  - `i016.rs` ← `tests/int_i16_galois.rs`
  - … `i128.rs`, `u008.rs` … `u128.rs` (zero-padded).
- `tests/fixed.rs` — new aggregator binary.
- `tests/fixed/`:
  - `helpers.rs` ← `tests/common/mod.rs` content
  - `helpers_f16.rs` ← `tests/common/f16.rs`
  - `u008.rs` ← `tests/fixed_u8_galois.rs` … `u128.rs`
  - `i008_float_q.rs` ← `tests/fixed_i8_float_q_galois.rs`, …
  - `i008_float_q_f16.rs` ← `tests/fixed_i8_float_q_galois_f16.rs`, …
  - `u008_float_q{,_f16}.rs` analogously
  - `nz_iso.rs` ← `tests/fixed_nz_iso_galois.rs`
- `tests/common/` directory deleted.
- `tests/macros_feature_smoke.rs` stays top-level (not core- or
  fixed-specific).
- `tests/compile_fail.rs` + `tests/compile_fail/` stay top-level
  (user instruction).

Within each moved file, update `mod common;` / `mod common_f16;`
declarations and `use common::*` paths to point at the new sibling
module names. Drop the `_galois` suffix from filenames — every test
in this tree is a galois battery, so the qualifier is dead weight.
