# Plan 44 — `hifi/epoch.rs` §3 GNSS scales (Sprint 3 of hifitime integration)

## Goal

Land twelve `hifitime::Epoch` Conns covering the four GNSS time
scales — GPST, GST, BDT, QZSST — as `hifi/epoch.rs` §3, mirroring the
shape established for ETAI / EUTC in Sprint 2 (MR !64).

Each scale ships three Conns:

- `E{S}HDUR` — `Conn<Epoch, HD>` (degenerate iso, `iso!`).
- `E{S}NANO` — `Conn<Extended<Epoch>, i128>` (ConnL, OFDTNANO shape).
- `E{S}F064` — `Conn<F064, Extended<Epoch>>` (ConnL, F064DURN shape).

Reference epochs are exposed by hifitime as `GPST_REF_EPOCH` /
`GST_REF_EPOCH` / `BDT_REF_EPOCH` (with QZSST sharing GPST's). All
four GNSS scales are exact integer-second offsets of TAI, so no
leap-second / iterative algorithmic concerns enter the picture — the
projections compose cleanly with `ETAIHDUR` / `ETAINANO` / `ETAIF064`.

## Tasks

### T1 — Arb strategies (`src/prop/arb.rs`)

Four new public strategies plus a small refactor for the helper that
computes per-scale `(min, max)` nanosecond bounds.

Each `arb_hifi_{gpst,gst,bdt,qzsst}_nanos_in_range()` returns
`impl Strategy<Value = i128>` with the same shape as
`arb_hifi_unix_nanos_in_range`:

- Range `[min_n, max_n]` where:
  - `min_n = HD::MIN.total_nanoseconds()` (unshifted — the per-scale
    `ceil`'s subtraction would underflow HD beyond this).
  - `max_n = HD::MAX.total_nanoseconds() − S_REF_EPOCH.to_tai_duration().total_nanoseconds()`.
- `prop_oneof![1 min_n, 1 min_n+1, 1 max_n-1, 1 max_n, 1 0, 4 in-range_signed_i128]`.
- Boundary partner of `max_n - 1` is `min_n + 1` (matching MR !64
  round-5 fix).

**Decision:** factor the per-scale logic through a single
private helper `arb_hifi_scale_nanos_in_range(offset_ns: i128)` to
avoid four near-identical bodies. The four public strategies are
2-line shims.

QZSST's strategy delegates to GPST's offset; both still get a
distinct public name so the call site documents intent.

// ── EGPSNANO ─────────────────────────────────────────────────────

#[inline]
fn gpst_ref_tai_ns() -> i128 {
    hifitime::GPST_REF_EPOCH.to_tai_duration().total_nanoseconds()
}

#[inline]
fn gpst_min_nanos() -> i128 {
    hd_min_nanos()
}

#[inline]
fn gpst_max_nanos() -> i128 {
    hd_max_nanos() - gpst_ref_tai_ns()
}

fn egpsnano_ceil(e: Extended<Epoch>) -> i128 {
    let max_n = gpst_max_nanos();
    match e {
        Extended::NegInf => i128::MIN,
        Extended::Finite(e) => e.to_gpst_duration().total_nanoseconds(),
        Extended::PosInf => max_n + 1,
    }
}

fn egpsnano_inner(n: i128) -> Extended<Epoch> {
    let min_n = gpst_min_nanos();
    let max_n = gpst_max_nanos();
    if n < min_n {
        Extended::NegInf
    } else if n > max_n {
        Extended::PosInf
    } else {
        let tai_total_ns = n + gpst_ref_tai_ns();
        let tai_dur = HD::from_total_nanoseconds(tai_total_ns);
        Extended::Finite(Epoch::from_tai_duration(tai_dur))
    }
}

crate::conn_l! { /* doc + ceil/inner */ pub EGPSNANO : Extended<Epoch> => i128 { ... } }

### End-to-end scenario

```rust
use connections::hifi::{EGPSF064, EGPSNANO, ETAINANO};
use connections::extended::Extended;
use hifitime::{Epoch, GPST_REF_EPOCH};

// GPST reference epoch is zero on the GPST rung.
assert_eq!(EGPSNANO.ceil(Extended::Finite(GPST_REF_EPOCH)), 0);

// On the TAI rung, it's the +80-year integer-second offset.
let tai_ns = ETAINANO.ceil(Extended::Finite(GPST_REF_EPOCH));
assert_eq!(tai_ns, 2_524_953_619_000_000_000);

// And via f64 GPST seconds, 0.0 round-trips back to GPST_REF_EPOCH.
let zero_f = connections::float::ExtendedFloat::Extend(0.0_f64);
assert_eq!(EGPSF064.ceil(zero_f), Extended::Finite(GPST_REF_EPOCH));
```

## Review (sprint close)

- **BDT had to route through `to_duration_in_time_scale(TimeScale::BDT)`** instead
  of `to_bdt_duration()`. The two are equivalent for interior values
  but the latter is implemented as `self.to_tai_duration() -
  BDT_REF.to_tai_duration()` (`hifitime/src/epoch/mod.rs:800`), which
  routes through TAI and saturates `HD::MAX + BDT_REF` at the upper
  bound — breaking the iso round-trip at `Epoch::from_bdt_duration(HD::MAX)`.
  The other GNSS scales (GPST/QZSST/GST) implement `to_*_duration` as
  `to_time_scale(scale).duration` (which short-circuits for the
  matching tag), so they don't have this asymmetry. Caught by
  `ebdthdur_laws::roundtrip_ceil` proptest at `b = HD::MAX`. Fix
  applied to all four EBDT* Conns; documented inline in the §3.4 banner.
- **All four scales implemented as `iso!` for HDUR**, matching the
  original plan. The hifitime quirk above forced the
  `to_duration_in_time_scale` routing, but BDT remains a true iso
  modulo that workaround.
- **Inner is the simpler form `Epoch::from_{s}_duration(HD::from_total_nanoseconds(n))`**
  for all four scales. The original plan suggested mirroring
  `eutcnano_inner`'s pre-shift, but `from_*_duration` for GNSS
  scales stores the scale-relative duration directly — no anchor
  shift needed (unlike `from_utc_duration` which expects J1900-
  anchored). The simpler form matches the cleaner factoring.
- **GPST/QZSST agreement proptests** (`egpsnano_eq_eqzsnano`,
  `egpshdur_eq_eqzshdur`) confirm that the two scales produce
  identical numeric output on every Epoch input despite distinct
  scale-tag dispatch. The Conns differ only in the tag of the
  output Epoch (when `inner` is invoked) — observable under
  `Debug`, equal under instant-based `Eq`.
- **No `#[ignore]`s introduced.** All hifi::epoch tests green on
  the first run after the BDT fix.
- **f32 GNSS bridges still deferred.** F032E{GPS,QZS,GST,BDT}
  unimplemented; revisit if a downstream needs them.
- **MR iid prediction**: 66.
