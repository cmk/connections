# Plan 01 ‚Äî Convert `ignore` doctests to real, tested examples

## Goal
Eliminate the rendered-docs "This example is not tested" note by turning
the crate's 14 ` ```ignore ` doctest blocks into real, compiled-and-run
doctests wherever feasible; for the handful that genuinely cannot compile
in-crate, drop the false-Rust framing and use an honest ` ```text `
illustrative block instead. No library code changes ‚Äî docs only.

## Motivation

`rustdoc` renders every ` ```ignore ` block with a crossed-circle badge
whose tooltip reads "This example is not tested." We have 14 of them.
Most are real, compilable examples that were marked `ignore` only to
avoid wiring up imports or feature gates. CI and pre-push already run
doctests with `--features fixed,time,hifi,uhlc,proptest,macros` (every
stable feature; only nightly `f16` is excluded), so a real doctest that
touches `fixed` / `proptest` / `time` types is exercised on every push.

**Empirical decision point (Tier B proptest idioms).** `proptest! { #[test]
fn ‚Ä¶ }` and `law_battery! { ‚Ä¶ }` expand to `#[test]` items. A doctest is
wrapped in `fn main()`, so a nested `#[test]` may not compile. During
implementation each is first attempted as a real doctest (matching the
existing `prop.rs:21` `rust,no_run` style); any that will not compile as
a doctest falls back to Tier C ` ```text `. The final per-block outcome
is recorded in the Review section, not guessed here.
