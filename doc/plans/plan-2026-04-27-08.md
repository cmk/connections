# Plan 20 — Pre-publish cleanup: namespace flatten + drop `half`

## Goal

Two cleanups before the first crates.io publish, landing as one MR
(MR !34) with three commits:

1. Drop `bf16` support entirely (no stabilization path in std).
2. Replace `half::f16` with std `f16` behind an opt-in `f16` cargo
   feature (nightly-only when enabled); drop the `half` crate dep.
3. Flatten the public namespace by pulling `src/conn/<sub>` up one
   level and folding `cast.rs` into `conn.rs`. Rename
   `conn::std` → `int` to avoid shadowing the std prelude. Apply
   the RHS-wins placement rule inside `float/`.

## Motivation

- `connections::conn::cast::ceiling` triples the same noun. Users
  write `connections::conn::float::F064` etc. — the `conn::`
  prefix is redundant in every per-domain path. Flattening drops
  one level everywhere except the core `Conn`/operations module.
- `half` adds a build-graph dep solely for two niche float types
  on stable rustc. `bf16` has no RFC; `f16` is nightly-only behind
  `#![feature(f16)]` (tracking #116909). Gating f16 behind a
  cargo feature lets stable users skip both, and lets f16-curious
  users opt in via nightly without dragging the rest of the crate
  off stable.

## Sequencing

Three commits, each green per the "every commit must leave
`cargo test` passing" rule:

```
T1 (drop bf16) → T2 (gate f16 + drop half) → T3 (flatten + rename)
```

T1 and T2 are pure code changes (no path renames); T3 is the
mechanical move plus the cast.rs fold. T3 also picks up the
cast-cleanup work that landed in main (MR !32: drop
maximize/minimize, π-driven doctests, edge-input prose) and
re-applies it to the new `conn.rs` home for the cast helpers.

## Tasks

### T1 — Drop bf16 support

- `src/conn/float.rs`: remove `impl_float_ext!(half::bf16)`,
  `pub type B016`, `shift16_bf16`, `clamp16_bf16`, and bf16 tests.
- `src/conn/float/{f32,f64}.rs`: remove `F032B016` / `F064B016`
  blocks, helpers, and tests.
- `src/property/arb.rs`: remove `arb_bf16`, `extended_float_bf16`.
- Doc references in `src/lib.rs`, `README.md` updated.

### T2 — Gate f16, drop `half`

- `Cargo.toml`: drop `half` from `[dependencies]`; add `f16 = []`
  feature with a nightly-required note.
- `src/lib.rs`: prepend `#![cfg_attr(feature = "f16", feature(f16))]`.
- `src/conn/float.rs`: gate `impl_float_ext!(f16)`, `F016`, the
  16-bit ULP machinery, and tests behind `feature = "f16"`.
- `src/conn/float/{f32,f64}.rs`: gate `F032F016` / `F064F016` and
  their tests; replace `half::f16` with std `f16` (`x as f16`
  casts; `widen_f16_*` helpers for fn-pointer use).
- `src/property/arb.rs`: gate `arb_f16`, `extended_float_f16`.
- Doctests using f16 prepend `# #![feature(f16)]` so they compile
  on nightly.

### T3 — Flatten namespace + rename + apply RHS-wins in `float/`

- `src/conn/std.rs` → `src/int.rs`; `src/conn/std/` → `src/int/`
- `src/conn/fixed.rs` → `src/fixed.rs`; `src/conn/fixed/` →
  `src/fixed/`
- `src/conn/float.rs` → `src/float.rs`; `src/conn/float/` →
  `src/float/`
- `src/conn/time.rs` → `src/time.rs`; `src/conn/time/` →
  `src/time/`

Cast fold:

- The free fns from `src/conn/cast.rs` (upper, ceiling, lower,
  floor, median, round, truncate, interval, midpoint, plus their
  `*1`/`*2` lifters) move into `src/conn.rs` as siblings of the
  `Conn` struct. The `#[cfg(test)] mod tests` blocks merge.
- `cast.rs` is deleted.
- The cast.rs cleanup work from main MR !32 (drop maximize /
  minimize, F064F032+π doctests, "Behavior on edge inputs"
  module section, Tier-1 review fixes) is preserved through the
  fold — the new `conn.rs` reflects main's final cast.rs.

RHS-wins inside `float/`:

- `F064F032` (currently `src/conn/float/f64.rs`) → `src/float/f32.rs`.
- `F032F016` and `F064F016` (currently in `f32.rs` / `f64.rs`)

## Deferred

- `bf16` support is gone for now. If a downstream crate wants
  bfloat16 it can build it directly atop `Conn` + a custom 16-bit
  ULP machinery, just like `agogo` builds the audio sample-rate
  ladder atop this crate.
- `f16` lives behind the cargo feature until std stabilizes it.
  When that happens, drop the `#[cfg(feature = "f16")]` gates
  and the crate-level feature attribute; keep the cargo feature
  as a compatibility no-op for one minor version, then remove it.
- Re-exporting `Conn` at the crate root: not in scope this MR.
  Users still reach for `connections::conn::Conn`. Trivial to add
  later.
