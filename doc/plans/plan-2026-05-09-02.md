# Plan 2026-05-09-02 вҖ” FloatвҶ’Q-format Conns (Phase 2)

## Context

Plan 2026-05-09-01 ("FloatвҶ’Extended<intN> and FloatвҶ’FixedI/U<Frac>
Conns") shipped Phase 1 (24 `FloatвҶ’Extended<intN>` Conns) under MR
!91 and explicitly deferred Phase 2 вҖ” the Q-format target rungs вҖ”

Phase 1 established the macro pattern (`float_ext_int!` /

Two Phase-1 fragments carry forward as commits on this branch:

- **Round-2 fix from MR !91** (already committed as `a54d6e0`):

## Scope decision

Phase 1 of plan-01 estimated 186 Phase-2 Conns (62 per source Г— 3
sources). To keep this MR a manageable review surface and protect
against macro-design slips, this sprint **scopes to T7 (macros) +
T8 (f32 source) + T9 (f64 source) = 124 Conns across 10 host
files**, deferring T10 (f16 source, all composed) to a small
follow-up MR.

Rationale:
- T10 is structurally trivial once T8 and T9 land (every Conn is a
  one-line `compose_k!(F032F016, F032Q<frac>)`); it is feature-gated;
  it adds no new math. Deferring it does not block any downstream
  work and isolates the f16 toolchain risk.
- T7 macro design is the highest-risk piece; better to land f32 +
  f64 first to validate, then add the gated f16 pass-through.

### T2 вҖ” `float_fixed!` macro (full-triple Conns)

Lives in `src/fixed.rs` (next to `ext_int!` вҖ” these macros reach
into fixed-crate types, so the float-side doesn't host them).
Emits:

```rust
pub const $NAME: Conn<ExtendedFloat<$float>, Extended<$Fixed<$Frac>>>
```

via `crate::conn_k!`. Body sketch:

```rust
fn ceil(x: ExtendedFloat<$float>) -> Extended<$Fixed<$Frac>> {
    use ExtendedFloat::*;
    match x {
        Bot => Extended::NegInf,
        Top => Extended::PosInf,
        Extend(v) => $crate::__float_fix_ceil_body!($float, $Fixed, $Frac, v),
    }
}

fn inner(b: Extended<$Fixed<$Frac>>) -> ExtendedFloat<$float> {
    $crate::__float_fix_inner_body!($float, $Fixed, $Frac, b)
}

fn floor(x: ExtendedFloat<$float>) -> Extended<$Fixed<$Frac>> {
    /* mirror of ceil with floor */
}
```

The three `__float_fix_*_body!` private helpers parallel the
Phase-1 `__float_ext_int_*_body!` shape.

#### Ceil/floor body math

The fixed-crate Q-format value `q: $Fixed<$Frac>` has bit-pattern
`q.to_bits()` and real value `q.to_bits() / 2^F` where
`F = <$Frac as Unsigned>::U32`. We compute via **f64 intermediate
precision**, regardless of source-float type, because:

- f64 has 53 mantissa bits (fits any host `bits` up to 53 exactly,
  bigger hosts via the round-down helpers); 11-bit exponent covers
  every `2^F` up to F=1023.
- f32 sources cannot represent `2^128` (saturates to +вҲһ), so a
  same-precision-as-source intermediate breaks the i128/u128 +
  F=128 case. Routing through f64 sidesteps this without per-host
  branching.

Ceil body:

```rust
// (Where $Bits is the host integer storage вҖ” i8/u8/.../i128/u128.)
if $v.is_nan() { return PosInf; }
if $v == <$float>::INFINITY { return PosInf; }
let v_f64: f64 = $v as f64;  // exact upcast (mantissa f32=24, f16=11 вүӨ 53)
let max_q = <$Fixed<$Frac>>::MAX;
let min_q = <$Fixed<$Frac>>::MIN;
let max_f: f64 = max_q.to_num::<f64>();
let min_f: f64 = min_q.to_num::<f64>();
if v_f64 > max_f { return PosInf; }
if v_f64 < min_f { return Finite(min_q); }
let scale: f64 = scale_pow2_f64(<$Frac as Unsigned>::U32);  // 2^F
let scaled: f64 = v_f64 * scale;
let bits: $Bits = scaled.ceil() as $Bits;  // Rust's saturating cast
Finite(<$Fixed<$Frac>>::from_bits(bits))
```

Floor body mirrors with `floor()` and `if v.is_nan() => NegInf`.

`scale_pow2_f64(F)` is a new const `pub(crate) fn` in `src/float.rs`
that returns `2.0_f64.powi(F)` via direct bit-pattern construction
(not `powi`, which isn't const). For F вүӨ 1023 (always true here):

```rust
pub(crate) const fn scale_pow2_f64(frac: u32) -> f64 {
    // Bias-1023 exponent for normal f64.
    // 2^0 has biased exponent 1023; 2^F has biased exponent 1023 + F.
    let biased = (1023 + frac as u64) << 52;
    f64::from_bits(biased)
}
```

#### Inner body

```rust
match $b {
    Extended::NegInf  => ExtendedFloat::Bot,
    Extended::PosInf  => ExtendedFloat::Top,
    Extended::Finite(q) => {
        let i: i128 = q.to_bits() as i128;
        // Round-down i to f64: exact for i вүӨ 53 bits, otherwise
        // returns the largest f64 вүӨ i (existing helper).
        let approx_f64: f64 = $crate::float::round_down_to_f64(i);
        let scale_neg: f64 = scale_pow2_f64_neg(<$Frac as Unsigned>::U32);
        let value_f64: f64 = approx_f64 * scale_neg;
        // Downcast to source float with round-toward-negative-infinity.
        // For f64 source, identity. For f32/f16, dispatch via a new
        // `RoundDownFromF64` sealed trait on the source float type.
        let value: $float = <$float as $crate::float::RoundDownFromF64>::from_f64_rd(value_f64);
        ExtendedFloat::Extend(value)
    }
}
```

Two new helpers:
- `scale_pow2_f64_neg(F: u32) -> f64` вҖ” returns `2^-F`, normal for
  F вүӨ 1022 and denormal for F in [1023, 1074]. We only need F вүӨ 128.
  Bit-pattern: `(1023 - F as u64) << 52` for F вүӨ 1022.
- `RoundDownFromF64` вҖ” sealed trait paralleling `RoundDownFromI128`:

```rust
#[doc(hidden)]
pub trait RoundDownFromF64: private::Sealed + Sized {
    fn from_f64_rd(v: f64) -> Self;
}
impl RoundDownFromF64 for f64 {
    fn from_f64_rd(v: f64) -> Self { v }  // identity
}
impl RoundDownFromF64 for f32 {
    fn from_f64_rd(v: f64) -> Self { round_down_f64_to_f32(v) }
}
#[cfg(feature = "f16")]
impl RoundDownFromF64 for f16 { вҖҰ }
```

`round_down_f64_to_f32(v: f64) -> f32` follows the same pattern as
the existing `round_down_to_f32(b: i128) -> f32` but operates on
the f64 value: cast to f32, check direction with the upcast f64 of
the result, shift32(-1, вҖҰ) if it overshot. Symmetric helpers ship
for f16 (gated).

### Design deviations from the plan

- **`BitsToF64Rd` sealed trait was not in the plan** вҖ” added during
  T9 rollout when the u128 host's L-only law battery surfaced a
  Galois failure at `Finite(u128::MAX)`. The naive `q.to_bits() as
  i128` path (planned in T7's `__float_fix_inner_body!` sketch)
  wraps the high bit for `u128` storage values above `i128::MAX`,
  producing negative f64s where the contract demands positive. The
  trait dispatches per host bit-width: i8/u8/i16/u16/i32/u32 use
  `as f64` (exact); i64/u64/i128 route through the existing i128
  helper; u128 routes through the new `round_down_to_f64_u128`
  which special-cases `2^128` (the f64 image of `u128::MAX`).

- **`RoundDownFromF64` sealed trait** вҖ” landed as planned, parallel
  to Phase 1's `RoundDownFromI128`. f64-source impl is the
  identity; f32 / f16 impls go through new
  `round_down_f64_to_f32` / `_f16` helpers.

- **`scale_pow2_f64` / `_neg` const helpers** вҖ” landed as planned
  via direct bit-pattern construction (`f64::from_bits` with
  bias-1023 exponent encoding). Both exact for our `frac вүӨ 128`.

- **Q-format arb generator pattern** вҖ” the plan suggested per-host
  named strategies in `src/prop/arb.rs`. In practice the inline
  `props_for_float_q!` macro per submodule is more compact; the
  arb is keyed on the host bit-width and frac type, not split out
  as a free function. No central `arb_fixed_iN_bits` was needed.
