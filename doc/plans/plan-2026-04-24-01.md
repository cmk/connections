### U1 — `f32fix` / `f64fix` / `ratfix`

New module contents in `src/fixed.rs`. Reference:
`/Users/cmk/Music/Software/Haskell/connections/src/Data/Connection/Fixed.hs`
lines 160–187.

#### Types

```rust
/// Extended IEEE float: finite values plus ±∞ and NaN, ordered
/// with `NegInf < Finite < PosInf` and NaN equal to neither.
/// Mirrors Haskell's `Extended (Fixed e)` over the float side so
/// adjoint laws can be stated over a complete order.
#[derive(Copy, Clone, Debug, PartialEq)]
pub enum Extended<T> {
    NegInf,
    Finite(T),
    PosInf,
    Nan,
}
```

#### Conns

One `Conn` const per (float-side × decimal-rung) pair:

- **Extended variants**, NaN-and-∞-aware:
  `F32F00` through `F32F12`: `Conn<Extended<f32>, Uni|Deci|...|Pico>`
  `F64F00` through `F64F12`: `Conn<Extended<f64>, ..>`
- **Saturating variants** for RT paths:
  `F32F00_SAT` .. `F32F12_SAT`: `Conn<f32, ..>` — NaN → `T::ZERO`,
  ±∞ → `T::MIN`/`T::MAX`, clamped finite values.
  `F64F00_SAT` .. `F64F12_SAT`: same for f64.
- **Rational variants**:
  `RATF00` .. `RATF12`: `Conn<Rational64, ..>` (exact, no Extended
  wrapping because Rational has no NaN).

Total: 7 rungs × (Extended_f32 + SAT_f32 + Extended_f64 + SAT_f64 +
RAT) = 35 new Conn constants. Cheap — each one is ~3 fn pointers.

Core implementation: `fn fixfix_float(n: f64) -> Milli` already
lives as a free fn inside each const body (same `fix_fix!` macro
shape used for the existing decimal ladder). Rational variant ports
Haskell's `ratfix` directly: reduce → multiply by `toRational prec`
→ `div` / `div + 1` depending on direction.

#### Proptests

Per pair:

- `adjoint_upper`: `ceil(x) ≤ b ⟺ x ≤ inner(b)`
- `adjoint_lower`: `inner(b) ≤ x ⟺ b ≤ floor(x)`
- `ceil_monotone`, `floor_monotone`
- `floor_le_ceil`
- `extended_roundtrip`: for Extended variants, `NegInf` / `PosInf`
  round-trip cleanly; `Nan` → `T::MIN` per Haskell convention.
- `sat_vs_extended_agree`: saturating and Extended variants give
  the same answer on finite non-overflowing inputs.

Total: ~6 props × 35 conns ≈ 210 new tests. Upstream grows from
343 → ~553. Build time acceptable for proptest (200 tests with
default config = seconds, not minutes).

### U2 — Crate-level doc legend

`src/lib.rs` gets a module-level doc comment spelling out the
convention:

```rust
//! ## Naming
//!
//! Adjoint-triple constants are named `fXYfZW` where `XY` and
//! `ZW` are the decimal-exponent codes of the two rungs:
//!
//! | code | type   | resolution       |
//! |------|--------|------------------|
//! | F00  | Uni    | 10⁰  s           |
//! | F01  | Deci   | 10⁻¹ s           |
//! | F02  | Centi  | 10⁻² s           |
//! | F03  | Milli  | 10⁻³ s           |
//! | F06  | Micro  | 10⁻⁶ s           |
//! | F09  | Nano   | 10⁻⁹ s           |
//! | F12  | Pico   | 10⁻¹² s          |
//! | F32  | f32    | IEEE single      |
//! | F64  | f64    | IEEE double      |
//! | RAT  | Ratio  | exact rational   |
//!
//! Cross-tier to the sample-rate tier uses `SN` suffixes (`S44`,
//! `S48`, `S88`, `S96`, `S176`, `S192`). Rate-pair conns
//! concatenate two `S`-codes (`S88S44` = S88 → S44). Saturating
//! variants of float conns append `_SAT`; the non-suffix form
//! takes `Extended<T>` to handle NaN/±∞ explicitly.
```
