## Goal

Two related additions, shipping in one branch:

1. **U0 — Rename the sample-tier conns** to the Haskell `fXYfZW`
   6-char shape. Today's `PICO_S48`, `S88_S44`, etc. are the only
   non-conforming names in the ladder.

### U1 — `f32fix` / `f64fix` / `ratfix`

New module contents in `src/fixed.rs`. Reference:
`/Users/cmk/Music/Software/Haskell/connections/src/Data/Connection/Fixed.hs`
lines 160–187.

#### Types

```rust
/// Extended IEEE float: finite values plus ±∞ and NaN, ordered
/// with `NegInf < Finite < PosInf` and NaN equal to neither.
/// Mirrors Haskell's `Extended (Fixed e)` over the float side so
/// adjoint laws can be stated over a complete order.
#[derive(Copy, Clone, Debug, PartialEq)]
pub enum Extended<T> {
    NegInf,
    Finite(T),
    PosInf,
    Nan,
}
```

#### Conns

One `Conn` const per (float-side × decimal-rung) pair:

- **Extended variants**, NaN-and-∞-aware:
  `F32F00` through `F32F12`: `Conn<Extended<f32>, Uni|Deci|...|Pico>`
  `F64F00` through `F64F12`: `Conn<Extended<f64>, ..>`
- **Saturating variants** for RT paths:
  `F32F00_SAT` .. `F32F12_SAT`: `Conn<f32, ..>` — NaN → `T::ZERO`,
  ±∞ → `T::MIN`/`T::MAX`, clamped finite values.
  `F64F00_SAT` .. `F64F12_SAT`: same for f64.
- **Rational variants**:
  `RATF00` .. `RATF12`: `Conn<Rational64, ..>` (exact, no Extended
  wrapping because Rational has no NaN).

Total: 7 rungs × (Extended_f32 + SAT_f32 + Extended_f64 + SAT_f64 +
RAT) = 35 new Conn constants. Cheap — each one is ~3 fn pointers.

Core implementation: `fn fixfix_float(n: f64) -> Milli` already
lives as a free fn inside each const body (same `fix_fix!` macro
shape used for the existing decimal ladder). Rational variant ports
Haskell's `ratfix` directly: reduce → multiply by `toRational prec`
→ `div` / `div + 1` depending on direction.

#### Proptests

Per pair:

- `adjoint_upper`: `ceil(x) ≤ b ⟺ x ≤ inner(b)`
- `adjoint_lower`: `inner(b) ≤ x ⟺ b ≤ floor(x)`
- `ceil_monotone`, `floor_monotone`
- `floor_le_ceil`
- `extended_roundtrip`: for Extended variants, `NegInf` / `PosInf`
  round-trip cleanly; `Nan` → `T::MIN` per Haskell convention.
- `sat_vs_extended_agree`: saturating and Extended variants give
  the same answer on finite non-overflowing inputs.

Total: ~6 props × 35 conns ≈ 210 new tests. Upstream grows from
343 → ~553. Build time acceptable for proptest (200 tests with
default config = seconds, not minutes).

### U2 — Crate-level doc legend

`src/lib.rs` gets a module-level doc comment spelling out the
convention:

```rust
//! ## Naming
//!
//! Adjoint-triple constants are named `fXYfZW` where `XY` and
//! `ZW` are the decimal-exponent codes of the two rungs:
//!
//! | code | type   | resolution       |
//! |------|--------|------------------|
//! | F00  | Uni    | 10⁰  s           |
//! | F01  | Deci   | 10⁻¹ s           |
//! | F02  | Centi  | 10⁻² s           |
//! | F03  | Milli  | 10⁻³ s           |
//! | F06  | Micro  | 10⁻⁶ s           |
//! | F09  | Nano   | 10⁻⁹ s           |
//! | F12  | Pico   | 10⁻¹² s          |
//! | F32  | f32    | IEEE single      |
//! | F64  | f64    | IEEE double      |
//! | RAT  | Ratio  | exact rational   |
//!
//! Cross-tier to the sample-rate tier uses `SN` suffixes (`S44`,
//! `S48`, `S88`, `S96`, `S176`, `S192`). Rate-pair conns
//! concatenate two `S`-codes (`S88S44` = S88 → S44). Saturating
//! variants of float conns append `_SAT`; the non-suffix form
//! takes `Extended<T>` to handle NaN/±∞ explicitly.
```

## Deferred

- **`Conn::then` composition primitive** — still deferred until a
  closure-capturing variant design is settled (bare `fn`-pointer
  `Conn` can't store the two input Conns needed for composition).
  Tracked in `doc/plans/plan-2026-04-23-01.md` Deferred §.
- **`F32Fxx` float-source conns.** An `inner` that narrows i64 →
  f32 can collapse up to ~120 000 consecutive Rung values onto the
  same f32 (at Pico scale). The adjoint law still holds, but
  `ceil` / `floor` walk the full collapse class per call — the
  proptest domain turns into a multi-minute shrink. f32 callers
  widen losslessly at the boundary
  (`F64F06.ceil(FloatExt::Finite(arg_f32 as f64))`) and get the
  same adjoint answer; if a later need justifies the extra Conn
  constants, they should use binary-search correction rather than
  the linear walk.
- **`RATFxx` rational-source conns.** No current downstream
  consumer; deferred to the first sprint that needs exact
  rational → rung arithmetic. The Haskell `ratfix` is the
  reference; the wrapper would be `Conn<Rational64, Extended<Rung>>`
  (source is totally ordered and has no NaN, so no `FloatExt`
  wrapping on the source side).
- **`Extended<T>` generalisation beyond the decimal ladder.** The
  type is library-wide but only exercised by the new float conns.
  Any connection whose target cannot represent the source's full
  range (e.g. a future `f64 → Extended<u8>`) should consume the
  same type.

## Review

### Design deviations

- **`Extended<T>` lacks a `Nan` variant.** An early design had
  one, motivated by "ceil of NaN needs somewhere to land." The
  correct answer per `doc/design.md` is to put NaN handling on
  the *source* side via `FloatExt<T>` (NaN is reflexive under its
  `PartialOrd`, and maps to `PosInf_target` / `NegInf_target`
  under ceil / floor). Target `Extended` stays a pure
  range-extension wrapper.
- **F32Fxx dropped from U1.** See Deferred §. The in-Rust
  `Conn<FloatExt<f32>, Extended<Rung>>` is lawful in principle,
  but the linear-walk correction loop needed to reach the
  smallest lawful `c` in an f32 collapse class makes the conn
  uselessly slow (multi-minute proptest shrinks). Until a
  binary-search correction lands, `F64Fxx` with callers widening
  f32 → f64 at the boundary is the shipped path.
- **Proptest scope tightened to F64F06 and F64F12.** The original
  plan covered all 7 rungs × multiple variants ≈ 210 tests. The
  shipped coverage is 2 rungs × 7 properties = 14 proptests
  (plus the 14 existing `FxxFyy` rung-pair suites, unchanged).
  The two chosen rungs are structurally distinct (PREC ≪
  mantissa vs PREC near mantissa); the macro expansion guarantees
  uniform coverage within each.

### Recommendations

- When the downstream `agogo` adopts `F64Fxx`, add one compile-
  time example per rung to `doc/design.md` so argv-boundary users
  see the `f32 as f64` widening pattern and don't reach for a
  non-existent `F32F06`.
