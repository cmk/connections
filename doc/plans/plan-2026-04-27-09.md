# Plan 21 — `property` → `prop` cleanup

## Goal

Tighten the property-test scaffolding before publish. Four
mechanical changes, no behavioral change:

1. Rename `property` module to `prop` (one-syllable, less noise).
2. Split `prop::laws` into `prop::conn` and `prop::lattice` —
   ~96 predicates currently live in one 700-line file; the
   conn-vs-lattice split is the natural fault line.
3. Rename `cast_foo` → `conn_foo` (the `cast_*` prefix is a
   leftover from the now-defunct `conn::cast` submodule; with cast
   folded into `conn`, every predicate that exercises a `Conn` is
   simply `conn_*`).
4. Rename `symmetric_foo` → `biheyting_foo` (the trait is
   `Symmetric` but the laws characterize bi-Heyting algebras; the
   existing `biheyting_neg_le_coneg` already uses the right
   prefix).

Plus a fifth sub-task that's not really a rename:

5. Alphabetize all predicate definitions within `prop::conn` and
   `prop::lattice`. Today the file uses ad-hoc section dividers
   (Heyting / Coheyting / Bi-Heyting / Symmetric / Boolean / …)
   that don't survive alphabetization; drop them.

## Sequencing

Four commits, each green. The split + alphabetize go in different
commits to keep the diff readable — the rename moves predicates
into different modules (lots of path changes); alphabetizing within
those modules is a pure reorder that's easier to review separately.

```
T1 (property → prop) → T2 (split laws → conn + lattice) → T3 (cast_/symmetric_ renames) → T4 (alphabetize)
```

T1 is a pure module rename: every callsite swaps `property` for
`prop`. T2 splits the contents of `prop::laws` into `prop::conn` +
`prop::lattice` and updates callers (`laws::xxx` → `conn::xxx` or
`lattice::xxx`). T3 renames the predicates and propagates to all
test sites. T4 reorders the definitions within the two modules
alphabetically.

## Tasks

### T1 — Rename `property` → `prop`

- `git mv src/property.rs src/prop.rs`
- `git mv src/property src/prop`
- `src/lib.rs`: `pub mod property;` → `pub mod prop;`.
- All callers under `src/` and `tests/`: `crate::property::*` →
  `crate::prop::*`, `connections::property::*` →
  `connections::prop::*` (doctests).
- `Cargo.toml`: nothing to change (`testing` feature gates
  `prop::arb`, gating mechanism unchanged).
- `[package.metadata.docs.rs]` comment block already mentions
  `property::arb` — update to `prop::arb`.

### T2 — Split `prop::laws` → `prop::conn` + `prop::lattice`

Predicate split:

| Predicate prefix    | New module      |
|---------------------|-----------------|
| `heyting_*`         | `prop::lattice` |
| `coheyting_*`       | `prop::lattice` |
| `biheyting_*`       | `prop::lattice` |
| `symmetric_*`       | `prop::lattice` (renamed in T3) |
| `boolean_*`         | `prop::lattice` |
| `conn_*` (existing) | `prop::conn`    |
| `cast_*`            | `prop::conn` (renamed in T3) |

- `src/prop/laws.rs` is split into `src/prop/conn.rs` and
  `src/prop/lattice.rs`. Each new file gets its own module-level
  doc, focused on its own predicate family.
- `src/prop.rs`: replace `pub mod laws;` with `pub mod conn; pub mod
  lattice;`.
- Callers update `prop::laws::xxx` → `prop::conn::xxx` or
  `prop::lattice::xxx` based on prefix. Most callers pull a single
  family; a few will need imports from both submodules.

### T3 — Rename `cast_*` → `conn_*` and `symmetric_*` → `biheyting_*`

- Inside `prop::conn`: every `pub fn cast_*` becomes `pub fn
  conn_*` (16-ish names: upper1/lower1/ceiling1/floor1/upper2/
  lower2/ceiling2/floor2_id_*, interval/midpoint/round/truncate
  variants, median variants).
- Inside `prop::lattice`: every `pub fn symmetric_*` becomes `pub
  fn biheyting_*` (13 names).
- Verify no name collisions with the existing `conn_*` /
  `biheyting_*` predicates (spot-checked: none — the existing
  `conn_galois_l` etc. use different stems than the ex-`cast_*`,
  and `biheyting_neg_le_coneg` doesn't collide with the renamed
  `biheyting_involution` etc.).
- All callers in `src/` and `tests/`: search-replace
  `cast_(\w+)` → `conn_$1` and `symmetric_(\w+)` →
  `biheyting_$1`.

### T4 — Alphabetize predicate definitions

- Sort all `pub fn` definitions in `prop::conn` alphabetically by
  name; drop the section dividers.
- Same in `prop::lattice`.
- Module-level doc blocks at the top stay (they're per-module
  context, not per-section).
