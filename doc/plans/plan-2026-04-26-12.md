# Plan 13 тАФ Unsigned binary-fixed Conns + flip fmt to blocking

## Goal

Add five unsigned binary-fixed-point modules (`conn::fixed::u08` /
`u16` / `u32` / `u64` / `u128`) mirroring the existing signed
ladder, with frac-level sets extended to include the canonical
Q1.N formats (Q1.7 MIDI velocity, Q2.14 14-bit MIDI, Q1.15 audio,
Q1.31 / Q1.63 / Q1.127 normalised amplitudes). Preface the work
with a CI-hardening commit flipping `fmt` from advisory to
blocking, now that main is provably fmt-clean.

## Locked design decisions

### D1. Frac-level sets per backing (option B+ from the Q&A)

| Module | Levels                                              | Count | Conns |
|--------|-----------------------------------------------------|-------|-------|
| u08    | `{U0, U1, U2, U3, U4, U6, U7, U8}`                   | 8     | 28    |
| u16    | `{U0, U2, U4, U8, U12, U14, U15, U16}`               | 8     | 28    |
| u32    | `{U0, U4, U8, U16, U24, U31, U32}`                   | 7     | 21    |
| u64    | `{U0, U8, U16, U32, U48, U63, U64}`                  | 7     | 21    |
| u128   | `{U0, U16, U32, U64, U96, U127, U128}`               | 7     | 21    |
| **Total** |                                                  |       | **119** |

Each module covers the same levels as its signed counterpart and
adds one (u32 / u64 / u128) or two (u08, u16) Q1.N levels for
unsigned-specific use cases.

### D2. Macro shape mirrors signed, no negative-saturation logic

Each module ships a `fix_fix_u<width>!` macro structurally
identical to the signed equivalent. The signed boundary fixup
`if x.to_bits() == FINE_MIN { return Coarse::MIN; }` drops out тАФ
unsigned has no negative range, so only the upper-end saturation
is required. The `floor`-side `FINE_MAX` boundary fixup is kept
(needed at the saturation plateau for the Galois law).

`u08`/`u16`/`u32`/`u64` widen through the next-larger unsigned
(`u16`/`u32`/`u64`/`u128`). `u128` follows the i128 precedent тАФ
`1_u128.checked_shl(SHIFT)` detects the SHIFT=128 endpoint and
`u128::checked_mul + saturate` handles overflow in `inner`.

### D3. Type aliases mirror signed where they don't collide

Each module exposes 3-digit zero-padded `pub type U<frac>` aliases
matching the signed convention. The `u128` module is the
exception: the typenum imports `extra::U127` and `extra::U128`
collide with the alias names, so `u128.rs` ships *no* type aliases
and downstream callers spell `FixedU128<U127>` directly. This
asymmetry is documented in the module docstring.

### D4. Cross-sign Conns deferred

`I??U??` / `U??I??` between FixedI<W> and FixedU<W> values are out
of scope. Plan 09's Deferred section already lists this; carry
forward.

### D5. CI: `fmt` blocking

`.gitlab-ci.yml` drops `allow_failure: true` from the `fmt` job.
With main provably fmt-clean (post-MR !14's rustfmt-skip pin and
post-MR !18's forward-port), `allow_failure: true` is now a footgun
that lets formatter drift slip in. Make it a hard gate.
