# Plan 21 — `ExtendedFloat<f32|f64>` numerical-trait expansion + Newton-step doctests

## Goal

Two coupled changes:

1. **Expand `ExtendedFloat<f32|f64>` numeric surface.** Add `Mul`,
   `Neg`, `Rem` (matching the existing `Add`/`Sub`/`Div`/`From<u8>`
   macro), the `*Assign` siblings, plus inherent transcendental
   methods (`sin`/`cos`/`tan`/`sqrt`/`exp`/`ln`/…), constants, and
   predicates. The minimal surface that landed with MR !32 was
   "just enough to thread the chain combinators." The new surface
   is "enough that user closures in `round1`/`truncate1`/etc. can
   invoke standard math directly without unwrapping."

2. **Doctest pass.** Replace every loose `assert!(... >= ...)` in
   `src/conn.rs` with a tight `assert_eq!` against a concrete
   illustrative value. Add new Newton-step doctests for `round1`
   plus a shared variant for `truncate1`/`floor1`/`ceiling1` that
   exercises the new arithmetic surface end-to-end. Lift the
   Haskell `Cast.hs` laws into rustdoc `# Laws` sections, and add
   the median NaN-as-N5-lub example. Bind `pi32_err = pi32 - pi64`
   once at the top of the π-related doctests so the f32 rounding
   error reads as a *named* error rather than an opaque literal.

The parallel agent on `connections-prepublish-cleanup` is unaffected.

## Tasks

### T1 — `Mul` / `Neg` / `Rem` + `*Assign` siblings

Extend `impl_float_arith!` at `src/float.rs:137` so each invocation
also produces `Mul`, `Neg`, `Rem`, plus `AddAssign`, `SubAssign`,
`MulAssign`, `DivAssign`, `RemAssign`. Bot/Top arms:

| Op | (Bot, Bot) | (Top, Top) | mixed | endpoint × Extend(_) | Extend × Extend |
|----|-----------|-----------|-------|----------------------|-----------------|
| `Mul` | conservative `Extend(NaN)` | `Extend(NaN)` | `Extend(NaN)` | `Extend(NaN)` | `Extend(a * b)` |
| `Neg` | `Top` | `Bot` | — | — | `Extend(-a)` |
| `Rem` | `Extend(NaN)` | `Extend(NaN)` | `Extend(NaN)` | `Extend(NaN)` | `Extend(a % b)` |

Same conservative simplification rule as `Div`. `*Assign` impls
delegate: `*self = (*self) op other`.

Property tests in `float.rs::tests`:
- `mul_extends_passes_through_f64` (proptest)
- `neg_double_is_id_f64` (proptest)
- `neg_endpoints_flip` (deterministic — `-Bot == Top`, `-Top == Bot`)
- `rem_zero_pin_f64` (deterministic — `Extend(a) % Extend(0) == NaN`)
- `add_assign_matches_add_f64`, `sub_assign_matches_sub_f64`,
  `mul_assign_matches_mul_f64`, `div_assign_matches_div_f64`,
  `rem_assign_matches_rem_f64` (proptest)

### T2 — Inherent transcendentals + constants + predicates

Add an `impl ExtendedFloat<T>` block (per-T via the macro) carrying
the methods that f32/f64 already ship. Bot/Top dispatch:

| Method | Bot | Top |
|--------|-----|-----|
| `abs` | `Top` | `Top` |
| `signum` | `Extend(-1.0)` | `Extend(1.0)` |
| `recip` | `Extend(0.0)` | `Extend(0.0)` |
| `sqrt` | `Extend(NaN)` | `Top` |
| `cbrt` | `Bot` | `Top` |
| `sin`, `cos`, `tan`, `asin`, `acos` | `Extend(NaN)` | `Extend(NaN)` |
| `atan` | `Extend(-PI/2)` | `Extend(PI/2)` |
| `atan2(y, x)` | dispatch each operand | dispatch | (route through inner T's atan2 if both Extend) |
| `exp`, `exp2` | `Extend(0.0)` | `Top` |
| `ln`, `log2`, `log10` | `Extend(NaN)` | `Top` |
| `powi(self, n: i32)` | `Top` (n even +) / `Bot` (n odd +) / `Extend(0.0)` (n < 0) | `Top` |
| `powf(self, p)` | dispatch on (self, p) | dispatch | (route through inner if all Extend) |
| `mul_add(self, a, b)` | dispatch | dispatch | route through if all Extend |

Constants on `ExtendedFloat<f32>` / `ExtendedFloat<f64>`:

```rust

pub const TAU: Self = Extend(<T>::consts::TAU);
pub const E: Self = Extend(<T>::consts::E);
pub const FRAC_PI_2: Self = Extend(<T>::consts::FRAC_PI_2);
pub const NAN: Self = Extend(<T>::NAN);
pub const INFINITY: Self = Top;

```

Predicates:


### T4 — `round1`/`truncate1`/`floor1`/`ceiling1` doctests

`round1`: Newton step on sin near π via `|a| a - a.tan()`.

Shared variant for `truncate1`/`floor1`/`ceiling1`: closure
`|a| Extend(2.0_f64) * Extend(std::f64::consts::PI) - a` produces a
positive value strictly between f32's floor of π and ceiling of π,
so the three rounding modes pick observably different f32 endpoints.

| Lifter | Expected (probe-confirmed) |
|--------|----------------------------|
| `round1` | `Extend(std::f32::consts::PI)` |
| `truncate1` | `Extend(3.1415925_f32)` |
| `floor1` | `Extend(3.1415925_f32)` |
| `ceiling1` | `Extend(std::f32::consts::PI)` |

### T6 — Lift Haskell `Cast.hs` laws + median NaN example

Add `# Laws` block-quoted rustdoc sections to: `upper1`, `ceiling`,
`ceiling1`, `lower1`, `floor`, `floor1`, `round`, `round1`,
`round2`, `truncate`, `truncate1`, `truncate2`, `median`. Each cite
the corresponding `property::laws` predicate where one exists.

Add `median` NaN example using ExtendedFloat<f32> with N5-aware
lub/glb. The doctest binds `nan = Extend(0.0_f32 / 0.0_f32)` once
and exercises both finite-only and NaN-bearing cases. `9.0` result
for `(1.0, 9.0, NaN)` follows from N5: the lub of the incomparable
`(NaN, _)` pair escalates to `Top`, and the median formula reduces
to the median of the two finite arguments.

### T8 — Tighten cast doctests + `pi32_err` binding refactor

Audit `src/conn.rs` for:
- `assert!(... [<>]= ...)` — replace with `assert_eq!`
- `assert!(... || ...)` — replace
- Tautological `assert_eq!(f(c, x), c.method(x))` — replace with
  illustrative literals
- Prose claiming widening "gains" precision or "recovers digits"

Bind `pi32_err = pi32 - pi64` at the top of `upper`, `ceiling`,
`round` (and similar f32-π doctests) and assert through it. The
same pi32_err literal recurs across multiple doctests so the
reader internalizes "this 8.7e-8 number is f32's rounding error
for π."

### T7 — `cast_median_associative` property predicate

Add to `src/property/laws.rs`:

```rust
pub fn cast_median_associative<A: Copy + Eq>(
    c: &Conn<(A, A), A>, w: A, x: A, y: A, z: A,
) -> bool {
    let lhs = median(c, median(c, x, w, y), w, z);
    let rhs = median(c, x, w, median(c, y, w, z));
    lhs == rhs
}
```

Plus one proptest in `conn.rs`'s test mod over `ORDERED_PAIR`.
