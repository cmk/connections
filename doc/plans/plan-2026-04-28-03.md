# Plan 25 — Fold `int/` and `NonZero` into `fixed/`

## Goal

Recognize that every `int/` Conn is a Q*N*.0 fixed-point Conn, and
that `NonZero<T>` is a punctured Q*N*.0 fixed-point. Fold both into
the existing `src/fixed/` submodule — file-by-file, mechanical merge
— and add the cross-crate iso Conns that the merge unlocks
(`FixedI8<U0> ↔ i8`, …). The float/fixed split is the only top-level
domain distinction.

While we have all the `fixed/` files open for the merge, **rename the
existing intra-fixed Conn prefixes from `I`/`U` to `Q`** (Q-format
wrappers). After the rename, the prefix letter alone disambiguates
"Fixed wrapper" (`Q`) from "std primitive" (`I`/`U`); the host type's
sign and bit-width come from the module path. This kills the
right-side digit-disambiguation rule that would otherwise be
required after the merge.

The crate is unpublished, so `connections::int::*` path changes
and the `I*I*` → `Q*Q*` rename are refactors — no version bump (and
no need for deprecation re-exports). T5 also resets `Cargo.toml` to
`0.0.0` to reflect the pre-publish state; the first crates.io release
will be `0.0.1`.

### Naming convention (post-rename)

| Prefix | Shape | World                                | Source of host bit-width / sign         |
|--------|-------|--------------------------------------|-----------------------------------------|
| `Q`    | 1L+3D | `FixedI<N>` / `FixedU<N>` Q-format    | module path (`fixed::i8` → signed 8-bit, `fixed::u8` → unsigned 8-bit). Digits = frac bits. |
| `I`    | 1L+3D | signed std primitive (`iN`)           | digits = bit-width                      |
| `U`    | 1L+3D | unsigned std primitive (`uN`)         | digits = bit-width                      |
| `N`    | 1L+3D | `NonZero<iN>` / `NonZero<uN>`         | module path (signed if in `fixed::iN`). Digits = bit-width. |

Cross-module name collisions (e.g., `Q008Q004` in both `fixed::i8`
and `fixed::u8`) are allowed and resolved by qualified import, per
existing CLAUDE.md rule.

### T0 — `Conn::new_iso` constructor

Add to `src/conn.rs`:

```rust
impl<A, B> Conn<A, B> {
    /// Construct an iso Conn where floor = ceil (degenerate Galois;
    /// the connection is its own adjoint). Both directions must be
    /// total bijections on the representable range.
    pub const fn new_iso(forward: fn(A) -> B, back: fn(B) -> A) -> Self {
        Self::new(forward, back, forward)
    }
}
```

(Exact signature depends on the existing `Conn::new` shape — audit
`src/conn.rs` and match it. If `Conn::new` is generic over `Fn`
closures rather than `fn` pointers, mirror that; if it stores
function values rather than pointers, this constructor stores them
the same way.)

The 10 cross-crate iso constants in T4 use this directly. The 20
existing fixed-point iso Conns in `src/fixed/*.rs` (where floor =
ceil today) can be migrated to `new_iso` opportunistically — but
that is **not** part of this sprint's scope; tracked as deferred.

Doctest on `new_iso` itself:

```rust
let id: Conn<i32, i32> = Conn::new_iso(|x| x, |x| x);
assert_eq!(id.outer(42), 42);
assert_eq!(id.inner(42), 42);
```

### T3 — NonZero family

Ten new `pub const`s, one per primitive:

- Signed (5): `N008I008`, `N016I016`, `N032I032`, `N064I064`,
  `N128I128`. Type: `Conn<NonZeroI8, Extended<i8>>`, etc.
- Unsigned (5): `N008U008`, `N016U016`, `N032U032`, `N064U064`,
  `N128U128`. Type: `Conn<NonZeroU8, Extended<u8>>`, etc.

Skip `isize`/`usize` (consistent with int family).

The naming prefix `N` is a new 1L+3D shape on the left side, with
`I`/`U` (std primitive) codes on the right. Sign is implicit by
module path (`fixed::i8::N008I008` is signed, `fixed::u8::N008U008`
is unsigned), matching the `Q` convention.

The signed and unsigned cases differ at `Finite(0)`:

- **Signed `nz_int_ext!`**: `inner(Finite(0))` collapses to `-1`
  (the largest NonZero ≤ 0). `floor` of `Finite(0)` = `-1`,
  `ceil` of `Finite(0)` = `+1` — adjoints differ at zero. This is
  the asymmetric-adjoint showcase (alongside `CHARU032` from Plan
  24).
- **Unsigned `nz_uint_ext!`**: `Finite(0)` is below all NonZero
  representatives; `inner(Finite(0))` saturates to `1` (smallest
  NonZeroU). `inner(NegInf)` also saturates to `1`.

These were originally Plan 24 in the survey; they fold into Plan
25 because (1) NonZero<T> is structurally a punctured Q*N*.0
fixed-point, so it belongs with fixed; (2) avoids a transient
`nonzero/` top-level submodule that would just be reshuffled here.

### T4 — Cross-crate iso constants

`FixedX<U0>` (Q*N*.0) and `X` itself. Naming uses the `Q`
convention from §Goal:

- Left side `Q000` = Q-format wrapper with 0 frac bits (sign
  implicit by module: `fixed::i8::Q000` = `FixedI8<U0>`,
  `fixed::u8::Q000` = `FixedU8<U0>`).
- Right side `I008`/`U008` = std primitive (`i8`/`u8`); 1L+3D `I`/`U`
  prefix means "std primitive of N bits."

Different prefix letters on the two sides (`Q` vs `I`/`U`) signal
unambiguously that the Conn straddles a Fixed wrapper and a std
primitive. No disambiguation rule needed.

Conn body: pure transmute via `FixedX::<U0>::from_bits` /
`to_bits` (which for U0 is the identity on the underlying integer
representation).

```rust
pub const Q000I008: Conn<FixedI8<U0>, i8> = Conn::new_iso(
    |q| q.to_bits(),
    |i| FixedI8::<U0>::from_bits(i),
);
```

### Design corrections caught by proptests

The plan got the NonZero Conn shape wrong twice; both were caught by
property-test runs and required mid-sprint fixes:

1. **Original orientation `Conn<NonZero<X>, Extended<X>>`** (T9 fix
   commit). Proptest `u008n008_galois_l` failed at
   `(NonZero(1), NegInf)`: there is no NonZero strictly below the
   smallest representable NonZero, so `inner(NegInf)` cannot be
   placed below the source-side bottom. Flipped to
   `Conn<Extended<X>, NonZero<X>>` (Extended on left, NonZero on
   right per the right-side-coarse rule).

2. **`Conn<Extended<X>, NonZero<X>>`** (T10 fix commit). The
   *symmetric* boundary problem at the top: `(PosInf, NonZero::MAX)`
   broke `galois_l` because `inner(NonZero::MAX) = Finite(MAX)` is
   strictly below `PosInf`. The NonZero target type has no top/bottom
   infinity to mirror Extended's, so the Conn cannot be a full
   triple over Extended. Final form drops the source-side Extended
   wrapper: `Conn<X, NonZero<X>>` with bare primitive source. iN
   already covers the exact value range; the Conn sandwiches
   NonZero's puncture at zero (signed) or saturates `0 → NonZero(1)`
   on the unsigned side.

The asymmetric-adjoint character described in the plan still holds
for the signed Conn (`floor(0) = -1`, `ceil(0) = +1`); both Galois
laws hold because the asymmetry brackets the puncture exactly.
Unsigned uses single-sided `Conn::new_left` — right-Galois fails at
the unsigned bottom plateau by type-theoretic necessity.

  tighter (no infinities to break the boundary plateau).
