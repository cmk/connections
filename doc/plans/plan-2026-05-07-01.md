# Plan 45 — Sprint 4: hifi/epoch.rs §4 relativistic scales (TT, ET, TDB)

## Goal

Add §4 to existing `epoch.rs` (kept monolithic per post-Sprint-2
decision), shipping **3 Conns** for the relativistic time scales:
**F064-only projections** (ETDTF064, ETDEF064, ETDBF064).

This is **revised down from the master plan's 5 Conns**, after
discovering an intrinsic boundary failure in TT's HDUR/NANO
projections.

## Why F064 only — the boundary discovery

Verified `hifitime::Duration::add` at
`hifitime/src/duration/ops.rs:155-211`: **saturates at MAX/MIN, does
not wrap or panic**.

For TT the TAI offset is **+32.184 s** (TT is `future of` TAI). At
TAI HD::MAX the forward projection saturates:

```
to_tt_duration(Epoch_at_TAI_HD::MAX) = HD::MAX  (saturated; lost 32.184 s)
```

Then `from_tt_duration(HD::MAX)` creates an Epoch whose TAI is
`HD::MAX - 32.184 s`. Under `Epoch::Eq`, `back ∘ forward` differs
from the original by 32.184 s ⇒ `iso!` round-trip fails.

A `conn_l!` demotion doesn't recover the law either: at the same
input, `ceil(a) ≤ b ⟺ a ≤ inner(b)` fails on the inner side because
no HD value `b` exists with `inner(b) = a` (i.e., `from_tt_duration(b)`
has TAI ≥ HD::MAX − 32.184 s for any `b`).

GNSS scales avoid this because their TAI offsets are **subtractive**
(GPST = TAI − 19 s, etc.); the result of `to_*_duration(TAI_HD::MAX)`
sits in HD's interior. UTC similarly subtracts leap-seconds. ET and
TDB subtract `J2000_TAI` in their `to_time_scale` arms (see
`hifitime/src/epoch/mod.rs:330,346`), so they don't saturate at
TAI HD::MAX either — but they're lossy at the ~1 ns level (per
`hifitime/tests/epoch.rs:2366,2420`), so HDUR / NANO would imply
false precision.

### Why F064 escapes

The F064 walk operates on the underlying TAI Duration; the
comparison frame is the per-scale `to_*_seconds()`. Fast-paths
(`v > tt_max_secs_f64()` → `PosInf`; `v ≤ tt_min_secs_f64()` →
`Finite(HD::MIN)`) intercept boundary values before the walk,
sidestepping the +32.184 s saturation. F064's f64 plateau is also
≥ 1 s above ~10¹⁵ s magnitude, so the lossy 1 ns iteration in
ET/TDB is well below the f64 ULP at typical use ranges.

### What's deferred

- `ETDTHDUR`, `ETDTNANO`: blocked by TAI HD::MAX boundary saturation.
  Ship if/when a downstream user surfaces the need; would require
  `Conn<Extended<Epoch>, Extended<HD>>` (Extended on both sides) to
  carry the saturation cleanly.
- `ETDEHDUR`, `ETDENANO`, `ETDBHDUR`, `ETDBNANO`: blocked by ≤ 1 ns
  iterative round-trip imprecision. Adding integer-ns projections
  would invite users to over-trust precision the scale doesn't have.

## Tasks

### T1 — §4 banner + float-bound helpers (TT shared with ET/TDB)

**Problem:** The three F064 Conns each need fast-path bounds. TT
needs `tt_min_secs_f64()` / `tt_max_secs_f64()` deriving from
`HD::MIN.total_ns() / 1e9` and `HD::MAX.total_ns() / 1e9`
respectively (TT's projection of TAI HD::MAX would saturate, but
the f64 plateau is ≥ 1 s at that magnitude — so the bound is
effectively the HD-projected range minus the +32.184 s safety
margin). ET/TDB use `J2000_REF_EPOCH.to_tai_duration()` as the
implicit subtractive offset, mirroring the GNSS pattern.

**Approach:** Add `j2000_tai_ns()`, then per-scale `*_min_nanos`,
`*_max_nanos` (interior asymmetric: `min = HD::MIN.total_ns()`
unshifted; `max = HD::MAX.total_ns() - offset_ns`), and
`*_min_secs_f64`, `*_max_secs_f64` (via integer-division pattern).
ET/TDB share `j2000_tai_ns()`; TT's offset is `0 + TT_OFFSET_NS`
where `TT_OFFSET_NS = 32_184_000_000` — the f64 fast-path uses
this as a safety margin against the +32.184 s saturation.

**Source:** `J2000_REF_EPOCH` at `hifitime/src/timescale/mod.rs:36`.
`TT_OFFSET_MS = 32_184` (= 32.184 s when interpreted as ms!) at
`hifitime/src/epoch/mod.rs:70`.

### T2 — ETDTF064 (conn_l!)

**Problem:** `F064 → Extended<Epoch>` via 1ns ULP walks on TAI
Duration, comparison frame `to_tt_seconds`.

**Approach:** Mirror `egpsf064_*` template:

```rust
#[inline]
fn epoch_to_tt_f64(e: Epoch) -> f64 {
    e.to_tt_seconds()
}

def_walk_helpers!(
    f64_etdt_walks,
    f64,
    Epoch,
    shift_epoch_tai,
    epoch_to_tt_f64
);
```

Plus `etdtf064_ceil` / `etdtf064_inner` parallel to `egpsf064_*`.

### T3 — ETDEF064 (conn_l!)

**Problem:** F064 → Extended<Epoch> for ET. ET is lossy via NAIF
Newton-Raphson (≤5 iters, 1e-9 s tol per
`hifitime/src/epoch/mod.rs:306-331`).

**Approach:** Same template as ETDTF064, just `to_et_seconds` widen.
The walk absorbs the ~1ns iteration imprecision into the ULP descent.
Doc-comment cites hifitime's documented limit (≤10 ns per
`initializers.rs:227-231`).

### T4 — ETDBF064 (conn_l!)

**Problem:** F064 → Extended<Epoch> for TDB. ESA polynomial sin/cos
with 3 fixed iters (`mod.rs:332-346`). Same lossy character at the
ns level (≤1 ns per `tests/epoch.rs:2366`).

**Approach:** Same template. `to_tdb_seconds` widen.

## Deferred

- **`ETDTHDUR` / `ETDTNANO`** (TT integer-Duration projections): blocked
  by `Duration::add` saturation at TAI HD::MAX with TT's +32.184 s
  forward offset. Iso roundtrip diverges by 32.184 s; ConnL doesn't
  recover the boundary either. Re-shippable if a downstream user
  surfaces the need by moving to `Conn<Extended<Epoch>, Extended<HD>>`
  with explicit upper-bound saturation; not justified by current
  demand.
- **`ETDEHDUR` / `ETDENANO` / `ETDBHDUR` / `ETDBNANO`** (ET/TDB
  integer projections): blocked by ≤1 ns iterative round-trip
  imprecision inherent to NAIF Newton-Raphson and ESA polynomial
  algorithms. Integer-ns projections would invite users to over-
  trust precision the scales themselves don't have.
- **`f64_etde_walks` / `f64_etdb_walks` Kani harnesses**: widens
  call `f64::sin` inside iterative correction loops. CBMC's symex
  of `sin` at any magnitude tier exceeds practical solver budgets.
- **Conn-level Kani harnesses for ETDTF064 / ETDEF064 / ETDBF064**:
  per Plan 43's "F064 Galois-laws over the walk" defer pattern. The
  walk-step bound is the headline result for F064 Conns; full-
  domain laws stall on the same walk symex.
- **F032 epoch bridges across all scales**: f32 plateau is too wide
  at epoch magnitudes (~10⁹ s × 2⁻²³ ≈ 100s of ULP at typical
  values). Defer indefinitely.
