# Plan 32 ‚Äî `floor_le_ceil` cleanup: convert non-triple markers to one-sided

## Goal

Eliminate every shipped Conn that claims "triple" status (`ViewL + ViewR`)
but doesn't actually satisfy `floor(a) ‚â§ ceil(a)` for every input. After
this lands, every triple marker in the crate is a true adjoint triple
in the categorical sense ‚Äî its `inner` is order-reflecting and the
two-sided helpers (`round`, `truncate`, `interval`, `midpoint`, `median`)
produce mathematically correct results on every input.

## Motivation

A triple `f ‚ä£ g ‚ä£ h` (in this crate's terms: `ceil ‚ä£ inner ‚ä£ floor`)
satisfies the rounding sandwich `floor(a) ‚â§ ceil(a)` everywhere if and
only if `inner` is order-reflecting (equivalently: `g` is fully faithful;
equivalently: the unit of `f ‚ä£ g` is iso; equivalently: the counit of
`g ‚ä£ h` is iso). Proof both directions:

- **Sufficiency**: from the closure laws `inner(floor(a)) ‚â§ a ‚â§ inner(ceil(a))`,
  transitivity gives `inner(floor(a)) ‚â§ inner(ceil(a))`, and order-reflection
  of `inner` lifts this to `floor(a) ‚â§ ceil(a)`.
- **Necessity**: assume `h(a) ‚â§ f(a)` everywhere. Take `x, y ‚àà B` with
  `g(x) ‚â§ g(y)`. Then `x ‚â§ h(g(x)) ‚â§ f(g(x)) ‚â§ y` ‚Äî first step is
  `g ‚ä£ h` kernel, second is the assumption at `a = g(x)`, third is
  L-Galois `f(a) ‚â§ b ‚ü∫ a ‚â§ g(b)` with the given hypothesis. So
  `g` is order-reflecting.

Three Conn families currently ship as triples but have non-injective
(hence non-order-reflecting) `inner`s, so they violate the sandwich
and produce wrong results from `round`/`truncate`:

| Family | Saturation site | Concrete failure |
|---|---|---|
| `ext_int!` (14 invocations) | `_inner` collapses `[-i16::MIN, BELOW]` to `NegInf` and `[ABOVE, i16::MAX]` to `PosInf` | `floor(PosInf) = i16::MAX` vs `ceil(PosInf) = ABOVE` (e.g. `i16::MAX = 32767 > ABOVE = 128` for `I008I016`) |
| `fix_fix_i*!` / `fix_fix_u*!` (~119 Conns) | `_inner = c * RATIO` saturates to `FINE_MAX/MIN` for any non-trivial RATIO | `floor(FINE_MAX) = Coarse::MAX` (FINE_MAX fixup) vs `ceil(FINE_MAX)` (small Coarse value) |
| `STDRU128` | `stdru128_inner` saturates `Finite(n > max_nanos)` to `Finite(StdDuration::MAX)` | `kernel_r` violated at `b = Finite(max_nanos + 1)`: `floor(inner(b)) = Finite(max_nanos) < b` |

The latent bug in the parent Haskell library (`f09sys` at `Finite SystemTime::MAX`)
is the same shape as STDRU128 and is missed by the parent's tests due
to 0-biased boundary sampling. Will be flagged upstream (T6).

## Tasks

### T1 ‚Äî STDRU128 ‚Üí ConnL

Convert `STDRU128` from `triple!` to `Conn::new_l(stdru128_ceil, stdru128_inner)`.

- `src/time/duration.rs`: replace the `triple!` invocation with a direct
  `pub const STDRU128: ConnL<Extended<StdDuration>, Extended<u128>> = ...`
  (mirroring the post-Plan-27 pattern of `TIMENANO`/`OFDTNANO`).
- Drop `stdru128_floor` (it was a thin alias for `stdru128_ceil` anyway).
- Update the doctest to remove the `.floor()` line and explain the
  `floor = ceil` collapse.
- Migrate the `law_battery!` invocation from `full` to `l_only` (drop
  `galois_r`, `closure_r`, `kernel_r`, `monotone_r`).
- Add boundary spot tests: `0`, `max_nanos`, `max_nanos + 1`, `u128::MAX`.

### T2 ‚Äî `ext_int!` macro emits ConnL

Rewrite the `ext_int!` macro in `src/fixed.rs` to emit `Conn::new_l`
instead of a triple marker.

- Drop the `_floor` const fn from the generated impl block.
- Drop the `ViewR` impl.
- Change `pub struct $NAME;` to `pub const $NAME: ConnL<Extended<$A>, $B> = Conn::new_l(_ceil, _inner);`
  (or keep the struct shape and only impl `ViewL`, whichever fits the
  test infrastructure).
- 14 invocations across `src/fixed/i16.rs`, `i32.rs`, `i64.rs`, `i128.rs` ‚Äî
  no per-call-site changes needed beyond the macro rewrite.
- Migrate `tests/int_*_galois.rs` from `ext_int_props!` (which tests
  `galois_upper` AND `galois_lower`) to a one-sided property macro
  (`single_sided_props!`-equivalent for L). The L-side battery covers
  `galois_l`, `closure_l`, `kernel_l`, `monotone_l`, `idempotent`.

### T3 ‚Äî `fix_fix_i*!` / `fix_fix_u*!` macros emit ConnL

Same shape as T2 across `src/fixed/{i,u}{8,16,32,64,128}.rs`.

- Modify each `fix_fix_i*!` / `fix_fix_u*!` macro to drop `_floor` and
  the `ViewR` impl.
- The FINE_MAX/MIN fixups in `_floor` were specifically engineered to
  preserve `galois_r` at the boundary. With ConnL they're moot; drop
  the function.
- The FINE_MIN fixup in `_ceil` (`return FixedI8::<Coarse>::from_bits(i8::MIN)`)
  IS still needed for `galois_l` at the boundary ‚Äî keep it.
- Migrate `props_for_pair!` invocations from `law_battery!` `full` to
  `l_only` across the in-module test blocks (`src/fixed/i*.rs` `mod tests`)
  and the integration tests (`tests/fixed_u*_galois.rs`).

### T4 ‚Äî Strengthen `law_battery!` `full`

Add `floor_le_ceil` as a required check in the `full` subset of
`law_battery!` so future triple markers can't silently re-introduce the
flaw.

- `src/prop/conn.rs:417-481`: add `fn floor_le_ceil` test to the `full`
  arm. Source-side strategy already in scope (`$f`); calls
  `crate::prop::conn::floor_le_ceil(&t, a)` for some triple marker
  reference.
- The test needs the marker (not just the L-Conn / R-Conn), since
  `floor_le_ceil` is `Triple`-bound. Either: change the macro to require
  a triple-marker argument (inconvenient for one-sided users), or split
  the existing `full` into `full` (L+R views, no sandwich) and
  `triple` (L+R views + sandwich). Latter is cleaner.
- Document the equivalence in `prop/conn.rs:160-167` (the `floor_le_ceil`
  predicate's docstring): "`g` order-reflecting ‚ü∫ `floor ‚â§ ceil`
  everywhere".

### Spot checks

- `STDRU128` at `Finite(max_nanos + 1)`: `inner` returns
  `Finite(StdDuration::MAX)` (current), `ceil(inner(b)) = Finite(max_nanos)`,
  `Finite(max_nanos) ‚â§ Finite(max_nanos + 1)` ‚úì (kernel_l holds, kernel_r
  no longer applies ‚Äî single-sided).
- `Q004Q000` at `Fine(7.9375)`: `floor = ceil = Coarse(8)` after
  ConnL conversion (loses the `Coarse(127)` saturation answer);

  since `round` requires `Triple`), forcing the user to use `ceil`
  directly.
- `I008I016` at `Extended::PosInf`: `ceil(PosInf) = 128 = ABOVE`,
  `floor` no longer accessible; `round` no longer callable.

## Deferred

- **Paired `*CEIL` / `*FLOOR` Conns** for the demoted families. Some
  downstream user may genuinely want both rounding directions on
  Q-format conversions. Out of scope for Plan 32; revisit if a real
  use case surfaces.
- **`fix_fix_*!` redesign with wider source backing** so `_inner = c * RATIO`
  doesn't overflow. Would let us ship true triples for cross-precision
  Q-format. Major API redesign; out of scope.
- **Plan 27 reversal**. The 6 time/ Conns that Plan 27 demoted are
  fine as ConnL ‚Äî keep them as ConnL. (They could be re-promoted to
  triples with appropriate rung type changes, but the current shape
  works and no user has complained.)
- **Lattice trait family tests**. Plan 32 is scoped to Conn lawfulness;
  the unwired `prop::lattice::*` predicates and missing concrete
  `Heyting`/`Boolean` impls are tracked separately.
