# Plan 06 — Reparent fixed module: decimal becomes `fixed::decimal`

## Goal

Move the existing decimal SI-prefix ladder out of `src/conn/fixed.rs`
into `src/conn/fixed/decimal.rs`, leaving a thin module root, so the
`conn::fixed::*` namespace can host upcoming `fixed`-crate-native
binary sub-modules in a follow-up sprint.

## Dependency Graph

```
T1 (move + module root) → T2 (import-path updates) → T3 (verify + commit)
```

(All three are landed in a single commit since the import updates are
mechanical fall-out of the move and have to be in the same commit to
keep the build green.)

## Tasks

### T1 — Move `fixed.rs` → `fixed/decimal.rs`

`mkdir src/conn/fixed && git mv src/conn/fixed.rs
src/conn/fixed/decimal.rs`. No content changes inside the file; const
names (`F03F00`, `F12F06`, `F64F06`, etc.) preserved verbatim.

### T2 — New `fixed.rs` module root + import-path updates

Replace `src/conn/fixed.rs` with a thin module root declaring
`pub mod decimal;` plus a one-paragraph doc comment explaining that
binary sub-modules are coming in a follow-up.

Mechanical import-path updates (no functional changes):

| File | Change |
|------|--------|
| `src/conn/sample.rs` | `use crate::conn::fixed::Pico` → `::decimal::Pico`; one doc-link updated |
| `src/property/arb.rs` | `use crate::conn::fixed::{HasResolution, Micro, Pico}` → `::decimal::{...}` |
| `src/conn.rs` | doctest example + tests-mod use stmt rewritten through `decimal::` |
| `src/lib.rs` | type-code table + bullet examples + doctest example rewritten |
| `src/property/mod.rs` | three doc-comment examples rewritten |

### T3 — Verify and commit

`cargo test --workspace`, `cargo clippy --all-targets -- -D warnings`,
`cargo doc --no-deps`. Single commit `task: Reparent fixed module —
decimal becomes fixed::decimal`.

### Properties (must pass)

No new properties. The existing decimal-module proptests keep passing
unchanged from main — only their import paths in test files update.

- `cargo clippy --all-targets -- -D warnings` — clean.
- `cargo doc --no-deps` — clean (one pre-existing warning on
  `src/property/mod.rs:8` about the `[arb]` link being gated; not

End-to-end: a downstream consumer's `use connections::conn::fixed::Pico`
breaks; the migration is one path edit to
`use connections::conn::fixed::decimal::Pico`. Documented in the MR
description as a breaking change.

## Deferred

- **`fixed::{i08,i16,i32,i64}` binary sub-modules** — the actual
  motivation for reserving the namespace. Held back into a follow-up
  MR so this restructure stays small and reviewable. The follow-up
  will use `property::laws` predicates from the start, add `Ple`
  impls for `FixedI*<F>`, and ship saturating-`inner` implementations
  total over the full primitive bit range. The `doc/design.md`
  "Triple-only properties and the role of injectivity" section
  travels with that follow-up since it justifies the saturating
  design.

## Review

Sprint shipped as planned. Single commit, clean diff (664 ins / 654
del — almost entirely the `fixed.rs → fixed/decimal.rs` rename
recognised by git's similarity heuristic plus the new module root).
No `#[ignore]`d properties.

Original scope (this restructure + the four binary sub-modules + the
`design.md` doc + plan/review) was on a previous attempt of this
branch (saved at tag `wip-pre-rebase`). After main absorbed the
property-module-as-public-API sprint, splitting into two MRs let the
restructure land cleanly and let the binary work be ported to use
`property::laws` predicates from the start rather than via a messy
rebase.
