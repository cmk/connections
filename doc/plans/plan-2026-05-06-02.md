# Plan 39 — Kani SMT verification of Galois-connection laws

## Goal

Add a [Kani](https://model-checking.github.io/kani/) proof tree under
`src/kani_proofs/` that promotes every proptest-asserted Galois law on
the integer / Q-format / NonZero / iso Conn families from sampled
spot-checking to **SMT-verified for the entire bit-width domain**.

For the float layer, prove the property the existing
`ulp_steps_bounded` proptest only samples: the f64 → f32 ULP-walk in
`src/float/f32.rs` converges in **≤ 2 iterations** for every finite
non-NaN `f64` — i.e. a hard upper bound on walking behavior.

## Dependency Graph

```
T1 (Cargo.toml + lib.rs cfg-gated module)
  │
  ├──► T2 (int_narrow harnesses)
  ├──► T3 (uint_sat harnesses)
  ├──► T4 (nz_ext harnesses + no-panic)
  ├──► T5 (fix_fix_signed Q-format ladder)
  ├──► T6 (fix_fix_unsigned Q-format ladder)
  ├──► T7 (iso_family harnesses)
  └──► T8 (float/f32.rs walk-step probe shim)
        │
        ├──► T9  (float_walk: walking-bound theorem)
        └──► T10 (float_weaker: finite-domain properties)

T2…T10 ──► T11 (verification gate)
```

T2…T7 are Group 1 (integer / Q-format / NonZero / iso); T9 / T10 are
Group 2 (floats). Independent across siblings — can be implemented
and run in any order after T1.

## Tasks

### T1 — Cargo + module scaffolding

- `Cargo.toml`: add `[lints.rust] unexpected_cfgs = { level =
  "warn", check-cfg = ["cfg(kani)"] }` so `cargo build` / `cargo
  test` don't warn about the proof harnesses' `#[cfg(kani)]` gates.
  No new dependency: Kani injects its own `kani` crate at build time
  when `cargo kani` runs.
- `src/lib.rs`: add `#[cfg(kani)] mod kani_proofs;` after the
  existing `pub mod prop;` declaration.
- `src/kani_proofs.rs` (new): parent module file declaring the eight
  submodules (`int_narrow`, `uint_sat`, `nz_ext`, `fix_fix_signed`,
  `fix_fix_unsigned`, `iso_family`, `float_walk`, `float_weaker`).
  Per CLAUDE.md "no `mod.rs`" rule: parent file co-located with
  child directory.

### T2 — `int_narrow.rs` (10 Conns × 6 props = 60 harnesses)

Local `prove_int_narrow!` macro emits one `mod $conn { #[kani::proof]
fn galois_l/monotone_l/closure_l/kernel_l/idempotent_l/roundtrip_ceil
}` for each Conn. Predicates pulled from `crate::prop::conn` —
already unconditionally `pub` per `src/prop/conn.rs:16,25,30,35,48,80`.

Coverage: every `int_int_narrow!` Conn in `fixed::{i8, i16, i32, i64}`
— `I016I008` … `I128I064`.

### T3 — `uint_sat.rs` (14 Conns × 5 props = 70 harnesses)

Same shape as T2 but R-Galois: `galois_r`, `monotone_r`, `closure_r`,
`kernel_r`, `idempotent_r`. **`roundtrip_floor` intentionally
omitted** — Kani's first-pass run produced a counterexample at `b =
-1` for every Conn in the family: `inner` clips negatives to `0`, so
`floor(inner(-1)) = 0 ≠ -1`. The property does not hold for
right-Galois single-sided Conns whose source is unsigned. The
existing proptest in `tests/int_i*_galois.rs` correctly omits it for
the same reason. Doc comment in the harness file records the
omission.

### T4 — `nz_ext.rs` (60 harnesses across signed + unsigned)

- Signed (full triple): `i8…i128 ↔ NonZero<i8…i128>` — eight props
  per Conn (galois_l, galois_r, closure_l, closure_r, idempotent_l,
  idempotent_r, **ceil_never_panics**, **floor_never_panics**). The
  no-panic harnesses prove the `<NonZero<_>>::new(_).unwrap()` calls
  in `_ceil` / `_floor` (`fixed.rs:375-390`) cannot panic — Kani
  witnesses that the input is provably non-zero on every branch.
- Unsigned (L-only): four props per Conn (galois_l, closure_l,
  idempotent_l, ceil_never_panics).

`arb_nz!` helper macro: `let v: $A = kani::any(); kani::assume(v !=
0); <$NZ>::new(v).unwrap()`.

### T5 — `fix_fix_signed.rs` (Q-format ladder, signed)

L-Galois only per Plan 32 (`src/fixed/i8.rs:136-143` doc). Five props
per Conn: `galois_l`, `monotone_l`, `closure_l`, `kernel_l`,
`idempotent_l`.

Coverage tiered by host width:

- **i8 / i16** — full ladder (21 + 15 = 36 Conns). 8/16-bit symbolic
  state spaces dispatch in milliseconds; full coverage is cheap.
- **i32 / i64 / i128** — boundary pairs only (`Q_max → Q_0`,
  `Q_max → Q_(max-1)`, plus a representative middle rung).
  Structurally-identical harness from narrower widths covers the
  logic; these wider cases anchor the bit-width extremes.

### T6 — `fix_fix_unsigned.rs` (Q-format ladder, unsigned)

Same shape and tiering as T5, applied to `fix_fix_uN!` Conns:
- u8 — 10 representative pairs from the 28-Conn ladder
- u16 — 5 boundary pairs (incl. `Q016Q015` = canonical signed-PCM)
- u32 — 3 boundary pairs (incl. `Q032Q031`)
- u64 — 3 boundary pairs (incl. `Q064Q063`)
- u128 — 3 boundary pairs (incl. `Q128Q127`)

### T7 — `iso_family.rs` (10 Conns × 5 props = 50 harnesses)

Five props per Conn: `galois_l`, `galois_r`, `iso_roundtrip_l`,
`roundtrip_ceil`, `idempotent_l`. Iso means `inner ∘ ceil = id_A`
*and* `ceil ∘ inner = id_B`, so both directions are exact.

Coverage: every `iso!`-built Conn in `fixed::{i8…i128, u8…u128}` —
the `Q000I###` / `Q000U###` cross-crate bridges to the std primitives.

### T8 — Walk-step probe shim in `src/float/f32.rs`

Add two `pub(crate)` wrappers gated on `#[cfg(any(test, kani))]`:

```rust
pub(crate) fn ceil_walk_steps_for_proof(x: f64) -> (f32, u32);
pub(crate) fn floor_walk_steps_for_proof(x: f64) -> (f32, u32);
```

Mirror the existing `ceil_f64_f32` / `floor_f64_f32` bodies (`f32.rs:87,
104`) but return the `(z, steps)` tuple from the walk helpers instead
of dropping `_steps`. Behind `#[cfg(any(test, kani))]` so release
builds don't compile them.

### T9 — `float_walk.rs` — headline walking-bound theorem

Three tiers of walking-bound proof, in escalating input restriction:

| Tier | Input restriction        | Bound proven |
|------|--------------------------|--------------|
| T0   | finite, non-NaN          | walks ≤ 2    |
| T1   | additionally `\|x\| ≤ 1e6` | walks ≤ 1    |
| T2   | single binade `[1, 2)`   | walks ≤ 1    |

`#[kani::unwind(4)]` annotation gives the walk loop room to take up
to 3 iterations before forcing termination; the proof obligation is
that the loop exits within that bound. Each tier appears as its own
`#[kani::proof]` so they can be run independently.

T0 is the headline result: walks bounded for every finite non-NaN f64.
T1 / T2 are corollary tightenings on the well-behaved interior; pair
with T0 to cover saturation / subnormal boundaries.

### T10 — `float_weaker.rs` — finite-domain properties

Eight `#[kani::proof]` fns, all on finite non-NaN inputs:

| Harness                       | Predicate                                                       |
|-------------------------------|-----------------------------------------------------------------|
| `finite_in_finite_out_ceil`   | `inner(ceil(x))` is finite                                      |
| `finite_in_finite_out_floor`  | `inner(floor(x))` is finite                                     |
| `closure_l_finite`            | `x ≤ inner(ceil(x))`                                            |
| `closure_r_finite`            | `inner(floor(x)) ≤ x`                                           |
| `idempotent_l_finite`         | `(inner ∘ ceil)² == (inner ∘ ceil)`                             |
| `floor_le_ceil_finite`        | `floor(x) ≤ ceil(x)`                                            |
| `order_reflecting_finite`     | `inner(b1) ≤ inner(b2) ⟹ b1 ≤ b2`                              |
| `roundtrip_ceil_finite`       | `ceil(inner(b)) == b`                                           |

Each harness restricts to `x.is_finite() && !x.is_nan()` via
`kani::assume`. The full Galois law `galois_l` over the unrestricted
IEEE bit space is **not** verified here — CBMC's FP theory bit-blast
on `(ceil_f64_f32 ⨯ inner)` over arbitrary f64 explodes; defer.

## Deferred

- **Full `galois_l` / `galois_r` on float Conns.** CBMC's FP theory
  bit-blasts the walk-loop case-split; intractable on the
  unrestricted f64 domain. Weaker properties in `float_weaker.rs` cover
  the productive subset.
- **Time / `addr` Conns.** Would require Kani to symbolically execute
  `time` crate calendar arithmetic / leap-second tables;
  cost/benefit too poor.
- **Lattice axioms across composed Conns.** `compose!` / `triple!`
  generate fresh Conn closures whose proof obligations multiply with
  the chain length. Revisit once the per-Conn baseline is in.
- **Median / Birkhoff axioms over `(A,A)→A`.** Proptest already
  covers; SMT cost uncertain and not on the critical path.
- **F032F016 / F064F016.** Same shape as F064F032; once the f32 case
  is proven, these become a copy-paste exercise gated behind the
  `f16` feature.
- **Q-format full ladders for i32/i64/i128 host widths.** Boundary
  pairs only (per T5/T6); the structurally-identical narrower-width
  proofs cover the logic.
- **A `kani:` CI job in `.gitlab-ci.yml`** — too slow per-MR. Could
  be wired up as a tag-push-only advisory job mirroring
  `claude-review`'s `allow_failure: true` posture; deferred until
  user demand.

### Iterations during implementation

Two harness sets needed correction during the run; both surfaced bugs
in the harness, not in the production code:

1. **`uint_sat::*::roundtrip_floor`** failed across the family with a
   counterexample at `b = -1`. Diagnosis: the predicate `floor(inner(b))
   == b` requires `inner` to be surjective onto `B`, but `uint_int_sat!`'s
   `inner` clips the negative half of `B` to `0`. The property doesn't
   hold by construction — the existing proptest in
   `tests/int_i*_galois.rs` correctly omits it for the same reason.
   Removed the harness with a doc comment recording why.

2. **`float_weaker::finite_in_finite_out_{ceil,floor}`** failed for
   `|x| > f32::MAX`: a finite f64 like `1e40` saturates to f32::INFINITY
   under `ceil`, which widens back to f64::INFINITY (not finite). The
   property only holds when `|x| ≤ f32::MAX as f64`. Tightened the
   helper to `arb_in_f32_range_ef64()` and renamed the harnesses to
   `*_in_range`. Documented both the limit and *why* in the helper's
   doc comment.

Both are good examples of the "Kani is finding what proptest missed"
return on investment — but in this case the missing piece was that the
property as originally written *doesn't hold* outside the in-range
slice; not a production bug.

### Solver / unwind tuning

None needed. Per-harness wall times:

- 8/16-bit integer narrowings and Q-format ladders — under 50ms
  each. The full i8 ladder (105 harnesses) finished in under 5
  seconds wall.
- 32/64/128-bit narrowings — under 100ms each.
- Float walk T0 (full finite f64 domain) — 2 harnesses in ~2 seconds
  with `#[kani::unwind(4)]`. T1 / T2 with `#[kani::unwind(3)]` — even
  faster.
- Float weaker `finite_in_finite_out_*_in_range` — the longest float
  proof at 1.9 seconds; everything else under a second.

128-bit wider Q-format harnesses would have been the cost concern in
the original plan; in practice CBMC's bitvector solver dispatched
them within standard timeouts. No `--solver` overrides or
`--default-unwind` tuning required.

### Deviations from the plan

1. **Walking-bound result is sharper than predicted.** The plan said
   "T0 should pass within minutes; if not, drop to T1 / T2". T0
   passed in ~2s on the full finite-non-NaN domain; T1 / T2 became
   *additional* tightenings (proving the bound is actually 1 step,
   not 2, on the well-behaved interior) rather than fallbacks.

2. **No `kani` cargo feature added.** Kani sets `--cfg kani` itself
   when running `cargo kani`; no Cargo feature is needed. Just the
   `[lints.rust] check-cfg = ["cfg(kani)"]` entry to silence
   `cargo build` warnings about the unknown cfg name.

3. **Walk-step probe wrappers gated on `#[cfg(kani)]` only.** Plan
   said `#[cfg(any(test, kani))]`; tightened to `cfg(kani)` because
   the existing `ulp_steps_bounded` proptest already covers the
   non-Kani test path directly. With the broader gate, clippy
   under `--all-targets` flagged the unused wrappers.

### `#[ignore]`d properties

None. Every `#[kani::proof]` either verified, or the property simply
does not hold (in which case the harness was deleted with a
documentation comment, not ignored).

### Recommendations

- **Wire a `kani:` job into `.gitlab-ci.yml`** as advisory-only
  (`allow_failure: true`), tag-push-only. Per-MR is too slow but
  catching regressions on tagged releases is cheap.
- **f16 mirror** — once the `f16` feature is in steady use, copy
  `float_walk.rs` and `float_weaker.rs` shapes onto F032F016 / F064F016.
- **Composed-Conn lattice axioms** are the natural next sprint:
  prove `compose_l!` chains preserve the law battery for the
  worst-case 3-step composition.
- **Document the proof tree in README.md** under a new "Verification"
  section so downstream consumers know the laws are SMT-proven, not
  just sampled.
